# Decision 5 — Docs state current truth: purge the stale "no UI" line

**Decision.** crm's service doctrine docs — `crm/AGENTS.md` and its symlinked
`crm/CLAUDE.md` (one file; edit `AGENTS.md`) — currently assert that crm is "A
pure REST + MCP API with **no UI**…" and, later, that it has "**no UI** and no
token logic." That statement is now **false**: crm serves a human web landing
page (D1–D4). Per the suite doctrine — *docs state current truth; purge what the
change falsified, rewrite in place, no archaeology, no footnotes* — this Decision
**rewrites those lines in place** to state the landing-page truth.

**What changes (doc edits only — no code):**

- Remove the "**no UI**" claim wherever it appears in `crm/AGENTS.md` (the
  opening one-liner and the "What this app is" paragraph).
- Replace it with the current truth, in crm's own voice: crm is a loopback-only
  domain service that serves an **MCP surface for agents** (bearer-gated) **and a
  human web landing page** (session-cookie-gated) under `/srv/crm/`; it still runs
  **no token logic** — nginx remains the sole trust boundary for both doors. Keep
  the "no token logic" half (still true); drop only the "no UI" half (now false).
- Do **not** add a changelog, a "previously…", or a footnote. State the present
  as if it were always so.

**Why this is its own phase, performed by the build loop — not by the doc
author.** The doc author (this design+plan workspace) deliberately does **not**
edit `AGENTS.md` now: code and docs must evolve **together**, so the purge is
sequenced as a plan phase that the build loop performs **after** the landing page
exists (D1–D4 shipped). Doing it earlier would make the docs claim a UI the code
does not yet serve; doing it never would leave a false statement on `main`.

**Scope guard.** Only the "no UI" falsehood is purged. crm's other doctrine — the
five-entity/six-verb MCP domain, the header-trust model, "no token logic," the
loopback bind, the deploy/manifest sections — is **unchanged and still true**;
this Decision does not touch it.

**Verification.** This is a **documentation** Decision with no Go behavior, so it
mints no `R-XXXX-XXXX` ids; it is realized by a **structural** plan phase whose
done bar is a content check rather than an id-tagged test:

- `crm/AGENTS.md` (hence `crm/CLAUDE.md`) no longer contains the string
  `no UI` (case-insensitive) — `grep -i "no UI" crm/AGENTS.md` finds nothing.
- `crm/AGENTS.md` states the landing-page truth: it mentions that crm serves a
  human web **landing page** under `/srv/crm/` alongside the MCP surface.
- crm's `AGENTS.md`/`CLAUDE.md` symlink invariant is preserved (one file; the
  edit goes through `AGENTS.md`), and the suite stays green
  (`bin/check-migrations crm` and `cd crm && go build ./...` unaffected — this is
  a docs-only change).
