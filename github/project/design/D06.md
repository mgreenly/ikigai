# Decision 6 — The landing page and nginx fragment

## Decision

`github` serves the suite's uniform landing page and ships its path-routed nginx
fragment, with **no** consent flow and **no** OAuth routes (the GitHub App is
pre-installed; there is no per-user consent).

**`internal/web/landing.html` is a byte-exact clone of the canonical suite
landing** — the single `internal/web/landing.html` structure every other service
ships. **`crm` is the reference of record** (`crm`, `gmail`, `ledger`, and the
rest share this file identically modulo three text fields). github's `landing.html`
reproduces crm's markup and `<style>` block **verbatim**; only three fields carry
github's own copy:

- the `<title>` suffix — `{{.Service}} · github`;
- the eyebrow — **`GitHub connector`**;
- the one-line description — **`Github connects the suite to GitHub through one
  shared GitHub App and exposes repository, pull request, and issue actions as MCP
  tools.`** — it deliberately omits the producers' "…publishes typed events to the
  event plane" clause, because github owns no event-plane producer.

Everything else is identical to crm's: the centred `<main>` column, the top-left
`Home` link (`<a class="home" href="/">Home</a>`) as the first element inside
`<main>`, the `<section aria-labelledby="page-title">` wrapping the eyebrow, the
`<h1 id="page-title">{{.Service}}</h1>`, and the description, the
`<dl aria-label="Service details">` Service·Version·API grid whose **API** cell is
`<code>POST /mcp</code>` (Version cell `class="version"`), the `<style>` block, and
the embedded `static/` tree (the Carbon `tokens.css` and the woff2 fonts). The page
renders indistinguishably from `/srv/crm/`. The dynamic `service` and `version`
values are injected through `html/template`, so they are HTML-escaped. `web.go`,
`embed.go`, and the embedded font/CSS assets are already canonical (github's
`static/tokens.css` is byte-identical to crm's) and stay put.

Wiring in `Handlers` (D1), in `internal/githubapp/spec.go`:

```go
rt.Handle("GET /{$}", web.LandingHandler(rt.Service(), rt.Version()))
rt.Handle("GET /static/", http.StripPrefix("/static/", web.StaticHandler()))
rt.Handle("POST /mcp", rt.RequireIdentity(
    mcp.NewHandler(client, rt.Version(), rt.Service(), health, rt.Logger())))
```

**Fidelity is proven the canonical way — by cloning crm's own `internal/web` test
set**, not by github-bespoke guards. `github/internal/web/web_test.go` is crm's
`internal/web/web_test.go` reproduced verbatim, swapping crm's literals for
github's (the service name in the handler calls, the eyebrow `GitHub connector`,
the `<h1 …>github</h1>`, and the description) and adapting the two
composition-coupled tests to github's reality: github is a **non-producer** with no
`/feed`, and its Spec lives in `internal/githubapp/spec.go` (not inline in
`cmd/github/main.go`), so the root-route test omits the `/feed` stub and the
composition test reads `../githubapp/spec.go` and asserts github's own wiring
strings (`mcp.NewHandler(client, …, health, rt.Logger())`, no `rt.Events()` /
`rt.Subscriptions()` / Producer). The cloned tests live entirely within `github/`'s
own tree; nothing here tests against a sibling service.

**`etc/nginx.conf`** is the path-routed location fragment (not a vhost), mirroring
`gmail/etc/nginx.conf` with literal `MOUNT=/srv/github/` and port `3203`:

- `= /srv/github/.well-known/oauth-protected-resource` → open PRM bootstrap (no
  auth_request).
- `= /srv/github/` → bare mount root, gated by `auth_request /_session-authn`
  (dashboard browser session), proxied to `/` — the landing page.
- `/srv/github/static/` → session-gated static assets.
- `/srv/github/` (prefix) → every other path gated by `auth_request /_authn` (the
  bearer gate) with the identity-header hygiene and the 429-faithful `error_page`
  block, proxied to loopback `:3203` — this covers `/mcp`.
- `= /srv/github/pr` → `return 404`: the loopback-private twin (D5) must never be
  reachable through the public front door (defence in depth; the handler
  self-guard is primary).

github has **no `/feed`**, so — unlike the producer services — the fragment
carries no `/srv/github/feed` block; nothing under the mount serves a feed.

**`etc/manifest.env`** declares `APP=github`, `MOUNT=/srv/github/`, `PORT=3203`,
`MCP=true`, `DEFAULT=false`, and **no** `FEED=` line and no outbox retention keys
(non-producer, D1). `etc/deploy.env` mirrors the other connectors' workstation
routing.

## Rejected

- **A github-bespoke fidelity harness** — a byte-for-byte **golden fixture**
  (`internal/web/testdata/landing.golden.html`), a **structural-contract** marker
  check, a **token-integrity** subset check, and a hostile-input **HTML-escaping**
  test. No sibling service carries any of them, and they did not prevent the very
  drift they were meant to catch: phases 7–9 shipped a github-only landing variant
  (blue eyebrow, viewport-pinned Home link, renamed detail classes) with every one
  of those guards green, because a golden regenerated from a drifted template pins
  the drift rather than catching it. The suite verifies its landing by the shared
  crm-style test set over a **byte-exact canonical clone**; github now does the
  same, and the golden fixture is deleted.
- **A consent binary (`cmd/consent`) and OAuth callback routes.** gmail needs them
  for per-user Google OAuth; the GitHub App is installed once at the org, so there
  is no consent step and no callback — omitted entirely.
- **A `/feed` block in the fragment.** github publishes no events and mounts no
  feed; a block would guard a route that does not exist.
- **A shared landing handler.** The suite convention is each app owns its landing
  page, template, and embedded assets under its own tree; github embeds its own
  copy of `tokens.css` and the fonts.

## Verification

- `R-EVZ3-VXJZ` — `GET /` returns `200 text/html; charset=utf-8`; the service name
  appears exactly three times (title, `<h1>` heading, and the Service detail cell)
  and the running version exactly once. Asserted offline against `LandingHandler`
  with a distinct service/version pair.
- `R-XSOU-THYE` — the landing HTML links **only app-local** static assets: it
  contains `href="static/tokens.css"` (mount-relative) and none of the forbidden
  forms — an origin-absolute `href="/static/tokens.css"`, a cross-service
  `dashboard` / `/srv/dashboard` reference, or any `http://` / `https://` asset
  URL. Asserted offline against `LandingHandler`.
- `R-XTWR-79P3` — the document head preloads the two self-served woff2 fonts
  (`space-grotesk.woff2`, `ibm-plex-sans.woff2`) as
  `<link rel="preload" as="font" type="font/woff2" crossorigin href="static/fonts/…">`,
  and the served `tokens.css` references each of those files via a matching
  self-served `url('fonts/…')`. Asserted offline against `LandingHandler` and
  `StaticHandler`.
- `R-7NJI-UTHM` — `GET /` renders the canonical suite landing layout: the eyebrow
  `GitHub connector`, the exact one-line description above, the
  `<section aria-labelledby="page-title">`-wrapped `<h1 id="page-title">github</h1>`,
  and a `<dl aria-label="Service details">` Service / Version / API grid whose
  **API** cell is `<code>POST /mcp</code>` and whose Version cell carries
  `class="version"`. A degenerate page that shows only name + version fails.
  Asserted offline against `LandingHandler`.
- `R-7PZB-MCZ0` — the `Home` link (`<a class="home" href="/">Home</a>`) leads the
  page and uses the canonical anchored styling: a `.home` rule set
  `position: absolute; top: var(--space-8);` within a `position: relative` `main`,
  with `.home:hover, .home:focus-visible { color: var(--color-text); }`. Asserted
  offline against `LandingHandler`.
- `R-EX70-9PAO` — `GET /static/tokens.css` returns the embedded stylesheet with
  `Content-Type: text/css; charset=utf-8`, and each embedded woff2
  (`space-grotesk`, `ibm-plex-sans`, `ibm-plex-mono-400`, `ibm-plex-mono-500`)
  returns `Content-Type: font/woff2` with a non-empty body. Asserted offline
  against `StaticHandler`.
- `R-XV4N-L1FS` — the served `tokens.css` declares an embedded `@font-face` for
  each of the four fonts with a self-served `url('fonts/…')` source and the
  `'Space Grotesk'` / `'IBM Plex Mono'` families, and contains **no**
  origin-absolute `url('/static/fonts/…')`. Asserted offline against
  `StaticHandler`.
- `R-XWCJ-YT6H` — every `@font-face` block in the served `tokens.css` uses
  `font-display: optional` (the `font-display: optional` count equals the
  `@font-face` count) and the stylesheet contains no `font-display: swap`. Asserted
  offline against `StaticHandler`.
- `R-XXKG-CKX6` — mounted on an exact root pattern (`GET /{$}`), the landing
  handler answers `GET /` but does **not** shadow sibling routes: `POST /mcp`, `GET
  /health`, `GET /.well-known/prm`, and an unknown path each reach their own
  handler (the unknown path `404`s), none returning the landing `<h1 …>github</h1>`.
  github serves no `/feed`, so the feed route is not among the stubs. Asserted
  offline over an `http.ServeMux` wired like the composition root.
- `R-XYSC-QCNV` — the composition root `internal/githubapp/spec.go` mounts the
  landing at `GET /{$}` **ungated** (its `web.LandingHandler(…)` line is not wrapped
  in `RequireIdentity`) and keeps the MCP surface wired behind
  `rt.RequireIdentity(mcp.NewHandler(client, rt.Version(), rt.Service(), health,
  rt.Logger()))`. Asserted offline by reading `spec.go` and checking the wiring
  lines.
- `R-EYEW-NH1D` — the `etc/nginx.conf` fragment session-gates the bare mount root
  (`auth_request /_session-authn` on `= /srv/github/`), bearer-gates the mount
  prefix (`auth_request /_authn` on `/srv/github/`), returns `404` for
  `= /srv/github/pr`, opens the PRM well-known, and contains **no**
  `/srv/github/feed` block. Asserted by a fragment parse/grep test
  (`internal/web/nginx_test.go`, like gmail's) over the shipped file.
