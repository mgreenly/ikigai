# Decision 4 — The MCP tool surface

## Decision

`internal/mcp` declares github's domain tools over the **shared `appkit/mcp`
transport** (the migration off the former hand-rolled JSON-RPC layer is D8):
`Tools(client, logger)` returns the tool table, and `NewHandler` wraps it with
`appkitmcp.New(...)`, mounted at `POST /mcp` behind `rt.RequireIdentity`,
carrying **no** token logic (nginx has already introspected the request and
injected identity). The shared transport owns JSON-RPC dispatch
(`initialize`, `tools/list`, `tools/call`), the protocol revision, and the
chassis `health`/`reflection` tools; github declares only the domain verbs. Wire
format, result shape (`structuredContent` + mirrored text), and typed error codes
are D8's contract.

**Bare verb names.** Tools register **bare** verbs (no service prefix), exactly
like `gmail` (`toolPrefix = ""`): the suite discovery layer in `prompts`
re-qualifies each bare verb to `ikigenba_github_<verb>`. The surface is the 14
domain verbs (D3) plus the two chassis verbs the transport appends:

```
repos_list  repo_get
pr_list  pr_get  pr_comment  pr_review  pr_merge
issue_list  issue_get  issue_create  issue_comment  issue_update
file_get  file_put
health  reflection        (chassis-provided; not registered by github)
```

Each domain verb has a JSON input schema declaring its parameters (e.g. `pr_get`
requires `repo` and `number`; `pr_review` requires `repo`, `number`, `event`,
optional `body`) and a declared `outputSchema` (D8). `tools/call` decodes the
arguments and dispatches to the matching `internal/gh.Client` method through the
`GitHubClient` interface (kept, unchanged, for offline tests):

```go
// GitHubClient is D3's 14-method client slice the tools drive; it is the seam
// that lets the tool table run against a fake in offline tests.
type GitHubClient interface { /* ReposList … FilePut — see internal/mcp */ }
```

**`health` is the live auth proof.** The chassis `health` verb returns the health
envelope *and* drives a real authenticated GitHub call via github's `Health`
closure (D2: resolve the org installation / mint a token). Success means the app
credentials actually work against `@ikigenba`; failure is reported loudly in the
envelope, never a silent OK. This is the seam that realizes D2's `R-DMUT-QF4A`
end to end. github supplies the closure (`Spec.Health`); the transport renders it.

**`reflection` reports an empty event graph.** github is neither producer nor
consumer, so `reflection` self-describes **zero** published events and **zero**
subscriptions. The handler is built with an empty `eventplane/outbox.Registry`
and a nil subscriptions provider (`Spec.Events` empty, `Spec.Subscriptions` nil).

**Owner provenance → logs, never GitHub.** On every **write** verb
(`pr_comment`, `pr_review`, `pr_merge`, `issue_create`, `issue_comment`,
`issue_update`, `file_put`), the tool handler emits one structured `slog` line
carrying the `X-Owner-Email`, the `X-Client-Id`, the verb, and the target
(repo + number/path) — the audit record of who triggered the action. That owner
identity is passed to `slog` **only**; it is never forwarded into any GitHub
request body (D3 already omits owner fields). Read verbs need no such line. The
handler reads identity from the `server.Identity` value the transport threads
from the `X-Owner-Email`/`X-Client-Id` headers.

**Error surfacing.** A `Client` error (`ErrNotFound`, `ErrInvalid`, `ErrAppAuth`,
or a transport failure) becomes a tool result with `isError: true`, a clear
message, and a typed `structuredContent.code` from the shared vocabulary — never
a panic or a dropped connection. The exact failure→code mapping is D8.

## Rejected

- **Service-prefixed tool names** (`ikigenba_github_pr_get` at registration).
  Rejected to match the suite convention: peers register bare verbs and the
  `prompts` discovery layer qualifies them; double-qualifying would break
  discovery.
- **A generic passthrough `github_request` tool.** Rejected: an agent handed raw
  HTTP-to-GitHub bypasses the fixed, reviewable verb set and the bot-only
  attribution guarantees; the explicit verb surface is the product surface.
- **Stamping owner identity into the GitHub write** (comment trailer, commit
  author). Rejected by product — repos are mostly public and owner PII must not be
  published; the log line is the audit substitute.
- **Reporting a placeholder event in `reflection`.** github publishes nothing;
  reflecting a fake event would misdescribe the service.

## Verification

- `R-EEWI-J569` — `initialize` returns a protocol/serverInfo result, and
  `tools/list` advertises exactly the 14 **bare** domain verbs above, each with a
  non-empty input schema, plus the chassis-provided `health` and `reflection`
  (github registers neither itself — the reserved names come from the transport,
  and there is no duplicate). Asserted offline against the handler with a fake
  client.
- `R-EHCB-AONN` — `tools/call` for a verb with a missing or malformed required
  argument (e.g. `pr_get` without `number`) returns an `isError` result and does
  **not** call the client or panic. Asserted offline. (The typed `validation`
  code on that result is D8's argument-validation verification.)
- `R-EIK7-OGEC` — The tool handlers read caller identity from the `X-Owner-Email`
  and `X-Client-Id` request headers (threaded by the transport as `server.Identity`
  after `RequireIdentity`) and carry no bearer/token parsing of their own.
  Asserted offline by setting headers and observing them threaded to the
  log/dispatch.
- `R-EJS4-2851` — Each **write** verb emits exactly one structured log line
  carrying the caller's `X-Owner-Email`, the verb, and the target; and the GitHub
  request the write produces carries **no** owner-identifying field (cross-checked
  with D3). A read verb emits no such provenance line. Asserted offline with a
  captured `slog` handler and the fake client's recorded request.
- `R-EL00-FZVQ` — The `health` verb returns the health envelope and its success
  reflects a **real** authenticated GitHub call (D2 `R-DMUT-QF4A`): against the live
  suite it succeeds only when the app can authenticate to `@ikigenba`, and reports
  failure loudly (a non-OK envelope) when it cannot. Offline, the auth call is
  faked; the live proof is the `bin/start` + `health` smoke.
- `R-EM7W-TRMF` — The `reflection` verb reports an **empty** published-event set
  and **empty** subscriptions (github is neither producer nor consumer). Asserted
  offline against the handler.
- `R-ENFT-7JD4` — A `Client` error (`ErrNotFound` / `ErrInvalid` / `ErrAppAuth` /
  transport) from a `tools/call` becomes an `isError` tool result with a
  descriptive message; the transport stays up and returns a well-formed JSON-RPC
  response. Asserted offline with a fake client returning each error. (The typed
  code carried on that result is D8's failure→code mapping.)
