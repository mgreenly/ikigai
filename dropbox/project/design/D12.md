# Decision 12 — MCP surface over `appkit/mcp`: `internal/mcp` becomes the tool table

**Problem.** dropbox's `internal/mcp` carries the copy-pasted JSON-RPC transport
(`ServeHTTP`, the `initialize` / `notifications/initialized` / `tools/list` /
`tools/call` dispatch, `writeJSONRPCError`, the result envelopes, the local
`Identity` type) plus its own `health` and `reflection` implementations
(`toolHealth`, `toolReflection`, `renderSubscriptions`,
`reflectionUnknownTypeError`) — the same near-identical plumbing every service
duplicated. The chassis now owns all of it (appkit D8 transport, D9 standard
tools, consumed here as a fixed external contract). What is genuinely dropbox's is
its two **read** domain tools — `list` and `get` — their descriptors, schemas,
validation, pagination/cursor logic, and the sentinel→error-code mapping.

**Decision.** `internal/mcp` is reshaped from a transport into a **declaration**:

```go
// internal/mcp — what remains.
const Instructions = /* dropbox's MCP instructions string, verbatim */

// Tools declares dropbox's two domain tools over the domain Service: list and
// get (descriptors + schemas + handlers unchanged in wire content — the
// case-insensitive path scoping and cursor pagination of list, the required-path
// validation, 25 MiB too_large cap, rev-pin conflict, and base64 body of get,
// and the shared {error:{code,message}} sentinel envelope). health and
// reflection are NOT declared — the chassis registers them (their inputs all
// come from Router accessors).
func Tools(svc *dropbox.Service) []mcp.Tool

// NewHandler assembles the /mcp http.Handler at the composition root:
// appkit/mcp.New over Instructions + Tools(svc) + the Router-threaded
// Service/Version/Health/Events/Subscriptions.
func NewHandler(svc *dropbox.Service, rt *appkit.Router) (http.Handler, error)
```

- **Deleted:** the JSON-RPC plumbing (`jsonRPCRequest`, the `ServeHTTP`
  dispatch, `writeJSONRPCResult`/`writeJSONRPCError`, `idOrNull`, the local
  `Identity` type, the `handleToolCall`/`dispatchTool` router), the local
  `toolHealth`/`toolReflection`/`renderSubscriptions`/`reflectionUnknownTypeError`
  implementations, and the local result-envelope helpers
  (`toolResultText`/`toolResultErr`/`toolResultJSON` — replaced by the chassis
  `mcp.TextResult`/`mcp.ErrorResult`/`mcp.JSONResult`).
- **Kept:** the descriptor/schema helpers (`desc`/`obj`/`descTyp`, now producing
  `mcp.Tool` values), the `list` and `get` descriptions and schemas verbatim, the
  two handlers' bodies (the `svc.List` pagination + 8-char hash abbreviation, the
  `svc.Content` fetch + `maxGetBytes` cap + rev-pin), the `toolErr` sentinel
  mapping (`not_found`/`conflict`/`validation`/`internal`) and the
  `toolErrorJSON` body shape, and the `firstN` helper — the wire bytes agents see
  for `list`/`get` are unchanged.
- **The wire surface is unchanged for the domain tools:** the read tools
  (`list`, `get` declared) keep the same descriptions, schemas, results, and
  error envelopes, and the same `initialize` instructions. (The declared table
  has since grown — D19 adds the four write tools; the partition and
  chassis-ownership rules of this Decision are unchanged, and the current
  declared set and surface count are owned by D19's R-KRXK-IQE2.) `health` and
  `reflection` become the **chassis-standard** tools: the chassis `health`
  renders `appkit.Envelope(version, service, details)` **plus** the caller's
  `owner_email`/`client_id` (appkit/mcp — byte-identical to dropbox's current
  `toolHealth`, which added the same identity keys over the same reporter
  details), and the chassis `reflection` renders the `publishes`/`subscribes`
  index and per-type detail from `rt.Events()`/`rt.Subscriptions()`. The chassis
  owns their descriptions and their unknown-`event_type` corrective envelope; any
  dropbox test that asserted the retired **local** reflection error wording
  aligns to the chassis behavior (appkit D9 R-MOQJ: an `isError` result naming
  the known types). This matches the notify and crm conversions.

**The composition root.** `cmd/dropbox/main.go` swaps
`mcp.NewHandler(svc, rt.Version(), rt.Service(), rt.Health(), rt.Events(), rt.Subscriptions())`
for `handler, err := mcp.NewHandler(svc, rt); … rt.Handle("POST /mcp", rt.RequireIdentity(handler))`.
Mounting stays in `Spec.Handlers` (one line), parallel to appkit D8/D7: the tool
table closes over the domain Service built over `rt.DB()`, which only exists
inside the hook. The loopback `GET /content` and `GET /list` byte routes and the
producer/Workers wiring are untouched.

**Rejected.**
- **Keeping the local transport** — the status quo; a protocol fix would still be
  a many-service edit.
- **Declaring health/reflection in dropbox's table** — the chassis reserves the
  names (appkit D8/D9) precisely so services cannot fork the standard tools; a
  table declaring `health` or `reflection` fails `New`'s reserved-name check.
  dropbox's identity-carrying health and event-graph reflection are already the
  chassis behavior.
- **Dissolving `internal/mcp` into `cmd/dropbox`** — unlike the post-D11 web
  remnant, the `list`/`get` schemas, pagination/cursor logic, size cap, and
  sentinel→code mapping with their test harness are a real package.

**Verification** (through the composition-root seam — the appkit/mcp handler
built by `NewHandler` over a domain `dropbox.Service`, driven through the real
`ServeHTTP` seam via `server.New`, exactly as the crm/notify tool tests do; the
existing `list`/`get` behavioral tests keep their assertions and swap only the
handler constructor).
- R-QQJT-LKCV — `tools/list` through the assembled handler returns **exactly**
  the dropbox-declared domain tools **plus** `health` and `reflection` supplied
  by the chassis, and nothing else — proving the table/chassis partition (a
  table that declares its own health/reflection fails `New`'s reserved-name
  check; a surface missing them, or still advertising the retired `whoami`,
  fails the count). The declared set and its current count are pinned by D19's
  R-KRXK-IQE2.
