# Decision 24 â€” Canonical absolute paths at the Service seam + logged mutation errors

**Problem.** The filesystem API documents every path as absolute
(`docs/filesystem-api.md`: "All paths are absolute, slash-separated paths in the
shared mirror namespace"), but nothing enforces or repairs that. A relative
path (`e2e/x` instead of `/e2e/x`) is accepted verbatim: the mirror layer
confines it identically, but the index keys on the raw string (`foldPath` only
lowercases), so `stat("e2e/x")` and `stat("/e2e/x")` are different keys â€” and
the event append fails outright, because `eventplane/routing.ValidSubject`
requires a leading `/`. The observed failure mode (live, scripts e2e,
2026-07-15): `POST /mkdir?path=e2e-suite-check` half-succeeds (mirror + index,
no event), then `PUT /content?path=e2e-suite-check/artifact.txt` writes the
mirror file, dies in `outbox.Append` inside the index tx, rolls back, and
`writeMutationError` answers a bare, **unlogged** 500. The scripts runtime's
own `describe` example (`suite.files.put("reports/summary.pdf", â€¦)`) walks
straight into this. There is exactly one root, so a relative path has exactly
one meaning; the fix is to canonicalize, not reject.

**Decision.**

**1. One canonicalization seam.** A package-level helper in
`internal/dropbox`:

```go
// normalizePath maps any externally supplied mirror path onto its canonical
// absolute display form: leading slash, lexically cleaned. Empty input is
// returned empty (a caller-level "unset"/"root" sentinel, e.g. List's
// list-everything). Purely lexical â€” confinement stays Mirror.resolve's job.
func normalizePath(p string) string {
	if p == "" {
		return ""
	}
	return path.Clean("/" + p)
}
```

`path.Clean("/"+p)` is idempotent for already-absolute input (`/foo/bar` â†’
`/foo/bar`), collapses `//` and `.` segments, resolves `..` lexically within
the rooted namespace (`/a/../b` â†’ `/b`; escapes above the root clamp to `/`),
and strips trailing slashes (`/foo/` â†’ `/foo`; `Clean` never returns a
trailing slash except for `/` itself, which `validateWritePath` already
rejects for writes).

**2. Applied at the domain Service boundary, first thing, every verb.** Each
externally reachable `Service` method canonicalizes its path argument(s)
before any validation, mirror, index, or event use:

- writes: `Write(path)`, `Mkdir(path)`, `Delete(path)`, `Move(from, to)`
  (both arguments);
- reads: `Stat(path)`, `List(path, â€¦)` (prefix; empty stays empty =
  everything), `Content(path, rev)`.

This single seam covers every caller at once â€” the loopback HTTP handlers
(`/content`, `/mkdir`, `/move`, `/stat`, `/list`) and all six MCP tools, which
already funnel through these methods. The sync engine's download path
(`applyUpsert`/`applyMkdir`/`applyRename`/`applyDelete`) is **not** routed
through `normalizePath`: Dropbox delta entries already carry canonical
absolute display paths, and those internal apply methods are not an external
boundary. `validateWritePath`'s existing rejections (`""`, `"/"` â†’
`ErrValidation`) run **after** normalization and are unchanged.

**3. Mutation errors are logged before the 500.** `writeMutationError` grows a
logger: the loopback mutation handlers (`WriteHandler`, `MkdirHandler`,
`MoveHandler`) pass the Service's `*slog.Logger`, and any error that falls
through to the `internal error`/500 branch is first logged at `ERROR` with the
route and the underlying error. The 4xx branches (validation, not-found) stay
unlogged â€” they are the caller's mistake, already discriminated by D16's error
contract (R-BZLK-E9VW, R-C0TG-S1ML). The response bodies do not change; only
observability does.

**4. Docs state the contract.** `docs/filesystem-api.md`'s path paragraph
gains one sentence: relative input is accepted and canonicalized to the
absolute form (leading slash, cleaned); the absolute form is canonical and is
what the index, events, and `content_url`s carry.

**Rejected.**

- **Reject relative paths with `validation`/400.** Strictness without payoff:
  with a single root, `foo/bar` cannot mean anything but `/foo/bar`, so
  rejection adds a failure mode while preventing no ambiguity. It would also
  leave every existing relative-path caller (the scripts `describe` example
  among them) broken instead of retroactively correct.
- **Normalize in each HTTP handler / MCP tool.** Scatters the rule across two
  surfaces that must never drift; the Service methods are the single choke
  point both already share.
- **Normalize inside `foldPath`.** Overloads the case-folding lookup helper
  with input repair, and would silently re-key the sync engine's internal
  apply path too â€” the one caller that must stay verbatim.
- **Relax `eventplane/routing.ValidSubject`.** The leading `/` is the
  delimiter between kind and subject in routing keys
  (`dropbox:create/bills/**`); relaxing it reworks the plane's grammar to
  accommodate malformed input dropbox should repair at its own edge.

**Verification.**

- R-54T0-VFZG â€” the live-bug regression, end to end on the real substrate
  (real SQLite index, tempdir mirror, real eventplane outbox on the same DB):
  `PUT /content?path=e2e/artifact.txt` (no leading slash) answers 200; the
  mirror file exists; the index row's path is `/e2e/artifact.txt`; and the
  outbox now holds a `create` event whose subject is `/e2e/artifact.txt`. A
  wrong implementation (today's) fails at the event append with a 500 and an
  empty outbox.
- R-560X-97Q5 â€” canonicalization is uniform across verbs and reads: a file
  written as `/e2e/x` is found by `GET /stat?path=e2e/x` and listed by
  `GET /list?path=e2e`; `POST /mkdir?path=e2e//sub/../sub2` indexes
  `/e2e/sub2`; `DELETE /content?path=e2e/x` removes the file written as
  `/e2e/x`; `POST /move?from=e2e/a&to=e2e/b` moves the file indexed as
  `/e2e/a`. Discriminating property: the relative and messy spellings resolve
  to the **same index key** as their canonical absolute equivalents, not to
  parallel entries.
- R-578T-MZGU â€” the MCP surface normalizes identically through the shared
  Service seam: MCP `put` with `path: "e2e/mcp.txt"` succeeds and the
  subsequent MCP `get` with `path: "/e2e/mcp.txt"` returns the same bytes
  (and vice versa).
- R-58GQ-0R7J â€” a mutation that fails past the discriminated 4xx branches
  (injected failing event sink or store) still answers 500, and the handler
  first emits an `ERROR`-level `slog` record carrying the underlying error;
  today's implementation fails this by logging nothing.
