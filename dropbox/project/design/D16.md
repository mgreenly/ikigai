# Decision 16 — The filesystem write API: Service write methods + loopback routes

**Problem.** dropbox exposes only **reads** (`GET /content` bytes, `GET /list`
discovery) and the domain service is read-only ("there are no write verbs"). The
suite's services need to **create, overwrite, delete, move** files and
directories in the shared folder. That capability is delivered as an **expanded
dropbox API** (option B) — dropbox stays the sole owner of the directory and its
index; a service never touches the raw files, it calls dropbox. The loopback
routes are the primary, streaming, near-native surface (the MCP mirror of these
is D19).

**Decision.** Add mutating methods to `Service` and mount their loopback routes at
the composition root beside `/content`/`/list`. Each mutation runs the mirror
write **then** an atomic index+event(+upload-queue) transaction — the same outbox
discipline the download path uses.

```go
// internal/dropbox/service.go — the write surface (all take the caller's client
// id for event origin, D18, and enqueue an upload, D17).
func (s *Service) Write(ctx context.Context, path string, src io.Reader, clientID string) (FileRow, error)
func (s *Service) Mkdir(ctx context.Context, path, clientID string) error
func (s *Service) Delete(ctx context.Context, path, clientID string) (removed int, err error)
func (s *Service) Move(ctx context.Context, from, to, clientID string) error
```

**Routes** (bare internal paths; nginx strips the `/srv/dropbox/` prefix; mounted
through `Spec.Handlers` beside the existing byte routes, and **self-guarded by the
identity-header check** exactly like `/content` today — loopback-private, never
public):

| method + path | maps to | behavior |
|---|---|---|
| `GET /content?path=` | `Service.Content`/`Mirror.Open` | streaming read (D14) |
| `PUT /content?path=` | `Service.Write` | stream body → mirror, upsert index, emit event, enqueue upload; returns `{path,size,content_hash,rev}` |
| `DELETE /content?path=` | `Service.Delete` | remove file **or** directory (recursive); idempotent |
| `POST /mkdir?path=` | `Service.Mkdir` | create an (empty) directory (D15) |
| `POST /move?from=&to=` | `Service.Move` | rename/move a file or directory in one op |
| `GET /list?path=&cursor=&limit=` | `Service.List` | recursive discovery incl. directories (D15) |
| `GET /stat?path=` | `Service.Stat` | one entry's metadata (file or dir) |

- **The write seam.** `Write` streams to the mirror via `WriteFrom` (D14),
  computing hash+size, then in ONE tx upserts the `files` row, appends the
  lifecycle event (kind `create` if the path is new, kind `modify` if it
  existed — D22 addressing), and inserts the coalescing upload-queue row (D17); `Ring()` fires
  after commit. The event is emitted **iff** the index changed — the existing
  atomic-seam invariant, now driven from a local write instead of a Dropbox
  delta.
- **Local-commit contract.** A route returns success as soon as the bytes are
  durable in the mirror and the tx is committed. The push to Dropbox is
  **asynchronous** (D17) — the caller never blocks on the network. This is what
  meets the near-native bar and decouples suite writes from Dropbox availability.
- **Confinement.** Every path is resolved through `Mirror.resolve`; a path that
  escapes the mirror root (`../…`) is rejected with a validation error and
  **nothing** is written. The shared namespace is flat and unenforced between
  services (any service may write anywhere) — confinement is only to the mirror
  root, not per-service.
- **Origin.** The caller's `X-Client-Id` (nginx-injected for external callers; set
  by the calling service for loopback-to-loopback calls) is threaded into the
  emitted event's `origin` (D18).
- **Move semantics.** A cross-path move emits `delete` (subject `from`) +
  `create` (subject `to`) events (path-keyed consumers see the old path gone and the new
  present) but enqueues a **single** `files/move_v2` upload (no bytes
  re-uploaded, D17) — one op on the wire, honest events on the feed.

**Rejected.**
- **A shared filesystem mount (option A).** Punches through the process-isolation
  trust model and makes the index/events/hashes silently stale the instant a
  service writes behind dropbox's back. dropbox must own every mutation to keep
  the index coherent and to push.
- **Synchronous push (block the write on Dropbox).** Gates every write on
  Dropbox API latency and cascades Dropbox outages into the suite — rejected in
  favor of the local-commit + async-queue contract (D17).
- **Per-service path scoping.** Adds an ownership model and per-write identity
  enforcement to defend cooperating first-party services against each other,
  which is not the threat model on a single-tenant box where nginx is the sole
  trust boundary. Organization is convention, like the human inbox convention.

**Verification** (service tests over temp mirror+DB with a recording event sink;
routes through `net/http/httptest` against the shipped handler wiring; a fake
upload sink stands in for Dropbox — the *real* Dropbox contract is D17's live
smoke, not asserted here).
- R-K4RH-93AV — `PUT /content?path=/a/x.md` streams the body into the mirror and
  upserts the index; a subsequent `GET /content?path=/a/x.md` returns the exact
  bytes and the response reports the written `size`/`content_hash`.
- R-K5ZD-MV1K — a write whose resolved path escapes the mirror root (e.g.
  `../etc/x`) is rejected with a validation error and **no** file is created
  inside or outside the root (confinement holds).
- R-K77A-0MS9 — `DELETE /content` removes an existing file and is **idempotent**:
  deleting an already-absent path returns success (not an error) and emits no
  spurious event.
- R-K8F6-EEIY — `POST /move?from=&to=` relocates a file in one call — `from` is
  absent and `to` is present afterward — and emits a `delete` event for `from` plus a `create` event for `to`.
- R-K9N2-S69N — `GET /stat?path=` returns metadata for both a file and a
  directory (D15), and returns `not_found` for an unknown path.
- R-KAUZ-5Y0C — a `PUT` to a **new** path emits a `create`-kind event while a `PUT`
  overwriting an **existing** path emits a `modify`-kind event (the created-vs-modified
  discriminator is driven by index presence, not by the caller).
