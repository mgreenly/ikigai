# ANSI 256-Color Support

## Overview

Add 256-color support to ikigai's terminal rendering. Colors are stored inline as ANSI escape sequences within text buffers. Input strips color codes; internal code paths add colors at render time.

## Design Decision: Inline Escape Sequences

Colors are embedded directly in text as standard ANSI SGR (Select Graphic Rendition) sequences.

**Why inline:**
- Text and styling travel together naturally
- No parallel data structures to synchronize
- Appending styled text is just appending bytes
- Standard format compatible with external tools and logs

**Alternative considered:** Separate metadata arrays mapping byte ranges to styles. Rejected because character-level coloring (syntax highlighting, inline emphasis, colored words within sentences) would require complex span tracking that shadows every text operation.

## Scope

**256-color only** (SGR 38;5;N and 48;5;N). True color (24-bit RGB) is out of scope for initial implementation.

**Output only.** Colors are generated by internal code paths. User input has color codes stripped.

## ANSI Escape Sequence Format

256-color sequences:
```
\x1b[38;5;Nm    // Set foreground to color N (0-255)
\x1b[48;5;Nm    // Set background to color N (0-255)
\x1b[0m         // Reset all attributes
```

Sequence length: 10-12 bytes for 256-color, 4 bytes for reset.

## Architecture Impact

### Width Calculation (HIGH impact)

Three locations calculate display width by iterating UTF-8 codepoints:

| File | Function | Purpose |
|------|----------|---------|
| `src/scrollback.c` | layout calculation | Line wrapping for conversation history |
| `src/input_buffer/layout.c` | layout calculation | Line wrapping for editor area |
| `src/render_cursor.c` | cursor positioning | Screen position from byte offset |

All must skip escape sequences when summing character widths. Pattern:

```c
while (pos < len) {
    if (text[pos] == '\x1b' && pos + 1 < len && text[pos + 1] == '[') {
        // Skip CSI sequence: ESC [ ... terminal_byte
        pos += 2;
        while (pos < len && text[pos] >= 0x20 && text[pos] <= 0x3F) pos++;
        if (pos < len) pos++;  // Skip terminal byte (0x40-0x7E)
        continue;
    }
    // Normal UTF-8 width calculation
    ...
}
```

CSI sequences terminate at bytes 0x40-0x7E (uppercase letters, @, etc.). SGR specifically terminates at `m`.

### Input Stripping (MEDIUM impact)

`src/input.c` must detect and discard SGR sequences from pasted input.

Current escape buffer is 16 bytes, sufficient for 256-color SGR (~12 bytes max). If sequence is recognized as SGR (terminates with `m`), discard rather than emit as unknown.

### Framebuffer Sizing (MEDIUM impact)

`src/render.c` estimates buffer sizes as `text_len + newline_overhead + fixed_escapes`. With inline colors, text already contains escape bytes, so estimates remain accurate. However, if render layer adds additional escapes (reset codes at line boundaries), account for that overhead.

### Configuration (LOW impact)

Add to `~/.config/ikigai/config.json`:

```json
{
  "color_mode": "256"  // "off", "256", or "auto"
}
```

Respect `NO_COLOR` environment variable (https://no-color.org/). When set, disable all color output regardless of config.

Check `TERM` for `dumb` terminal detection.

## Color Palette

Define semantic color names mapped to 256-color indices:

| Semantic | Use Case | Suggested Color |
|----------|----------|-----------------|
| `USER` | User message text | Cyan (51) |
| `ASSISTANT` | Assistant response text | Default (no color) |
| `SYSTEM` | System messages | Yellow (226) |
| `ERROR` | Error messages | Red (196) |
| `TOOL_CALL` | Tool invocation | Magenta (201) |
| `TOOL_RESULT` | Tool output | Green (46) |
| `CODE` | Code blocks | Default with different background |
| `EMPHASIS` | Bold/highlight | Bright white (231) |

Palette should be configurable in future.

## Implementation Phases

### Phase 1: Width Calculation

Add escape-sequence-aware iteration to:
- `scrollback.c` display width calculation
- `input_buffer/layout.c` display width calculation
- `render_cursor.c` cursor position calculation

Create shared utility function `ik_skip_ansi_escape()` or inline the check.

Test: Insert text with embedded escapes, verify width calculation matches visible characters.

### Phase 2: Input Stripping

Modify `input.c` escape sequence handler:
- Recognize SGR sequences (terminate with `m`)
- Discard rather than emit unknown action
- Ensure buffer doesn't overflow on long sequences

Test: Paste colored text, verify escapes stripped from input buffer.

### Phase 3: Render Coloring

Add color emission in render layer:
- `event_render.c` when converting messages to display text
- Emit appropriate SGR sequence based on message role/type
- Emit reset (`\x1b[0m`) at end of colored span

Requires render layer to know message metadata. Options:
- Pass role/type through to render functions
- Add style parameter to scrollback append
- Use sentinel markers that render translates

### Phase 4: Configuration

- Add `color_mode` to config schema
- Check `NO_COLOR` environment variable
- Check `TERM` for dumb terminal
- Gate color emission on configuration

## Open Questions

1. **Code block syntax highlighting** - Future scope. Would require language detection and token-level coloring. Consider tree-sitter integration.

2. **User-configurable colors** - Future scope. Start with hardcoded palette, add config later.

3. **256 vs 16 color fallback** - Some terminals only support 16 colors. Could detect via terminfo or provide config option.
