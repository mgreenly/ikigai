# Decision 28 — Blackhole empty-normalization content

**Decision.** An extracted subject whose `Name` normalizes to the empty string (D3 — `Normalize` may return `""` for input with no alphanumeric content) has no valid identity or path, so it is **blackholed**: dropped at the ingest integration boundary. It produces **no subject row, no claims, and no page**; the content attached to it is discarded.

The guard lives at the top of the per-extracted-item loop in `planIntegration` (`internal/wiki/service.go`), before the subject is planned or its claims are collected:

```go
for _, item := range extracted {
    if Normalize(item.Name) == "" {
        continue // blackhole: no subject, no claims, no page
    }
    subject, isNew, err := s.plannedSubject(ctx, knownByNorm, item)
    ...
}
```

Because the item is skipped before `plannedSubject`, no `norm_name = ""` row is ever inserted (so the D3 `NOT NULL UNIQUE` constraint is never exercised with an empty key), its claim bodies are never added to the plan, and no page is compiled for it. Other extracted subjects in the same job are unaffected and integrate normally.

This is the **only** place empty-normalization is handled: `Normalize` stays a pure function returning `""` (D3, it does not special-case empty), and `GetByPath` independently rejects an empty path token (D11). The integrate path that hosts this guard is the single-worker, atomic commit of D4/D14.

**Rejected.**
- **Insert the subject with `norm_name = ""` and suppress only the page.** Rejected: it persists an identity-less subject, the empty key collides under `UNIQUE(norm_name)` the moment a second one appears, and it leaves orphan claims.
- **Hard-error the whole ingest job on an empty result.** Rejected: one pathological extracted name would discard an entire document's worth of valid subjects. Blackholing the single item is more robust — fail the bad unit (it vanishes), not the whole batch.
- **A placeholder/sentinel `norm_name`.** Rejected — invents fake identity and groups unrelated blackholed content under one subject.

**Verification.**
- R-Z5JL-2IBS — integrating an extraction containing an item whose `Name` normalizes to `""` (e.g. `"???"`) creates **no** subject for it: after the job, no subject row exists with `norm_name = ""` and the blackholed name is absent from the subject registry.
- R-Z6RH-GA2H — the claims carried by that blackholed item are **not persisted**: none of its claim bodies appear attached to any subject after integration (the content is discarded, not reattached to another subject).
- R-Z7ZD-U1T6 — in the same extraction, a sibling item with a non-empty normalization is unaffected: its subject is created and its page is compiled — proving the blackhole drops only the empty-normalizing item, not the batch.
