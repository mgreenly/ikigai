# Decision 8 — The retrieval seam (`internal/retrieve`): keyword now, hybrid later

**Decision.** Phase-1 retrieval is FTS5 keyword only, sitting behind a flat `Hit`/`Retriever` seam (research §4's shape) so a vector/hybrid lane drops in without touching the read path or `ask`. The FTS5 plumbing (`ftsPhrase`, bm25, external-content sync) is ported verbatim from wiki.bak.

```go
package retrieve

// Hit is a whole-page retrieval result, lane-agnostic. PageID is the stable
// fusion/dedup key every lane must populate; in phase 1 (one page per subject)
// PageID == SubjectID. A future chunked/vector lane populates them distinctly.
type Hit struct {
    SubjectID string
    PageID    string  // stable fusion/dedup key (= page key)
    Version   int     // pages.version retrieved (provenance)
    Score     float64 // lane-local: BM25 now; cosine / fused RRF later
    Snippet   string  // matched excerpt
    Title     string
}

// Retriever is the ONE seam ask (D9) and search depend on. Vector and hybrid lanes
// slot in behind this same interface without touching the read path.
type Retriever interface {
    Search(ctx context.Context, query string, k int) ([]Hit, error)
}

// keywordRetriever — the only lane wired in phase 1: FTS5 MATCH + bm25() over pages_fts.
type keywordRetriever struct{ db *sql.DB }
func NewKeyword(db *sql.DB) Retriever

// SearchLimits clamps caller k into the contract.
type SearchLimits struct{ Default, Cap int }
func (l SearchLimits) Resolve(k int) int   // k<=0 → Default; k>Cap → Cap; else k

// Service composes registry-first pinning over the retriever (search verb + ask's tool call this).
type Service struct { db *sql.DB; r Retriever; limits SearchLimits }
func NewService(db *sql.DB, r Retriever, limits SearchLimits) *Service
func (s *Service) Search(ctx context.Context, query string, k int) ([]Hit, error)
```

- **`ftsPhrase` (port verbatim)** — wraps arbitrary user text as a single quoted FTS5 phrase literal, so `"`, `*`, `OR`, `NEAR`, etc. are treated literally, not as FTS5 operators (injection/operator safety). Every query passes through it.
- **Keyword lane** — `WHERE pages_fts MATCH ftsPhrase(query) ORDER BY bm25(pages_fts) LIMIT k`, joined back to `pages` for the whole-page `Hit`. Best-first, capped at `k`.
- **Registry-first pin** (`Service.Search`) — resolve `normalize(query)` against `subjects`; if it names a subject with a page, pin that page at rank 1, then the retriever fills the remainder, deduped by `PageID`, up to the resolved limit. Phase-1 identity (name-only) reused as a retrieval prior. Lives in `Service`, not `Retriever`, so every lane inherits it.
- **FTS sync (write side, performed in D4's commit tx)** — `pages_fts` is external-content with **no triggers**; the page-store write path syncs it explicitly in the same tx: on UPDATE, issue the FTS5 `'delete'` with the **OLD** title/body (read *before* the row update), then insert the new row. Ported verbatim — the trap that, gotten wrong, leaves stale text matchable.
- **Empty is clean** — no lexical match and no registry hit → empty `[]Hit`, no error; the signal D9's honest-empty path keys on.
- **Vector follow-on** (decided, built later): `vectorRetriever` (embed query via agentkit's future embeddings API → brute-force cosine over BLOB vectors) and `hybridRetriever` (fan out, fuse with RRF `k=60`) implement the same `Retriever`; the composition root swaps them in. Fusion is `hybridRetriever`'s internal detail — `Search` stays flat `[]Hit`. Nothing in phase 1 changes when they land.

**Rejected.**
- **wiki.bak's `WholePage`-returning `Retriever`** — couples the seam to a whole-page struct and bakes registry-first into the interface; research §4's flat `Hit` with an explicit `PageID` fusion key is the cleaner forward seam for RRF. Adopted research's shape.
- **FTS triggers for sync** — fire outside the page-write tx's control and complicate the external-content delete-by-old-value dance; explicit in-tx sync is what wiki.bak proved correct. Rejected.
- **Returning scores as the ordering contract** — phase 1 exposes rank order; `Score` rides along for the future RRF fuse, but callers rely on order, not absolute scores (BM25 and cosine share no axis).

**Verification.**
- R-CLF2-TMI8 — `SearchLimits.Resolve`: `k<=0 → Default`, `k>Cap → Cap`, in-range `k` returned unchanged.
- R-CMMZ-7E8X — A query containing FTS5 operator characters (e.g. `"`, `*`, `OR`) matches literally via `ftsPhrase` without error (operator/injection safety).
- R-CNUV-L5ZM — The keyword lane returns pages matching the query, ranked best-first by `bm25()` and capped at `k`.
- R-CP2R-YXQB — `Service.Search` pins an exact normalized-name match's page at rank 1 ahead of purely lexical hits.
- R-CQAO-CPH0 — After a page is updated, search matches the new body and no longer matches text present only in the old body (external-content FTS sync correctness).
