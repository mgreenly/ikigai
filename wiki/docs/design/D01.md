# Decision 1 ‚Äî Dependency on the external agentkit (the prod-build wiring)

**Decision.** wiki depends on the external multi-provider SDK `github.com/ikigenba/agentkit` as a **published, proxy-fetched module** ‚Äî exactly as it would any third-party dependency ‚Äî and carries **no `replace`** for it in `wiki/go.mod`.

- `wiki/go.mod` pins a real release: `require github.com/ikigenba/agentkit v0.1.0` (advanced by `go get github.com/ikigenba/agentkit@vX.Y.Z` as wiki needs newer agentkit releases ‚Äî e.g. when the embeddings API lands for the future vector lane).
- The in-repo `appkit` and `eventplane` libraries remain committed replace-siblings (`replace appkit => ../appkit`, `replace eventplane => ../eventplane`) for now. agentkit is the **first** suite dependency to move to the published-library model; appkit/eventplane follow later and are out of scope for this design.
- **Production build:** `bin/ship` builds HEAD off-box in a throwaway worktree with `GOWORK=off` and network access. With no `replace`, agentkit resolves from the public module proxy at the pinned tag ‚Äî deterministic and reproducible, touching nothing in-repo. The dead in-repo `agentkit/` module (still imported by prompts/sites as `module agentkit`) is untouched and unaffected.
- **Local co-development:** wiki edits to agentkit's live source at `~/projects/agentkit` are picked up via a `replace github.com/ikigenba/agentkit => /home/mgreenly/projects/agentkit` directive in the **root `go.work` only** ‚Äî never in `wiki/go.mod`. `go.work` is local-dev-only and ignored under `GOWORK=off`, so this never reaches the prod build. wiki is added to the workspace by swapping `use ./wiki.bak` ‚Üí `use ./wiki`.
- agentkit takes its API key by explicit injection (`anthropic.New(apiKey, ‚Ä¶)`); it never reads the environment. wiki reads `ANTHROPIC_API_KEY` at its own composition root and passes it in (see later Decisions for the secret-handling seam).

**Rejected.**
- **Committed absolute-path `replace ‚Ä¶ => /home/mgreenly/projects/agentkit` in `wiki/go.mod`** ‚Äî works locally but breaks `bin/ship`: the throwaway worktree has no such path and builds with `GOWORK=off`. This is the failure mode the decision exists to avoid.
- **Move agentkit in-repo as a replace-sibling** (rename dead `agentkit/` ‚Üí `agentkit.bak/`, copy external source to `agentkit/`, commit `replace ‚Ä¶ => ../agentkit`). Matches the appkit/eventplane pattern and gives live edits with no tag bumps, but forks agentkit's source into two trees (mono-repo + the standalone `github.com/ikigenba/agentkit` repo that its module path and `v0.1.0` tag mark as the real source of truth) and touches two legacy services. Rejected in favor of keeping agentkit an independent published library ‚Äî the owner's stated direction for all shared libs.
- **Vendor (`go mod vendor`)** ‚Äî the suite uses replace-siblings / the proxy, not a committed `vendor/` tree; vendoring adds a large committed directory and a re-vendor step on every bump for no benefit here.

**Verification.**
- R-MV3L-QS7I ‚Äî A production-shaped build of wiki (`GOWORK=off`, with no `go.work` in scope and with the local `~/projects/agentkit` path absent) compiles successfully, resolving `github.com/ikigenba/agentkit` at its pinned tag from the module proxy.
- R-MWBI-4JY7 ‚Äî `wiki/go.mod` contains a versioned `require github.com/ikigenba/agentkit` and **no** `replace` directive for it (the only `replace` directives are the in-repo `appkit` and `eventplane` siblings).
