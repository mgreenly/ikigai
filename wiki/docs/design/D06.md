# Decision 6 — The extract stage (`internal/extract`)

**Decision.** A single-pass, tool-less, structured LLM call over the D5 `llm.JSON` helper: the whole source text plus a mechanical header → subjects, each with its claims. Ported from wiki.bak's extract, minus aliasing.

```go
package extract

type ExtractedSubject struct {
    Type       string   // "entity" | "event" | "concept"
    Kind       string   // freeform subtype, prompt-anchored ("person","company","method"…)
    Name       string   // primary name AS THE TEXT STATES IT (registry normalizes later)
    OccurredAt string   // ISO-8601 prefix; events only; "" otherwise
    Claims     []string // short self-contained prose statements, no pronouns
}

// DocumentHeader anchors relative-time resolution; rendered verbatim, never model-inferred.
type DocumentHeader struct {
    Source     string    // phase 1: "mcp:ingest_text"
    Title      string
    Tags       []string
    ReceivedAt time.Time // rendered "received on: YYYY-MM-DD" (never "today is")
}

type Extractor struct { c *llm.Client; site llm.CallSite }
func New(c *llm.Client, site llm.CallSite) *Extractor
func (e *Extractor) Extract(ctx context.Context, h DocumentHeader, text string) ([]ExtractedSubject, error)
```

- **Prompt** — port wiki.bak's `DefaultExtractPrompt`, **dropping the `aliases` field and the within-doc alias/co-reference clause** (no aliasing in phase 1). Keep verbatim the two load-bearing rules: the **salience gate** ("extract a subject only when it is identifiable — the Wikipedia-article test — AND the document makes a concrete claim about it; never invent a subject to hold a stray fact") and the **claims discipline** ("each claim is a short SELF-CONTAINED prose statement, no pronouns; state only what the document ASSERTS — do not infer or synthesize"). Keep the dialog-awareness clause and a worked example (rewritten without aliases). Closed-set type; `occurred_at` for events; relative time resolved against the header's received-on date, kept relative when unresolvable.
- **`validate`** (the D5 caller hook): every subject's `Type` ∈ `{entity,event,concept}`; `Name` non-empty after trim; ≥1 claim, each with non-empty `text`. A violation is a parse-level failure → bounded re-prompt. After a valid parse, `OccurredAt` is coerced `""` for non-events.
- **Empty is honest, not an error.** A text with no salient subject yields `[]`; the job completes with zero subjects. Extract never invents a subject to avoid emptiness.
- **Call site** (defaults, configurable via `ManifestExtras`): `anthropic.ModelSonnet46`, `Temperature: 0`, `DisableReasoning()`, `MaxParseRetries: 2`. Single-pass and tool-less — reads only the header + source text, reinforcing the anti-poisoning invariant (D4 R-MG31).

**Rejected.**
- **Keeping the `aliases` field** — feeds the deferred alias machinery; in phase 1 it is dead weight that invites the model to over-merge surface forms. Dropped.
- **Treating empty `subjects[]` as a failure/retry** — would pressure the model to fabricate subjects, violating the salience gate. Empty is a valid outcome.
- **Haiku for extract** — cheaper, but extraction error is frozen into the durable record and silently recompiled forever (research §5); the stronger model is warranted (still a config knob).

**Verification.**
- R-VYU0-BPAX — `validate` rejects an extraction containing a subject whose `Type` is outside `{entity,event,concept}` (a parse-level failure that triggers a re-prompt).
- R-W01W-PH1M — A valid extraction retains `OccurredAt` only for `Type=="event"` and empties it for `entity`/`concept`.
- R-W19T-38SB — A source with no salient subject yields an empty `[]ExtractedSubject` and a successful (non-error) extract — no invented subject.
- R-W2HP-H0J0 — The extract call site is configured with `Temperature: 0` and `DisableReasoning()` (deterministic extraction; confirmed via the capturing mock provider).
