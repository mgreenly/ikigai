# Decision 2 — Service skeleton: package layout, Spec wiring, and the config/secret composition root

**Decision.** wiki is a single `module wiki` built on the appkit chassis, laid out like crm/notify. This decision fixes the module shape and the composition root only; the internals of each stage (schema, extract, compile, retrieve, ask, worker, MCP surface) are their own Decisions.

**Package layout:**

```
wiki/cmd/wiki/main.go        — composition root: build Config, read ANTHROPIC_API_KEY, construct the
                                shared agentkit Provider + domain Service, declare appkit.Spec, wire the Worker.
wiki/internal/db/            — db.go (//go:embed migrations → db.FS) + migrations/*.sql
wiki/internal/wiki/          — domain core: types (Subject, Claim, Page, Job), Service, stores
wiki/internal/llm/           — agentkit seam: holds the shared *Provider; the "json-mode helper"
                                (prompt → Send once → strip code fence → unmarshal → validate → retry-on-parse).
                                Provider is shared (stateless); a fresh *Conversation is built per call.
wiki/internal/extract/       — text → (subjects, claims)            [D-later]
wiki/internal/compile/       — claims → page, 12k cap               [D-later]
wiki/internal/ask/           — ask: subject-extraction → resolve → synthesize pipeline [D-later]
wiki/internal/worker/        — ingest poll-loop Worker              [D-later]
wiki/internal/mcp/           — JSON-RPC transport (copy crm) + tool declarations/dispatch [D-later]
wiki/internal/ids/           — ULID minting (copy crm verbatim)
```

**Spec wiring** (`cmd/wiki/main.go`) — all event-plane fields empty; wiki is on no event plane in phase 1, which is what makes `reflection` empty (verified in the MCP Decision):

```go
var svc *wiki.Service
appkit.Main(appkit.Spec{
    App: "wiki", Mount: "/srv/wiki/", Port: 3006, MCP: true,
    Migrations: db.FS,
    // Feed:"", Consumes:nil, Events:{}, Publishes:nil, Subscriptions:nil
    Config:  func(getenv func(string) string) (any, error) { /* build wiki.Config, incl. AnthropicKey */ },
    ManifestExtras: []appkit.ManifestKV{ /* model id, worker concurrency */ },
    Handlers: func(rt *appkit.Router) error {
        write := rt.DB()                                 // appkit's handle = the SINGLE WRITER (MaxOpenConns 1); see D17
        read := /* db.OpenRead(cfg.DBPath): same file, query_only, MaxOpenConns(N>1) — concurrent reads (D17) */
        prov := /* agentkit anthropic.New(cfg.AnthropicKey) */
        svc = wiki.NewService(wiki.Conns{Read: read, Write: write}, prov, cfg) // captured for the Worker closure
        rt.Handle("POST /mcp", rt.RequireIdentity(
            mcp.NewHandler(svc, rt.Version(), rt.Service(), rt.Health(), rt.Events(), rt.Subscriptions())))
        return nil
    },
    Workers: []func(context.Context) error{ func(ctx context.Context) error { return worker.Run(ctx, svc) } },
})
```

- **Port 3006** — the port the prior build used (`wiki.bak` `PORT=3006`); reused for continuity.
- `Handlers` runs after appkit opens+migrates the DB and before `Workers` start; the domain `Service` is built over the **two handles** (`wiki.Conns`, D17) and captured in a `var` the `Workers` closure reads (the crm/notify pattern). `rt.DB()` is the single writer; the read handle is opened here over the already-migrated file and runs no migrations.
- **Do not hand-spawn a goroutine in serve** — the ingest worker is a `Spec.Workers` entry; appkit runs it on the serve ctx and SIGTERM cancels it. A worker that *returns* is a structural fault that brings the server down, so it retries transient faults inside its loop and exits only on `ctx.Done()`.

**Config & secret seam:**
- **Non-secret knobs** (model id, worker concurrency) → `ManifestExtras`, round-tripping into `etc/manifest.env`, read at the composition root into a `wiki.Config` value that domain code receives by argument. The **12,000-char page cap is a product constant honored by value** — a named `const` in the compile package, not a user-tunable knob.
- **Secret** `ANTHROPIC_API_KEY` → read once at the composition root via `getenv`; **fail loudly at startup if absent** (crash, never serve degraded). Locally injected by `wiki/.envrc` (`source_up` + `export ANTHROPIC_API_KEY="$(cat ~/.secrets/ANTHROPIC_API_KEY)"`); its value never enters code beyond the single hand-off to `anthropic.New`.
- The agentkit Provider is constructed **once** and shared across requests; each LLM call builds a **fresh `*Conversation`** (Conversation is not concurrency-safe / single-flight).

**Rejected.**
- **Hand-spawning the ingest goroutine inside `serve`** instead of `Spec.Workers` — loses appkit's ctx-cancel/SIGTERM lifecycle and the structural-fault guarantee. Rejected.
- **Opening a second *writer* in the service** rather than using `rt.DB()` for writes — appkit owns open + migrate + the single-writer connection, and a second *write* handle would break the single-writer invariant. (A second **read-only** handle does **not** — it adds no writer; that is exactly D17's concurrent read pool.) Rejected for writes; the read handle is the sanctioned exception.
- **A bring-your-own secret mechanism** (flag, config file literal, hardcoded fallback) — violates the project secret rule; the only path is `~/.secrets` → `.envrc` → env → `getenv`. Rejected.
- **Port 3007** (research's guess) — the prior build is on 3006; reusing it avoids a manifest/nginx reallocation. Rejected.

**Verification.**
- R-6RVX-P1IG — Starting `serve` with `ANTHROPIC_API_KEY` unset (or empty) fails loudly at startup (non-zero exit / startup error) rather than starting the HTTP server.

(The package layout, Spec field values, and worker-vs-goroutine choice are structural — their behavioral proof is carried by the ids of the Decisions they enable: schema D-later, ingest/extract/compile D-later, retrieve/ask D-later, and the MCP surface D-later, which also verifies the empty `reflection` and the `health` envelope.)
