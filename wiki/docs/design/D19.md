# Decision 19 — Per-call-site configuration in production (retire the single global model)

**Decision.** The wiki had **one** `ModelID` constant threaded into every LLM site, and the two `ask` sites were hand-built as bare `llm.CallSite{Stage:"ask", Model: cfg.ModelID}` — the exact bare construction D6 warns against. The product now promises each site is configured **independently**, and (the point of the whole eval effort) we have **no evidence** the in-code choices are correct, so every model-quality lever must be an experiment knob, not a frozen literal. This decision replaces the global model with **full per-site configuration**: each production LLM call site owns a `DefaultCallSite()` carrying its designed values, and every model-tuning field — model, reasoning, temperature, output budget — is overridable per site from its own manifest knobs. The closed set of **production service** call sites is **extract, compile, ask-subject, ask-synthesis**. The **judge** (D21) is an eval-only site configured by the eval CLI (D22), not a service manifest knob.

## Each site owns a `DefaultCallSite()`

Every stage exposes a no-secret constructor returning its designed `llm.CallSite` (D5), so the composition root never hand-builds a bare one. The two `ask` sites gain the `DefaultCallSite` they lacked (D18 flagged this), and the `extract`/`compile` `DefaultCallSite(model)` signatures lose their `model` argument — the model now arrives via the config resolver below, not as a positional default.

```go
// each stage package:
func extract.DefaultCallSite() llm.CallSite      // Stage "extract",      Temperature 0, DisableReasoning(), MaxTokens 16384, MaxParseRetries 2
func compile.DefaultCallSite() llm.CallSite       // Stage "compile",      Temperature 0, DisableReasoning(), MaxTokens 16384, MaxParseRetries 2
func ask.DefaultSubjectCallSite() llm.CallSite     // Stage "ask-subject",  Reasoning Level("low"), MaxTokens 16384
func ask.DefaultSynthesisCallSite() llm.CallSite   // Stage "ask-synthesis",Reasoning Level("low"), MaxTokens 16384
```

The `Stage` label is the site's stable identity — it is the `CallRecord.Stage` (D13) and the eval/inspection key. `ask`'s two sites split into the distinct labels **`ask-subject`** and **`ask-synthesis`** (formerly both `"ask"`), so their records and configs are separable; the D13 stage label set widens accordingly (still a closed set).

## Per-site config resolved from per-site knobs

`Config.ModelID string` is removed. `Config` carries a resolved `CallSites` value; each field is that site's `DefaultCallSite()` with its env overrides applied:

```go
type CallSites struct {
    Extract      llm.CallSite
    Compile      llm.CallSite
    AskSubject   llm.CallSite
    AskSynthesis llm.CallSite
}
type Config struct {
    CallSites         CallSites
    WorkerConcurrency int
    SearchDefault     int
    SearchCap         int
    Provider          agentkit.Provider
    LLM               *llm.Client
}
```

A single resolver applies a site's four model-tuning knobs onto its default; **the same function** the eval CLI (D22) uses to apply flag overrides, so production env and CLI flags drive the identical seam:

```go
// resolveCallSite layers <PREFIX>_MODEL / _REASONING / _TEMPERATURE / _MAX_TOKENS onto base.
// An unset knob leaves the base default; a present-but-malformed knob is a startup error (fail loudly).
func resolveCallSite(getenv func(string) string, prefix string, base llm.CallSite) (llm.CallSite, error)
```

**Manifest knobs** (`ManifestExtras`, replacing the single `MODEL_ID`) — four per site, defaulted to each `DefaultCallSite`'s designed value so an all-unset environment reproduces today's behavior exactly:

```
EXTRACT_MODEL        EXTRACT_REASONING        EXTRACT_TEMPERATURE        EXTRACT_MAX_TOKENS
COMPILE_MODEL        COMPILE_REASONING        COMPILE_TEMPERATURE        COMPILE_MAX_TOKENS
ASK_SUBJECT_MODEL    ASK_SUBJECT_REASONING    ASK_SUBJECT_TEMPERATURE    ASK_SUBJECT_MAX_TOKENS
ASK_SYNTHESIS_MODEL  ASK_SYNTHESIS_REASONING  ASK_SYNTHESIS_TEMPERATURE  ASK_SYNTHESIS_MAX_TOKENS
```

**Value parsing (shared by env and CLI flags, fail-loud on garbage):**
- `_MODEL` — a model id string; empty/unset keeps the default. (Validation against a known-model set is the provider's concern, not config's.)
- `_REASONING` — one of `off` → `DisableReasoning()`, or `low` / `medium` / `high` → `agentkit.Level(...)`. Any other non-empty value is a **startup error** naming the valid set — never a silent fallback.
- `_TEMPERATURE` — a float parsed to `*float64`; the literal `default` (or unset) means `nil` (provider default). A non-numeric, non-`default` value is a startup error.
- `_MAX_TOKENS` — a positive int; unset keeps the default; `0`, negative, or non-numeric is a startup error (a production JSON site must never run at the 4096 adapter default — D18).

The composition root (`cmd/wiki/main.go`) builds each stage from its resolved site: `extract.New(client, cfg.CallSites.Extract)`, `compile.New(client, cfg.CallSites.Compile, …)`, `ask.New(subjectStore, pageStore, client, cfg.CallSites.AskSubject, cfg.CallSites.AskSynthesis)`. No bare `llm.CallSite{…}` remains.

**Rejected.**
- **Model-only production knob (reasoning/temp/budget frozen in code).** Considered and rejected by the user: per-site config is the entire point. We have no metrics proving extract's temp-0/reasoning-off or the 16384 budgets are right, so freezing them in code is freezing an unproven guess — every model-quality lever must be experiment-tunable, and a knob proven good in the eval (D20–D22) must be settable in production without a recompile. (The fail-loud parsing keeps the freedom from silently corrupting a determinism-critical site.)
- **A new per-site override struct (`CallSiteConfig{Model,Reasoning,…}`) layered over `CallSite`.** Redundant — `llm.CallSite`'s fields are already exported, so the resolver mutates the default in place; a parallel override type is a second shape to keep in sync. Rejected.
- **Keeping the single `MODEL_ID` and deriving per-site models from it.** Re-creates the global; you could never give extract a different model from compile. Rejected — that is the bug.
- **Leaving the two ask sites bare-constructed and only splitting their `Stage`.** Bare construction is the drift D6 indicts; a `DefaultCallSite` per ask site is what makes the config resolver uniform across all four. Rejected.

**Verification.**
- R-GGIG-AN7W — With `EXTRACT_MODEL` and `COMPILE_MODEL` set to **different** values, `NewConfig` yields `CallSites.Extract.Model` and `CallSites.Compile.Model` each equal to its own knob (not equal to each other) — proving the model is resolved per site, not from one shared global.
- R-GHQC-OEYL — A non-model knob set for one site reaches only that site: e.g. `ASK_SYNTHESIS_REASONING=low` (or a `_TEMPERATURE`/`_MAX_TOKENS` override) produces the configured value on `CallSites.AskSynthesis` while the other three sites retain their defaults.
- R-GIY9-26PA — With **all** per-site knobs unset, every resolved site equals its stage `DefaultCallSite()`: extract carries `Temperature==0`, `DisableReasoning()`, `MaxTokens>=16384` (the 4096 default excluded), and the four sites' models all equal the in-code default model — so an unconfigured environment reproduces prior behavior with no regression.
- R-GK65-FYFZ — Parsing fails loudly at startup on a malformed knob: an unrecognized `_REASONING` (not `off`/`low`/`medium`/`high`) returns a startup error naming the valid set; a non-numeric `_TEMPERATURE` (other than `default`) and a `_MAX_TOKENS` that is non-positive or non-numeric each return a startup error — none silently fall back to a default.
- R-GLE1-TQ6O — `ask.DefaultSubjectCallSite()` and `ask.DefaultSynthesisCallSite()` carry the distinct stage labels `ask-subject` and `ask-synthesis` and `MaxTokens>=16384`, and the composition root constructs `ask` from them (no bare `llm.CallSite{Stage:"ask",…}` remains), so the two ask sites are independently configured and separately recorded.
