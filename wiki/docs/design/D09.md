# Decision 9 â€” `ask` (`internal/ask`): subject-extraction pipeline, grounded/cited/honest-empty

**Decision.** `ask` answers a question by **extracting the subjects the question names, resolving each to a wiki subject by exact normalized name, reading those subjects' pages, and synthesizing a cited answer from them** (product: "Ask works like an agent"). Control flow is a **deterministic three-stage pipeline** in Go; the LLM is invoked twice as tool-less structured generations through the `llm.JSON[T]` seam (D5) â€” once to extract names, once to synthesize. There is **no keyword retrieval, no tool-loop, and no source/claims drill-down**: ask draws on the compiled pages only. It is strictly read-only, permanently.

```go
package ask

type Answer struct {
    Found     bool       // false â†’ the wiki holds nothing on the question
    Text      string     // synthesized answer, or the fixed honest-empty sentence
    Citations []Citation // the gathered pages the answer drew on
}
type Citation struct { Subject, Title string }

type Asker struct {
    subjects    *wiki.SubjectStore  // GetByNormName â€” exact, normalized resolution
    pages       *wiki.PageStore     // GetBySubject  â€” the compiled page per subject
    c           *llm.Client
    extractSite llm.CallSite         // stage 1 config (NER over the question)
    synthSite   llm.CallSite         // stage 3 config (grounded synthesis)
}
func New(subjects *wiki.SubjectStore, pages *wiki.PageStore, c *llm.Client, extractSite, synthSite llm.CallSite) *Asker
func (a *Asker) Ask(ctx context.Context, owner, question string) (Answer, error)
```

**Flow â€” extract â†’ resolve & gather â†’ synthesize:**

1. **Extract (one LLM call).** `llm.JSON[extractResult](ctx, c, extractSite, question, validate)` returns the candidate subject **names** the question concerns. One call returns the whole list; the validator only requires a well-formed (possibly empty) list.
   ```go
   type extractResult struct { Subjects []string `json:"subjects"` }
   ```
2. **Resolve & gather (deterministic â€” no LLM).** For each extracted name: `normalize(name)` â†’ `SubjectStore.GetByNormName` â†’ on hit, `PageStore.GetBySubject`. Dedup resolved subjects by id; preserve first-seen order. Resolution is **strict normalized-exact** â€” a partial or variant name ("Gygax" for "Gary Gygax") resolves to nothing (aliases / fuzzy matching are a later release, per product). This is **best-effort partial**: when several names are extracted, every one that resolves is gathered; the ones that don't are silently skipped.
   - **Zero subjects resolve â†’ honest-empty: return `Answer{Found:false, Text:<fixed sentence>, Citations:nil}` and make NO synthesis call.** The synthesis model never runs against an empty page set, so it cannot fabricate from world knowledge â€” the deterministic anti-fabrication lever, carried over from the old keyword gate to the subject-resolution gate.
3. **Synthesize (one LLM call).** `llm.JSON[answerResult](ctx, c, synthSite, synthPrompt(question, gathered), validate)` over the gathered pages only.
   ```go
   type answerResult struct {
       Found     bool       `json:"found"`
       Text      string     `json:"text"`
       Citations []Citation `json:"citations"`
   }
   ```
   The shared `ExtractJSON` carve and the bounded parse-retry both live inside `llm.JSON[T]` (D5) â€” `ask` carries no JSON-from-decoration logic of its own.

**Grounding & anti-fabrication (pages-only):**
- A `Found:true` answer **must carry â‰Ą1 citation**; a synthesis reply claiming found with no citation is **downgraded to honest-empty**.
- Each citation is **validated against the gathered set** (the pages actually read this turn). A citation to anything not gathered is **dropped** (no fabricated pages/citations reach the caller). Because the gathered set came straight from the page store, a surviving citation is by construction a real page.
- The synthesis prompt is fed **only the gathered page bodies** â€” never raw claims or original source text. Drill-into-source is a later release (product non-goal); no such tool or path exists in `ask`.
- A full NLI entailment verifier (answer-claims vs cited page spans) remains a noted known gap / fast-follow, out of scope here.

**`ask` is strictly read-only â€” permanently** (product constant). `Asker` only reads (`GetByNormName`, `GetBySubject`); no write path exists on any branch, including the honest-empty and parse-failure paths.

**Prompts.**
- `extractSite.System` â€” instruct the model to return only the proper-noun **subjects** the question is *about* (the entities/events/concepts whose page would answer it), as a JSON list of names, and nothing else; no answering, no speculation. Empty list when the question names no subject.
- `synthSite.System` â€” answer **only** from the supplied pages; cite each page drawn on by subject id + title; if the supplied pages do not answer the question, return `found:false`; never fabricate a fact, page, or citation; surface contradictions across pages with both sides rather than silently resolving; you are read-only. JSON output schema + example.

**Call sites** (defaults, configurable):
- `extractSite`: `anthropic.ModelSonnet46`, `Temperature: 0`, `Reasoning: DisableReasoning()` â€” deterministic NER.
- `synthSite`: `anthropic.ModelSonnet46`, `Reasoning: Level("low")`, `Temperature` provider-default.

**Relationship to the public surface.** The MCP `ask` tool (D10) calls `Asker.Ask` and returns the `Answer{Found,Text,Citations}` contract. (D10 no longer surfaces an ask `Sources` field â€” `read_source` is gone; D10 is updated accordingly.) `ask` no longer composes over `retrieve.Service`; the fate of that seam is D8.

**Rejected.**
- **Agentic tool-loop (`resolve_subject`/`read_page` tools, model drives the loop)** â€” its behavior depends on whether the model elects to call the tools, which makes the honest-empty gate fuzzy and the stages hard to pin with checkable ids. The deterministic pipeline tests each seam in isolation and keeps the empty-gate decision in Go. Rejected.
- **Keyword pre-flight probe over page bodies (old D9/D8 path)** â€” subject-extraction-only is the product decision; word-matching page bodies is a later release. The "no LLM call at all on empty" property it bought is unattainable once extraction (inherently an LLM call) is the first step; the new gate saves the *synthesis* call instead. Removed.
- **`read_source` / raw-claims drill-down tool** â€” pages-only is the product decision; recovering what a page's compression lost is a later release. Dropped (and with it the `Answer.Sources` field).
- **Accepting `Found:true` with zero citations** â€” surfaces ungrounded claims; downgrade to honest-empty instead.
- **All-or-nothing resolution (any unresolved named subject â†’ empty)** â€” discards real, citable knowledge over one uningested name; best-effort partial answers from what resolves. Rejected.
