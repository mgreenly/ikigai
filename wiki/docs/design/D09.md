# Decision 9 — `ask` (`internal/ask`): grounded, cited, honest-empty agent

**Decision.** `ask` answers a question over the wiki's pages, drawing only on ingested content, with a **deterministic honest-empty gate that runs before any LLM call**. It is strictly read-only, permanently.

```go
package ask

type Answer struct {
    Found     bool       // false → the wiki holds nothing on the question
    Text      string     // synthesized answer, or the fixed honest-empty sentence
    Citations []Citation // pages the answer drew on
    Sources   []string   // job-ids surfaced from cited pages' inline [job-id] cites (populated when read_source was used)
}
type Citation struct { Subject, Title string }

type Asker struct {
    rs   *retrieve.Service
    pages wiki.PageStore
    src  wiki.SourceReader
    c    *llm.Client
    site llm.CallSite
    maxIter, readSourceCap int
}
func New(...) *Asker
func (a *Asker) Ask(ctx context.Context, owner, question string) (Answer, error)
```

**Flow — the honest-empty gate runs first:**
1. **Pre-flight keyword probe** — `rs.Search(ctx, question, k)`. **Zero hits → return `Answer{Found:false, Text:<fixed "the wiki has nothing on this">, Citations:nil}` with NO LLM call.** The strongest honest-empty lever in keyword phase 1 (research §5/§6 Q5): the model can't fabricate from world knowledge because it never runs. Tradeoff accepted — a question whose words match no page is treated as empty; the agent's own reformulation is available only once ≥1 hit exists.
2. **Hits exist → run the agent.** Per-request Conversation via `llm.Converse(askSite, askTools)` with three **in-process** `agentkit.NewTool` tools (no network hop):
   - `search(query, limit?)` → `rs.Search` → ranked hits (subject id, title, snippet).
   - `read_page(subject)` → that subject's full page body.
   - `read_source(job_id)` → the original raw text (`jobs.source_text`), size-capped at `readSourceCap` — the one-hop-to-bedrock tool for checking exact wording / resolving a contradiction, not a default.
   `MaxToolIterations` bounded (`maxIter`); the request `ctx` deadline is the wall-clock budget. `Send(question)` → drain events → final message → **`llm.ExtractJSON`** (the single shared carve from D5) → `json.Unmarshal` into `Answer`. `ask` does **not** carry its own fence-stripper — the ported private `stripCodeFence` is deleted in favor of the shared carve, so the agent's JSON answer survives the same prose/odd-fence/stray-backtick decoration that `ExtractJSON` handles for extract/compile.

**Grounding & anti-fabrication (bracket-presence discipline — research §5's phase-1 lever):**
- A `Found:true` answer **must carry ≥1 citation**; an agent reply claiming found with no citation is **downgraded to honest-empty**.
- Each citation's subject id is **validated against the page store**; a citation that doesn't resolve to a real page is **dropped** (no fabricated pages/citations reach the caller).
- A full NLI entailment verifier (answer-claims vs cited spans) is a noted known gap / fast-follow, out of scope for the thin slice.

**`ask` is strictly read-only — permanently** (product constant). The `Asker` and all three tools only read; no write path exists.

**Prompt** — port wiki.bak's `DefaultAskPrompt`, trimmed: keep the framing ("answer ONLY from the wiki's pages; if nothing, say so plainly; never fabricate a fact, page, or citation; you are read-only"), the corpus model, the procedure (search → read_page; read_source only to check wording/contradictions), answer craft (cite pages by subject id + title; surface contradictions with both sides, never silently resolve), budget discipline, and the JSON output schema + example. Drop the `lookup`/`timeline`/`related` tooling and their clauses.

**Call site** (defaults, configurable): `anthropic.ModelSonnet46`, `Reasoning: Level("low")`, `Temperature` provider-default, `maxIter: 8`, `readSourceCap: 8192`.

**Relationship to the public surface (research §6 Q2):** ask's internal `search`/`read_page` and the public inspect/search MCP tools (D10) **compose over the same `retrieve.Service` and page store** — one retrieval seam, no divergent logic — but are *declared* separately (agentkit `NewTool` specs vs the MCP tool map).

**Rejected.**
- **No pre-flight gate (agent always runs and self-reports empty)** — costs an LLM call on every empty question and leans on prompt-only honesty ("theater," research §5); the deterministic gate is the real lever. Rejected.
- **`lookup`/`timeline`/`related` tools** — `search`'s registry-first pin (D8) already favors exact-name questions; timeline/related are beyond phase-1 scope. Dropped.
- **Persisting an `asks` row** (wiki.bak) — phase 1 needs no ask history/accounting; ask stays stateless and read-only. Dropped.
- **Accepting `Found:true` with zero citations** — surfaces ungrounded claims; downgrade to honest-empty instead.
