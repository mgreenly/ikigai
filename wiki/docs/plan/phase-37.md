*Realizes design Decision 21 (the judge call site + scoring semantics). Depends on Phase 36 (the `internal/eval` `Case`/`GoldSubject` types), Phase 03 (the `llm.JSON` seam), and Phase 02 (`internal/wiki`, where `normalize` is exported as `wiki.Normalize`).*

`internal/eval` gains the two-stage scoring of an extract run against a blessed case. The observable end state:

- `internal/wiki` exports `Normalize(string) string` (the existing unexported `normalize`, unchanged in behavior), so eval reuses the **production** subject-identity normalizer rather than re-deriving it.
- **Subjects matched deterministically (no LLM):** a predicted and a gold subject are the same when `(Type, wiki.Normalize(Name))` agree; scoring partitions into `Found` / `Missed` / `Hallucinated` (each a `type/slug`). A name differing only by case/whitespace still matches.
- **Claims judged by meaning, one call per matched subject:** `Judge` (`NewJudge(client, site)`) wraps a pinned call site; `DefaultJudgeCallSite()` returns `{Stage:"judge", Model: anthropic.ModelOpus48, Temperature 0, Reasoning Level("low"), MaxTokens 16384, MaxParseRetries 2}`. For each matched subject, `Score` issues one judge call through `llm.JSON` whose validated structured verdict partitions that subject's gold claims into covered/missed and predicted claims into extra. A differently-worded predicted claim that conveys a gold claim counts **covered**. An out-of-range/malformed verdict re-prompts via the validate hook (bounded), then errors.
- **No judge call for unmatched subjects:** a missed subject's gold claims all count missed; a hallucinated subject's predicted claims all count extra.
- `Score(ctx, j, c, predicted)` returns a `CaseResult` (subject partition + summed `ClaimScore{Covered,Missed,Extra}` + retained per-subject claim text for eyeballing); `Aggregate(results)` returns `Totals` with subject and claim precision/recall overall and grouped by difficulty.
