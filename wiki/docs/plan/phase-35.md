# Phase 35 ‚Äî Per-call-site configuration: retire the single global model

*Realizes design Decision 19 (per-call-site configuration in production). Depends on Phase 14/15 (extract/compile `DefaultCallSite`), Phase 17 (the `ask` pipeline and its two sites), Phase 28 (`CallSite.MaxTokens`), and Phase 01/02 (the `wiki.Config` + `cmd/wiki/main.go` composition root).*

Each production LLM call site is configured **independently**, replacing the one global `ModelID` threaded into all of them. The observable end state:

- Every production site owns a no-secret `DefaultCallSite()` carrying its designed values: `extract.DefaultCallSite()` and `compile.DefaultCallSite()` **lose their `model` argument** (the model now comes from config, not a positional default) while keeping Temperature 0 / `DisableReasoning()` / `MaxTokens 16384` / `MaxParseRetries 2`; `ask` gains `ask.DefaultSubjectCallSite()` and `ask.DefaultSynthesisCallSite()` carrying the distinct stage labels **`ask-subject`** and **`ask-synthesis`** (formerly both `"ask"`) and `MaxTokens >= 16384`.
- `internal/wiki` gains `resolveCallSite(getenv, prefix, base)` that layers a site's four knobs ‚Äî `<PREFIX>_MODEL`, `<PREFIX>_REASONING` (`off`‚Üí`DisableReasoning()`, `low`/`medium`/`high`‚Üí`agentkit.Level`), `<PREFIX>_TEMPERATURE` (float or `default`‚Üínil), `<PREFIX>_MAX_TOKENS` (positive int) ‚Äî onto its `DefaultCallSite()`. An unset knob keeps the default; a **present-but-malformed** knob is a startup error naming the valid set (fail loudly, never silent fallback).
- `Config.ModelID string` is removed in favour of `Config.CallSites{ Extract, Compile, AskSubject, AskSynthesis llm.CallSite }`, each resolved in `NewConfig`. The single `MODEL_ID` `ManifestExtras` entry is replaced by the sixteen per-site keys (four per site), each defaulted to its `DefaultCallSite` value so an all-unset environment reproduces today's behavior (model `claude-sonnet-4-6`) with no regression.
- `cmd/wiki/main.go` constructs every stage from its resolved site ‚Äî `extract.New(client, cfg.CallSites.Extract)`, `compile.New(client, cfg.CallSites.Compile, ‚Ä¶)`, `ask.New(‚Ä¶, cfg.CallSites.AskSubject, cfg.CallSites.AskSynthesis)` ‚Äî with **no** bare `llm.CallSite{‚Ä¶}` left at the composition root.

**Done when:** R-GGIG-AN7W (different `EXTRACT_MODEL`/`COMPILE_MODEL` ‚Üí different per-site models), R-GHQC-OEYL (a non-model knob reaches only its site), R-GIY9-26PA (all-unset ‚Üí stage defaults, extract Temperature 0 / `MaxTokens>=16384`, no regression), R-GK65-FYFZ (malformed reasoning/temperature/max-tokens each fail loudly), and R-GLE1-TQ6O (the two `ask` sites resolve from `DefaultCallSite`s carrying `ask-subject`/`ask-synthesis` and `MaxTokens>=16384`; no bare construction remains) are each covered by clearly-named tests ‚Äî `wiki.NewConfig` driven by a fake `getenv` for the resolution/parsing ids, the stage `DefaultCallSite` tests for the labels/budgets ‚Äî and the suite is green.
