# Phase 38 — The `cmd/eval-extract` binary + the shipped gold case

*Realizes design Decision 22 (the `cmd/eval-extract` binary + the shipped gold case). Depends on Phase 36 and Phase 37 (the `internal/eval` library), and Phase 35 (the D19 flag parsers reused for the CLI flags).*

A second binary `cmd/eval-extract`, a thin shell over `internal/eval`, runs the dataset and prints a stamped scorecard; one blessed gold case ships in-repo. The observable end state:

- `internal/eval` gains `NewJSONLRecorder(io.Writer)` — a file-backed `llm.Recorder` that appends each `CallRecord` as one JSON line.
- A `Scorecard` type (`Dataset`, `Model`, `Config`, `Judge`, `[]CaseResult`, `Totals`) with `WriteHuman(w)` (readable per-case items + metrics, then aggregate + per-difficulty) and `WriteJSON(w)` (round-trippable JSON). Assembly logic lives in `internal/eval` so it is testable without a `main`.
- `cmd/eval-extract` reads `ANTHROPIC_API_KEY` and builds the shared provider + `llm.Client` as the service does, **failing loudly at startup if absent**; builds the extract site as `extract.DefaultCallSite()` + `-model/-reasoning/-temperature/-max-tokens` layered through the **D19 parsers**, and the judge as `eval.DefaultJudgeCallSite()` + `-judge-*`; wires the JSONL recorder (`-record`); then `LoadDataset → Run → Score → Scorecard` and prints (`-json` selects JSON, else human). Exit reflects whether the run **completed**, never a score threshold (no pass/fail).
- One blessed case ships under `testdata/eval/extract/<case-name>/` (`document.txt` + `gold.json`): a compact, factual source with a clearly blessable subject-and-claim set and a closed-set `difficulty`.

**Done when:** R-34NV-WDIP (flags layer onto the extract/judge `DefaultCallSite`s via the D19 parsers; `-reasoning low`→`Level("low")`, malformed flag fails loudly, no-flags keeps defaults + Opus-4.8 judge), R-35VS-A59E (a mock-backed dataset run assembles a `Scorecard` stamped with model/config/judge, one `CaseResult` per case + `Totals`), R-373O-NX03 (`WriteHuman`/`WriteJSON` both emit the stamp and per-case/aggregate/per-difficulty subject+claim metrics; JSON round-trips), R-38BL-1OQS (the JSONL recorder captures one record per extract round-trip and per judge call, each with request/response/params), R-39JH-FGHH (missing `ANTHROPIC_API_KEY` fails loudly before any load or call), and R-3ARD-T886 (the committed gold case loads clean via `LoadCase`, difficulty in `{easy,medium,hard}`, ≥1 gold subject with ≥1 claim) are each covered by clearly-named tests (CLI guts driven through `internal/eval` with a mock-backed client; no live LLM), and the suite is green.
