# Phase 61 — Wire the recording embedder at the composition root (close the D34 gap)

*Realizes design Decision 37 (wire the recording embedder at the composition root). Closes the gap left by Phase 54 (D34): the `recordingEmbedder`, the stage-widening migration, and the `EmbedSite` config all shipped and were unit-tested, but `cmd/wiki/main.go`'s `buildSpec` installed the **raw** `agentkit.Embedder` on both embed paths, so the running binary writes **zero** `embed-page`/`embed-query` rows. Depends on Phase 54 (the wrapper, migration, and config already exist) and Phase 59 (the after-commit `embedAndStore` page-embed call site). No new migration.*

Make the recording embedder the embedder the composition root actually installs — on both the page (document) side and the query side — so every real embed lands in `llm_calls`, and prove it with a test that bites the construction `main` runs, not a package-isolated wrapper test.

- **Make the wrapper injectable and reachable (`internal/llm`).** Change `recordingEmbedder.inner` from the concrete `*agentkit.Embedder` to a minimal embedder interface (the `Embed(ctx, texts, role)` method only) so a fake embedder can be injected and the behavior tested **offline**. Capture the provider name, model, and dimensions the record reports at construction (from `EmbedSite`) instead of reading them off a concrete inner. Add an **exported** constructor — `llm.NewRecordingEmbedder(inner, recorder, stage, provider, model, dims)` — returning a value that satisfies the existing `wiki.PageEmbedder` interface. The wrapper stays in `internal/llm`; the existing R-ZBIV/R-ZCQR behavior (one record per round-trip, error on the record) must remain green.
- **Install both wrappers at the composition root (`cmd/wiki/main.go`).** In `buildSpec`, build one `LLMCallStore` recorder over the service DB — the same recorder already handed to `llmClient` via `WithRecorder` — and build **two** recording wrappers around the single `agentkit.Embedder`: stage `embed-page` and stage `embed-query`, sharing that recorder. Install the page wrapper via `wiki.WithPageEmbedder(cfg.EmbedSite.Model, pageRec)` (its parameter is already the `wiki.PageEmbedder` interface — only the wrapped value changes) and pass the query wrapper to `queryEmbedder`, **widening** `queryEmbedder`'s parameter from the concrete `*agentkit.Embedder` to the embedder interface. The raw `agentkit.Embedder` is no longer installed anywhere.
- **Extract the construction into one testable helper (`cmd/wiki`).** Factor the embedder-wrapper construction into a single small helper in package `main` (e.g. `recordingEmbedders(inner, recorder, site) (page, query wiki.PageEmbedder)`) that `buildSpec` calls. This helper is the seam the coverage test drives, so the test pins the production construction rather than a parallel re-wiring. Keep it embedder-only — do **not** refactor the rest of `buildSpec` (the general testable-composition-root work is a later, separate Decision).

**Done when:** the suite is green (per design *Conventions*: `go build ./...`, `go vet ./...`, `gofmt -l .` with no output, `go test ./...`, `bin/check-migrations wiki`) and these ids are covered. The page/query recording ids are **wiring ids**: they are satisfied only by tests that exercise the production construction helper `buildSpec` calls (in package `main`), with a real `LLMCallStore` over a migrated temp SQLite and an injected fake embedder — a package-isolated `recordingEmbedder` unit test does **not** satisfy them.

- **R-LFOY-L6TZ** — the composition root's **page** path is the recording embedder: via the production construction helper, embedding a page (document role) under `WithJobID` lands exactly one `embed-page` `CallRecord` (model, `dims` in `params`, that job id) in the real `LLMCallStore`; a raw, non-recording embedder records nothing and fails this.
- **R-SIFE-Z88I** — the composition root's **query** path is the recording embedder: a query-role embed through the same production construction lands exactly one `embed-query` `CallRecord` with an **empty** job id.
- **R-JEC4-3M6O** — end-to-end observability: an `Ingest` processed by the worker through the production-constructed `Service` leaves an `embed-page` row in `llm_calls` tied to that ingest job, and `LLMCallStore.List` (the `llm_calls` verb's source) returns it.
- **R-NWE2-CUPE** — testability seam & regression guard: the recording embedder accepts an injectable, offline-fakeable embedder, and the composition-root embedder construction is a single helper `buildSpec` calls; the three ids above are covered **only** by tests exercising that helper, not by an isolated wrapper unit test.
