# Decision 2 — Service skeleton: package layout, Spec wiring, and the config/secret composition root

**Decision.** wiki is a single `module wiki` on the appkit chassis, laid out like crm/notify. This decision fixes the module shape and composition root only; each stage's internals are their own Decisions.

**Package layout:**

```
wiki/cmd/wiki/main.go        — composition root: build Config, read ANTHROPIC_API_KEY, construct the
                                shared agentkit Provider + domain Service, declare appkit.Spec, wire the Worker.
wiki/internal/db/            — db.go (//go:embed migrations → db.FS) + migrations/*.sql
wiki/internal/wiki/          — domain core: types (Subject, Claim, Page, Job), Service, stores
wiki/internal/llm/           — agentkit seam: holds the shared *Provider; the "json-mode helper"
                                (prompt → Send once → strip code fence → unmarshal → validate → retry-on-parse).
                                Provider is shared (stateless); a fresh *Conversation is built per call.
wiki/internal/extract/       — text → (subjects, claims)            [D-later]
wiki/internal/compile/       — claims → page, 12k cap               [D-later]
wiki/internal/ask/           — ask: subject-extraction → resolve → synthesize pipeline [D-later]
wiki/internal/worker/        — ingest poll-loop Worker              [D-later]
wiki/internal/mcp/           — JSON-RPC transport (copy crm) + tool declarations/dispatch [D-later]
wiki/internal/ids/           — ULID minting (copy crm verbatim)
```

**Spec wiring** (`cmd/wiki/main.go`) — all event-plane fields empty; wiki is on no event plane in phase 1, which is what makes `reflection` empty (verified in the MCP Decision):

```go
var svc *wiki.Service
appkit.Main(appkit.Spec{
    App: "wiki", Mount: "/srv/wiki/", Port: 3006, MCP: true,
    Migrations: db.FS,
    // Feed:"", Consumes:nil, Events:{}, Publishes:nil, Subscriptions:nil
    Config:  func(getenv func(string) string) (any, error) { /* build wiki.Config, incl. AnthropicKey */ },
    ManifestExtras: []appkit.ManifestKV{ /* model id, worker concurrency */ },
    Handlers: func(rt *appkit.Router) error {
        write := rt.DB()                                 // appkit's handle = the SINGLE WRITER (MaxOpenConns 1); see D17
        read := /* db.OpenRead(cfg.DBPath): same file, query_only, MaxOpenConns(N>1) — concurrent reads (D17) */
        prov := /* agentkit anthropic.New(cfg.AnthropicKey) */
        svc = wiki.NewService(wiki.Conns{Read: read, Write: write}, prov, cfg) // captured for the Worker closure
        rt.Handle("POST /mcp", rt.RequireIdentity(
            mcp.NewHandler(svc, rt.Version(), rt.Service(), rt.Health(), rt.Events(), rt.Subscriptions())))
        return nil
    },
    Workers: []func(context.Context) error{ func(ctx context.Context) error { return worker.Run(ctx, svc) } },
})
```

- **Port 3006** — reused from the prior build (`wiki.bak` `PORT=3006`) for continuity.
- `Handlers` runs after appkit opens+migrates the DB and before `Workers`; the domain `Service` is built over the **two handles** (`wiki.Conns`, D17) and captured in a `var` the `Workers` closure reads. `rt.DB()` is the single writer; the read handle opens over the already-migrated file and runs no migrations.
- **Do not hand-spawn a goroutine in serve** — the ingest worker is a `Spec.Workers` entry; appkit runs it on the serve ctx and SIGTERM cancels it. A worker that *returns* is a structural fault that brings the server down, so it retries transient faults inside its loop and exits only on `ctx.Done()`.

**Config & secret seam:**
- **Non-secret knobs** (model id, worker concurrency) → `ManifestExtras` → `etc/manifest.env`, read at the composition root into a `wiki.Config` passed to domain code by argument. The **12,000-char page cap is a product constant honored by value** — a named `const` in the compile package, not a knob.
- **Secret** `ANTHROPIC_API_KEY` → read once at the composition root via `getenv`; **fail loudly at startup if absent** (crash, never serve degraded). Locally injected by `wiki/.envrc` (`source_up` + `export ANTHROPIC_API_KEY="$(cat ~/.secrets/ANTHROPIC_API_KEY)"`); never enters code beyond the hand-off to `anthropic.New`.
- The Provider is constructed **once** and shared; each LLM call builds a **fresh `*Conversation`** (Conversation is not concurrency-safe / single-flight).

**Rejected.**
- **Hand-spawning the ingest goroutine inside `serve`** — loses appkit's ctx-cancel/SIGTERM lifecycle and the structural-fault guarantee.
- **Opening a second *writer* in the service** rather than `rt.DB()` — appkit owns open+migrate+the single-writer connection; a second *write* handle breaks the single-writer invariant. (A second **read-only** handle does **not** — it adds no writer; that is D17's concurrent read pool, the sanctioned exception.)
- **A bring-your-own secret mechanism** (flag, config literal, hardcoded fallback) — violates the project secret rule; the only path is `~/.secrets` → `.envrc` → env → `getenv`.
- **Port 3007** (research's guess) — the prior build is on 3006; reuse avoids a manifest/nginx reallocation.

**Verification.**
- R-6RVX-P1IG — Starting `serve` with `ANTHROPIC_API_KEY` unset (or empty) fails loudly at startup (non-zero exit / startup error) rather than starting the HTTP server.

(The package layout, Spec field values, and worker-vs-goroutine choice are structural — their behavioral proof is carried by the ids of the Decisions they enable: schema, ingest/extract/compile, retrieve/ask, and the MCP surface, which also verifies the empty `reflection` and the `health` envelope.)
