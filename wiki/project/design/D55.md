# Decision 55 — Delete the `internal/db` chassis shim (keep the read handle) and true up the docs

**Decision.** Two cleanups that land the reference shape whole, mirroring crm's
and notify's shim deletion — with one wiki nuance kept and named.

1. **`internal/db` shrinks to its legitimate content.** The `Open` and `Migrate`
   wrapper functions in `internal/db/db.go` are **deleted** — appkit owns DB
   opening and the migration run (`Spec.Migrations: db.FS`). What genuinely
   remains, and is explicitly kept:
   - the embedded migration set (`db.FS` via `//go:embed migrations/*.sql`;
     `go:embed` cannot cross modules);
   - **`internal/db/open_read.go`'s `OpenRead` and `Path`** — wiki's **second,
     query-only read handle** (D17 concurrency: a pooled read pool separate from
     appkit's single writer). This is **not** a chassis shim: appkit opens the
     **writer** (`rt.DB()`), and wiki opens the **reader** against the same file
     at serve (`db.Path(ctx, rt.DB())` → `db.OpenRead(path)`). appkit provides
     one write handle; wiki's split-handle concurrency model needs a concurrent
     read pool that has no chassis equivalent — so it stays, and this Decision
     records why.

   Test harnesses that called `db.Open`/`db.Migrate` call appkit's db package
   directly — no test *assertions* change, only harness plumbing.

2. **The docs state the converted truth.** `wiki/AGENTS.md` (edit `AGENTS.md`;
   `CLAUDE.md` is its symlink — one file) and the design **spine**
   (`project/design/design.md`) state the current shape, per suite doctrine
   (docs state current truth; purge what the change falsified; no archaeology).
   The falsified claims: the read surface's assets are `//go:embed`-embedded in
   `internal/web` (now they ship on disk under `share/www`, served by the chassis
   via `Spec.WWW`/`rt.WWW()` — D52), and the MCP surface is a hand-rolled
   JSON-RPC handler (now a tool table over `appkit/mcp` — D53). AGENTS.md asserts
   little internal detail today, so its truth-up is light; the spine's "Web
   surface" embed paragraph and the "MCP tested via the JSON-RPC handler" testing
   line are rewritten in place (done in the authoring pass that introduced D52–D54).

**Rejected.**
- **Deleting `open_read.go` with the wrappers** — it is wiki's genuine second
  read handle (D17), not a chassis shim; conflating the two would break the
  read-pool concurrency model.
- **Deleting `internal/db` entirely** — the migrations embed must live in a wiki
  package.
- **Leaving the docs for a later pass** — the docs would assert wiring that no
  longer exists in the same tree that changed it; docs move with the code.

**Verification.** Structural — no `R-XXXX-XXXX` ids (the behaviors are already
pinned by the existing domain/MCP suites and the D51–D53 ids). Realized by a plan
phase whose done bar is mechanical:

- `grep -n "func Open\|func Migrate" wiki/internal/db/db.go` returns no matches,
  while `grep -n "go:embed migrations" wiki/internal/db/db.go` still matches and
  `grep -n "func OpenRead\|func Path" wiki/internal/db/open_read.go` still matches
  (the read handle is retained);
- `grep -rn "go:embed" wiki/internal/web/` returns nothing (the web embed is
  gone — D52) and `grep -rn "JSON-RPC\|writeJSONRPCError\|hand-rolled" wiki/AGENTS.md`
  returns nothing;
- the suite stays green (design *Conventions* commands) with no assertion
  changes.
