# Decision 6 ‚Äî The extract stage (`internal/extract`)

**Decision.** A single-pass, tool-less, structured LLM call over the D5 `llm.JSON` helper: the whole source text plus a mechanical header ‚Üí subjects, each with its claims. Ported from wiki.bak's extract, minus aliasing.

```go
package extract

type ExtractedSubject struct {
    Type       string   // "entity" | "event" | "concept"
    Kind       string   // freeform subtype, prompt-anchored ("person","company","method"‚Ä¶)
    Name       string   // primary name AS THE TEXT STATES IT (registry normalizes later)
    OccurredAt string   // ISO-8601 prefix (YYYY / YYYY-MM / YYYY-MM-DD); required for events, optional for entity/concept; "" only when the document states no defining date
    Claims     []string // short self-contained prose statements, no pronouns
}

// DocumentHeader anchors relative-time resolution; rendered verbatim, never model-inferred.
type DocumentHeader struct {
    Source     string    // phase 1: "mcp:ingest_text"
    Title      string
    Tags       []string
    ReceivedAt time.Time // rendered "received on: YYYY-MM-DD" (never "today is")
}

type Extractor struct { c *llm.Client; site llm.CallSite }
func New(c *llm.Client, site llm.CallSite) *Extractor
func (e *Extractor) Extract(ctx context.Context, h DocumentHeader, text string) ([]ExtractedSubject, error)
```

- **Prompt** ‚Äî port wiki.bak's `DefaultExtractPrompt`, **dropping the `aliases` field and the within-doc alias/co-reference clause** (no aliasing in phase 1). Keep verbatim the two load-bearing rules: the **salience gate** ("extract a subject only when it is identifiable ‚Äî the Wikipedia-article test ‚Äî AND the document makes a concrete claim about it; never invent a subject to hold a stray fact") and the **claims discipline** ("each claim is a short SELF-CONTAINED prose statement, no pronouns; state only what the document ASSERTS ‚Äî do not infer or synthesize"). Keep the dialog-awareness clause and a worked example (rewritten without aliases). Closed-set type. `occurred_at` is an ISO-8601 prefix the model fills **whenever the document states a defining date for the subject** ‚Äî required for an `event`, equally welcome on an `entity` (birth, publication) or `concept` (when first named); `""` only when the document states no such date. Relative time resolved against the header's received-on date, kept relative when unresolvable.
- **`validate`** (the D5 caller hook): every subject's `Type` ‚àà `{entity,event,concept}`; `Name` non-empty after trim; ‚â•1 claim, each with non-empty `text`; and **`OccurredAt`, when non-empty, is a valid ISO-8601 prefix** (`YYYY` / `YYYY-MM` / `YYYY-MM-DD`) for **any** subject type, and is **required** (non-empty, valid prefix) when `Type=="event"`. A violation is a parse-level failure ‚Üí bounded re-prompt. A stated `OccurredAt` is **retained as extracted on every subject type ‚Äî never coerced to `""` and never rejected for being present on a non-event**.
- **Empty is honest, not an error.** A text with no salient subject yields `[]`; the job completes with zero subjects. Extract never invents a subject to avoid emptiness.
- **Call site** ‚Äî the package **owns** its designed defaults and exposes them so wiring cannot silently drop them: `extract.DefaultCallSite(model string) llm.CallSite` returns `{Model: model, Temperature: ptr(0), Reasoning: DisableReasoning(), MaxParseRetries: 2, MaxTokens: <generous extract budget, D18>}` (further configurable via `ManifestExtras`). The non-zero `MaxTokens` (D18) keeps a large extraction off the 4096 adapter default that truncated jobs into `unexpected end of JSON input`. **The composition root constructs the extractor from `DefaultCallSite`, never a bare `llm.CallSite{Model: ‚Ä¶}`** ‚Äî the original drift was exactly that bare construction, which ran extraction at provider-default temperature, reasoning **on**, and **zero** retries (a single fenced-parse failure killed the job with no re-prompt). Single-pass and tool-less ‚Äî reads only header + source text, reinforcing the anti-poisoning invariant (D4 R-MG31).

**Rejected.**
- **Keeping the `aliases` field** ‚Äî feeds deferred alias machinery; in phase 1 it is dead weight that invites over-merging surface forms.
- **Treating empty `subjects[]` as a failure/retry** ‚Äî would pressure the model to fabricate subjects, violating the salience gate. Empty is valid.
- **Haiku for extract** ‚Äî cheaper, but extraction error is frozen into the durable record and silently recompiled forever (research ¬ß5); the stronger model is warranted (still a config knob).
- **The `occurred_at`-empty-unless-event rule** ‚Äî original D6 emptied (`coerce ""`), and the ported code went further and *rejected*, any date on a non-`event` subject. But the model naturally extracts a subject's defining date regardless of type (birth/publication/founding year), so the reject failed whole ingests **deterministically** (the "Gary Gygax" doc, where every `entity` carries a year) while the coerce variant silently discarded real durable temporal signal. `occurred_at` is now an optional, format-validated field on **every** subject type.

**Verification.**
- R-VYU0-BPAX ‚Äî `validate` rejects an extraction containing a subject whose `Type` is outside `{entity,event,concept}` (a parse-level failure that triggers a re-prompt).
- R-XJBY-H8JZ ‚Äî A valid extraction **retains** a non-empty `OccurredAt` on a non-`event` subject (`entity`/`concept`) exactly as extracted ‚Äî a stated date on a person or work is never emptied, dropped, or rejected for being present on a non-event.
- R-XKJU-V0AO ‚Äî `validate` rejects a subject whose non-empty `OccurredAt` is not a valid ISO-8601 prefix (any type) and rejects an `event` whose `OccurredAt` is empty (a parse-level failure ‚Üí re-prompt); a valid prefix on an `entity`, `concept`, or `event` passes.
- R-W19T-38SB ‚Äî A source with no salient subject yields an empty `[]ExtractedSubject` and a successful (non-error) extract ‚Äî no invented subject.
- R-W2HP-H0J0 ‚Äî The extract call site is configured with `Temperature: 0` and `DisableReasoning()` (deterministic extraction; confirmed via the capturing mock provider).
- R-4CK8-E688 ‚Äî `extract.DefaultCallSite` carries `Temperature: 0`, `DisableReasoning()`, and `MaxParseRetries: 2`, and the extractor the composition root builds is constructed from it ‚Äî so a parse/validate failure is actually re-prompted (a bad-then-good extraction succeeds through the production-configured extractor) rather than failing on the first attempt.
