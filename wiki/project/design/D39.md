# Decision 39 — The human web surface: gating, embedded assets, and the exact-root mount

> **Asset substrate superseded by D52.** The embedded (`//go:embed`) assets described below now ship on disk under `wiki/share/www`, served by the appkit chassis (`Spec.WWW` + `rt.WWW()` + chassis `/static/`); `internal/web` remains the router. The gating, exact-root, and session-gate facts and the R-ids below are unchanged — only where the bytes live changed.

**Decision.** wiki serves an HTML **human web surface** beside its MCP/`/health`/PRM JSON surfaces. This Decision owns the **spine common to that whole surface** — how it is gated, how its assets ship, and how its root is mounted without shadowing the JSON routes. The surface's *content* — the home/ask page, subject pages, and the orphan index — is owned by D42 (foundation/routing), D43 (home), D44 (ask), D45 (subject page), D46 (orphan index); the expansion of the session gate to cover the subject and asset sub-paths is owned by D47. wiki keeps everything it already is (MCP tools, `/health`, the RFC 9728 PRM doc); the web surface is **added beside** them, never in place of them.

(History: the web surface began as a single name+version landing card. It is now the read surface D42–D46 describe; the name+version it still must show lives in the home page **footer** (D43), preserving that promise. This Decision is rewritten to own only the cross-cutting gating/asset/mount spine — the part that is the same whether the root renders a card or a search box.)

**Two front doors, two audiences (the gate is the spine of this Decision).**
Under `/srv/wiki/` there are two kinds of caller, each authenticated differently:

- **Agents / MCP clients → bearer token.** `POST /mcp` (and `/health`, `/feed`) stay gated by `auth_request /_authn` (opaque bearer). **Unchanged** — the existing prefix `location /srv/wiki/` fragment keeps gating them.
- **Humans in a browser → login-session cookie.** The web surface is gated by `auth_request /_session-authn` — the dashboard-owned, loopback-only endpoint that validates the `dashboard_session` cookie against the web-session store. This is the **same** mechanism `sites` uses for its private tier. It is deliberately **coarse**: any logged-in dashboard user may view any app's human surface (no per-resource check) — correct for a read-only navigation surface inside the tenant boundary.

A browser cannot satisfy the bearer gate (it has no token), and the bearer gate is the wrong audience for a human page; so the web surface **must** be cookie-gated, not bearer-gated — and not fully public, since the suite exposes nothing un-gated except the PRM bootstrap doc.

**The seam — Go side (`cmd/wiki/main.go` `Spec.Handlers` hook).** The web routes are mounted **ungated in-process** — nginx is the sole gate, exactly as the MCP and PRM routes already trust nginx. The composition root mounts the D42 handler (which owns the route table — `GET /{$}`, `GET /subject/{type}/{slug}`, `GET /static/`):

```go
h := web.NewHandler(rt.Service(), rt.Version(), wiki.Mount, web.WithAsker(...), web.WithPageFinder(...), web.WithOrphanLister(...))
rt.Handle("/", h)   // ungated; nginx is the gate
```

- The root pattern `GET /{$}` (Go 1.22+ ServeMux) matches **only** the exact root path `/`, so it cannot shadow `/mcp`, `/health`, `/feed`, or `/.well-known/oauth-protected-resource`; the `GET /subject/{type}/{slug}` and `GET /static/` routes are likewise collision-free (D42). Those JSON routes are untouched.
- The web routes are **not** wrapped in `rt.RequireIdentity(...)` (that wrapper is the bearer gate, for MCP). The cookie gate lives entirely in nginx; in-process the routes are open and trust the front door — the established suite posture.
- `rt.Service()` (`"wiki"`) and `rt.Version()` (the build `VERSION`, e.g. `0.6.0`) feed the home footer (D43); `wiki.Mount` (`"/srv/wiki/"`) feeds the `<base href>` (D42).

**The seam — assets (the `wiki/internal/web/` package).** wiki embeds its **own** copy of the Carbon assets and templates, mirroring the dashboard precedent (`dashboard/ui/`):

- `wiki/internal/web/` holds the templates (the shared layout + `home`/`answer`/`subject`, D42) and `static/tokens.css` + the woff2 fonts, copied from the repo-root Carbon source (`design/tokens.css` + fonts; rules in `design/carbon.md`). They are embedded via `//go:embed` so the binary stays one static file — no on-disk asset dependency at runtime.
- The handler serves the templates and the embedded static assets under `GET /static/…`, which the pages reference (via `<base>`-relative `static/tokens.css`) for the CSS and fonts. The static route is ungated-in-process and session-gated in nginx (D47), like the pages it styles.
- Visual form (Carbon, `design/carbon.md`): monochrome neutrals, blue `#2563EB` the only signal color, Space Grotesk display / IBM Plex Sans body / IBM Plex Mono labels, 4px spacing grid, hairline borders, no JS required.

**The seam — nginx (`wiki/etc/nginx.conf`).** An **exact-match** location for the bare mount root session-gates and proxies to the app's `GET /{$}`, beside the existing gated prefix:

```nginx
location = /srv/wiki/ {
    auth_request /_session-authn;
    auth_request_set $wiki_owner $upstream_http_x_owner_email;
    proxy_set_header X-Owner-Email $wiki_owner;
    proxy_pass http://127.0.0.1:3001/;   # trailing slash → upstream sees "/"
    proxy_set_header Host $host;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_http_version 1.1;
}
```

The existing `location /srv/wiki/ { auth_request /_authn; … }` **prefix** stays for the agent paths (`/mcp`, `/health`, `/feed`); the exact-match `=` root wins for the bare root (and the exact-match PRM location is unaffected — exact matches over distinct paths coexist). **D47** adds the two further session-gated prefix locations (`/srv/wiki/subject/`, `/srv/wiki/static/`) that extend this same cookie gate over the rest of the human surface; this Decision owns the exact-root block, D47 owns the expansion.

**Stale-doc consequence.** wiki's `AGENTS.md`/`CLAUDE.md` "**no UI**" line is false (wiki serves a human surface). Its correction is a dedicated plan phase (Phase 65) so code and docs evolve together through the build loop. The "no token logic" half stays true: the web surface reads no token and runs no token logic — nginx is still the trust boundary.

**Rejected.**
- **A shared web handler in appkit.** Each app's surface diverges to serve that app's purpose; wiki owns its handler + templates + assets so divergence is a local edit, not a shared-library change. (The Carbon *tokens* are copied, not shared code.)
- **Bearer-gating the human surface (`auth_request /_authn`).** A browser carries no opaque bearer token, so the human pages would be permanently unreachable. The audience is a logged-in human; the cookie session is the right credential.
- **Serving the human surface fully public (no auth).** The suite exposes nothing un-gated except the PRM bootstrap doc; the surface is inside the tenant boundary. Coarse session-gating matches `sites`' private tier and keeps one consistent rule.
- **Wrapping the web routes in `RequireIdentity` in-process.** That is the bearer gate; it would reject browsers and duplicate the trust boundary nginx owns. The routes are ungated in-process by design — nginx is the sole gate, as for `/mcp` and PRM.
- **A catch-all `GET /` for the root instead of `GET /{$}`.** A bare `/` prefix would shadow `/mcp`, `/health`, `/feed`, and PRM. `{$}` matches the exact root only, so the root route is collision-free by construction (the other web routes carve their own collision-free prefixes, D42).
- **Reusing the dashboard's `ui` package.** wiki is a separate Go module; it embeds its own copy of the Carbon assets (the dashboard precedent is the *pattern* to copy, not a dependency to import).

**Verification.**
- R-LAND-PG01 — `GET /{$}` (exact root `/`) returns **200** with `Content-Type: text/html; charset=utf-8` — driven through the handler the composition root mounts (httptest), no DB/LLM/identity required for the response shell.
- R-LAND-NMVR — the rendered root page shows the **service name** (`rt.Service()`, `"wiki"`) and the **version** (`rt.Version()`) in its footer; built with a non-default injected version, the page shows that exact version (proving it renders `Version()`, not a hard-coded literal).
- R-LAND-CARB — the surface is Carbon-styled from **embedded** assets: the `wiki/internal/web` `embed.FS` contains `tokens.css` and the woff2 font(s), the served page references `tokens.css` (via the `<base>`-relative `static/tokens.css`), and a request for the embedded `tokens.css` asset returns 200 with a CSS content-type — proving the assets ship in-binary, not from disk.
- R-LAND-ROOT — `{$}` matches **only** the exact root: a request to a non-root path the root handler does not own (e.g. `/health`, `/mcp`, `/nope`) is **not** served the root page by this route — the root mount does not shadow the existing JSON routes.
- R-LAND-UNGT — the web routes are mounted **ungated in-process** (not wrapped in `RequireIdentity`): an httptest request with **no** identity header / no bearer still gets the 200 root page, proving the in-process routes trust nginx as the sole gate (the cookie gate is asserted in nginx, D47 / the fragment-check phase).
