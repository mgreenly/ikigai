# Decision 9 ‚ÄĒ `ask` (`internal/ask`): subject-extraction pipeline, grounded/cited/honest-empty

**Decision.** `ask` extracts the subjects a question names, resolves each to a wiki subject by exact normalized name, reads those subjects' pages, and synthesizes a cited answer from them. Control flow is a **deterministic three-stage pipeline** in Go; the LLM is invoked twice as tool-less structured generations through the `llm.JSON[T]` seam (D5) ‚ÄĒ once to extract names, once to synthesize. **No keyword retrieval, no tool-loop, no source/claims drill-down**: ask draws on the compiled pages only. Strictly read-only, permanently.

```go
package ask

type Answer struct {
    Found     bool       // false ‚Üí the wiki holds nothing on the question
    Text      string     // synthesized answer, or the fixed honest-empty sentence
    Citations []Citation // the gathered pages the answer drew on
}
// Citation is public: a cited page identified by its type/slug Path (D11),
// never the internal subject id. (The synthesis call cites internally by
// subject id; ask maps each validated citation to wiki.Path before returning.)
type Citation struct { Path, Title string }

type Asker struct {
    subjects    *wiki.SubjectStore  // GetByNormName ‚ÄĒ exact, normalized resolution
    pages       *wiki.PageStore     // GetBySubject  ‚ÄĒ the compiled page per subject
    c           *llm.Client
    extractSite llm.CallSite         // stage 1 config (NER over the question)
    synthSite   llm.CallSite         // stage 3 config (grounded synthesis)
}
func New(subjects *wiki.SubjectStore, pages *wiki.PageStore, c *llm.Client, extractSite, synthSite llm.CallSite) *Asker
func (a *Asker) Ask(ctx context.Context, owner, question string) (Answer, error)
```

**Flow ‚ÄĒ extract ‚Üí resolve & gather ‚Üí synthesize:**

1. **Extract (one LLM call).** `llm.JSON[extractResult](ctx, c, extractSite, question, validate)` returns the candidate subject **names**. The validator only requires a well-formed (possibly empty) list.
   ```go
   type extractResult struct { Subjects []string `json:"subjects"` }
   ```
2. **Resolve & gather (deterministic ‚ÄĒ no LLM).** For each extracted name: resolve via the shared `Resolver` (D25) ‚ÄĒ `Normalize(name)` against live subjects, falling back to the merge-alias table ‚ÄĒ on hit, `PageStore.GetBySubject`. Dedup resolved subjects by id; preserve first-seen order. Resolution is **exact-normalized** and honors **merge aliases** (a merged-away name resolves to its survivor); an un-merged partial/variant name ("Gygax" for "Gary Gygax") resolves to nothing (fuzzy/semantic matching is a later release). **Best-effort partial**: every name that resolves is gathered; the rest are silently skipped.
   - **Zero subjects resolve ‚Üí honest-empty: return `Answer{Found:false, Text:<fixed sentence>, Citations:nil}` and make NO synthesis call.** The synthesis model never runs against an empty page set, so it cannot fabricate from world knowledge ‚ÄĒ the deterministic anti-fabrication lever.
3. **Synthesize (one LLM call).** `llm.JSON[answerResult](ctx, c, synthSite, synthPrompt(question, gathered), validate)` over the gathered pages only.
   ```go
   type answerResult struct {
       Found     bool       `json:"found"`
       Text      string     `json:"text"`
       Citations []Citation `json:"citations"`
   }
   ```
   The `ExtractJSON` carve and bounded parse-retry live inside `llm.JSON[T]` (D5); `ask` carries no JSON-from-decoration logic.

**Grounding & anti-fabrication (pages-only):**
- A `Found:true` answer **must carry ‚Č•1 citation**; a found-with-no-citation reply is **downgraded to honest-empty**.
- Each citation is **validated against the gathered set**; a citation to anything not gathered is **dropped**. Since the gathered set came from the page store, a surviving citation is by construction a real page. Each surviving citation is mapped from subject to the public `type/slug` **Path** (D11) before reaching the caller ‚ÄĒ never an internal subject id.
- The synthesis prompt is fed **only the gathered page bodies** ‚ÄĒ never raw claims or original source text. No drill-into-source tool or path exists in `ask`.
- A full NLI entailment verifier is a noted known gap, out of scope here.

**`ask` is strictly read-only ‚ÄĒ permanently.** `Asker` only reads (`GetByNormName`, `GetBySubject`); no write path exists on any branch, including honest-empty and parse-failure.

**Prompts.**
- `extractSite.System` ‚ÄĒ return only the proper-noun **subjects** the question is *about*, as a JSON list of names, nothing else; no answering, no speculation; empty list when the question names no subject.
- `synthSite.System` ‚ÄĒ answer **only** from supplied pages; cite each page drawn on by subject id + title (`ask` maps id to public path after validation); return `found:false` if the pages don't answer; never fabricate a fact, page, or citation; surface contradictions with both sides; read-only. JSON output schema + example.

**Call sites** (defaults, configurable):
- `extractSite`: `anthropic.ModelSonnet46`, `Temperature: 0`, `Reasoning: DisableReasoning()` ‚ÄĒ deterministic NER.
- `synthSite`: `anthropic.ModelSonnet46`, `Reasoning: Level("low")`, `Temperature` provider-default.

**Relationship to the public surface.** The MCP `ask` tool (D10) calls `Asker.Ask` and returns `Answer{Found,Text,Citations}`, each citation a `{path, title}` (D10/D11). D10 surfaces no ask `Sources` field (`read_source` is gone). `ask` no longer composes over `retrieve.Service` (fate of that seam is D8).

**Rejected.**
- **Agentic tool-loop (`resolve_subject`/`read_page`, model drives the loop)** ‚ÄĒ behavior depends on whether the model elects to call tools, making the honest-empty gate fuzzy and stages hard to pin with checkable ids. The deterministic pipeline tests each seam in isolation and keeps the empty-gate in Go.
- **Keyword pre-flight probe over page bodies (old D9/D8 path)** ‚ÄĒ subject-extraction-only is the product decision. The "no LLM call on empty" property is unattainable once extraction (an LLM call) is the first step; the new gate saves the *synthesis* call instead.
- **`read_source` / raw-claims drill-down** ‚ÄĒ pages-only is the product decision. Dropped (and with it `Answer.Sources`).
- **Accepting `Found:true` with zero citations** ‚ÄĒ surfaces ungrounded claims; downgrade to honest-empty.
- **All-or-nothing resolution (any unresolved name ‚Üí empty)** ‚ÄĒ discards real, citable knowledge over one uningested name; best-effort partial instead.
