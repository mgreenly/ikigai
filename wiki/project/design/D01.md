# Decision 1 — Dependency on the external agentkit (the prod-build wiring)

**Decision.** wiki depends on `github.com/ikigenba/agentkit` as a **published, proxy-fetched module** with **no `replace`** in `wiki/go.mod`.

- `wiki/go.mod` pins a real release: `require github.com/ikigenba/agentkit v0.1.0` (advanced via `go get github.com/ikigenba/agentkit@vX.Y.Z`).
- In-repo `appkit`/`eventplane` stay committed replace-siblings (`replace appkit => ../appkit`, `replace eventplane => ../eventplane`). agentkit is the first lib to move to the published model; appkit/eventplane follow later (out of scope).
- **Prod build:** `bin/ship` builds HEAD off-box in a throwaway worktree with `GOWORK=off` + network. With no `replace`, agentkit resolves from the public proxy at the pinned tag. The dead in-repo `agentkit/` module (imported by prompts/sites as `module agentkit`) is untouched.
- **Local co-dev:** edits to `~/projects/agentkit` are picked up via `replace github.com/ikigenba/agentkit => /home/mgreenly/projects/agentkit` in the **root `go.work` only** — never in `wiki/go.mod`. `go.work` is ignored under `GOWORK=off`, so it never reaches prod. wiki joins the workspace by swapping `use ./wiki.bak` → `use ./wiki`.
- agentkit takes its API key by explicit injection (`anthropic.New(apiKey, …)`); never reads env. wiki reads `ANTHROPIC_API_KEY` at its composition root and passes it in.

**Rejected.**
- **Committed absolute-path `replace … => /home/mgreenly/projects/agentkit` in `wiki/go.mod`** — breaks `bin/ship`: the throwaway worktree has no such path and builds `GOWORK=off`. This is the failure mode the decision avoids.
- **Move agentkit in-repo as a replace-sibling** (appkit/eventplane pattern) — forks agentkit's source into two trees (mono-repo vs. the standalone `github.com/ikigenba/agentkit` repo its module path + `v0.1.0` tag mark authoritative) and touches two legacy services. Rejected to keep agentkit an independent published library.
- **Vendor (`go mod vendor`)** — the suite uses replace-siblings/proxy, not a committed `vendor/`; adds a large committed tree + re-vendor on every bump for no benefit.

**Verification.**
- R-MV3L-QS7I — A production-shaped build of wiki (`GOWORK=off`, no `go.work` in scope, local `~/projects/agentkit` path absent) compiles, resolving `github.com/ikigenba/agentkit` at its pinned tag from the module proxy.
- R-MWBI-4JY7 — `wiki/go.mod` contains a versioned `require github.com/ikigenba/agentkit` and **no** `replace` for it (the only `replace` directives are the in-repo `appkit` and `eventplane` siblings).
