# Decision 37 — Wire the recording embedder at the composition root (close the D34 gap)

**Decision.** The recording embedder that Decision 34 specified must be **installed by the composition root on both embedding paths** — the page (document) side and the query side — so that every real embed call lands in `llm_calls`, exactly as D34 promised. The raw `agentkit.Embedder` is **never** installed in production. And the Verification ids that prove this are covered **only** by a test that exercises the *actual production construction* `buildSpec` runs — never by a package-isolated unit test of the wrapper.

**The gap this closes.** D34's final paragraph already said "The composition root builds the recording embedder from the configured `EmbedSite` and the existing recorder." Phase 54 built the pieces — `recordingEmbedder`, the stage-widening migration, the `EmbedSite` config — and unit-tested them, then marked the phase green. But `cmd/wiki/main.go`'s `buildSpec` passed the **raw** `*agentkit.Embedder` straight into `wiki.WithPageEmbedder(...)` (the page path) and `queryEmbedder(...)` (the query path). The `recordingEmbedder` type is referenced only by its own `_test.go`. The net effect on the running binary: **zero `embed-page` / `embed-query` rows are ever written** — an operator probing `llm_calls` sees only `extract` and `compile`, and embedding activity (a real paid call) is invisible to `llm_calls`, `status`, and `jobs`. This is wiring drift: the feature is implemented and tested in its package but never reached from `main`.

**The seam.**

- **Make the wrapper injectable and reachable.** Change `recordingEmbedder.inner` from the concrete `*agentkit.Embedder` to a minimal embedder interface (the `Embed(ctx, texts, role)` method only) so a fake embedder can be injected and the recording behavior tested **offline** (no network, no `OPENAI_API_KEY`). The provider name, model, and dimensions the record reports are captured at construction (from `EmbedSite`) rather than read off a concrete inner. Add an **exported constructor** (`llm.NewRecordingEmbedder(inner, recorder, stage, provider, model, dims)`) returning a value that satisfies the existing `wiki.PageEmbedder` interface — the wrapper stays in `internal/llm`. (Code is not immutable; only migrations are. This refactor adds no migration.)
- **Install both wrappers at the composition root.** `buildSpec` builds one `LLMCallStore` recorder over the service DB (the same one already handed to `llmClient` via `WithRecorder`), then builds **two** recording wrappers around the single `agentkit.Embedder` — stage `embed-page` and stage `embed-query` — sharing that recorder. The page wrapper is installed via `wiki.WithPageEmbedder(cfg.EmbedSite.Model, pageRec)`; the query wrapper is passed to `queryEmbedder`, whose parameter is widened from the concrete `*agentkit.Embedder` to the embedder interface so a wrapper can be supplied. (`WithPageEmbedder` already takes the `wiki.PageEmbedder` interface, so the page side needs no signature change — only the wrapped value changes.)
- **Extract the construction into one testable helper.** The embedder-wrapper construction `buildSpec` performs is factored into a single small helper in package `main` (e.g. `recordingEmbedders(inner, recorder, site) (page, query wiki.PageEmbedder)`) that `buildSpec` calls. **This helper is the seam the coverage test drives** — so the test pins the production construction, not a parallel re-wiring. This is the narrow, embedder-only version of a testable composition root; the general extraction is out of scope (see *Scope*).

**The coverage bar (the anti-drift rule, scoped to this seam).** The page/query recording ids below are satisfied **only** by a test that (a) calls the production construction helper `buildSpec` actually uses and (b) asserts an end-to-end ingest leaves an `embed-page` row in the real `llm_calls` store. A unit test that constructs `recordingEmbedder` by hand and asserts it records — which is exactly what passed Phase 54 green while production recorded nothing — **does not** satisfy these ids. The verify step treats these as wiring ids: coverage must reach the behavior through `main`'s construction.

**Scope.** This Decision closes the D34 wiring gap **for the embedder only**. The general guard against this whole class of drift — a testable composition root for the entire service (`buildSpec` reduced to a thin shell over a `wire()` constructor), a verify-prompt contract that "covered ⇒ reached from `main`" for every runtime-capability id, and a `deadcode`-rooted-at-`main` reachability ratchet in the green suite — is deliberately **deferred to a later Decision**, to be taken up after the current feature set ships.

**Rejected.**
- **Editing D34 / Phase 54 in place.** D34's ids are shipped and frozen; the missing behavior is new verification, so it gets a new Decision and freshly minted ids — D34 stays unchanged.
- **Re-recording by hand inside `embedAndStore` / `queryEmbedder`.** Calling `Recorder.Record` directly at each call site duplicates the wrapper D34 already built and re-introduces two more drift surfaces. The wrapper is the seam; install it, don't reimplement it.
- **A package-isolated wrapper test as the coverage bar.** That is precisely the Goodhart hole that let Phase 54 flip green with production recording nothing. The bar must bite the construction `main` runs.
- **Recording embeds through the chat `llm.Client`.** Already rejected by D34 (different interface, different inputs); unchanged here.

**Verification.**

A raw, non-recording embedder records nothing and fails this test.
- R-SIFE-Z88I — The composition root's **query** embed path is the recording embedder: a query-role embed through the same production construction lands exactly one `embed-query` `CallRecord` with an **empty** job id.
- R-JEC4-3M6O — End-to-end observability: an `Ingest` processed by the worker through the production-constructed `Service` leaves an `embed-page` row in `llm_calls` tied to that ingest job, and `LLMCallStore.List` (the source the `llm_calls` verb reads) returns it — the operator-visible footprint whose absence surfaced this gap.
- R-NWE2-CUPE — Testability seam & regression guard: the recording embedder accepts an injectable, offline-fakeable embedder, and the composition-root embedder construction is a single helper `buildSpec` calls; the three ids above are covered **only** by tests that exercise that helper, and a package-isolated `recordingEmbedder` unit test does **not** satisfy them.
