# Decision 28 — Blackhole empty-normalization content

**Decision.** An extracted subject whose `Name` normalizes to `""` (D3 — `Normalize` may return `""` for input with no alphanumeric content) has no valid identity or path, so it is **blackholed**: dropped at the ingest integration boundary. It produces **no subject row, no claims, and no page**; the attached content is discarded.

The guard lives at the top of the per-extracted-item loop in `planIntegration` (`internal/wiki/service.go`), before the subject is planned or its claims collected:

```go
for _, item := range extracted {
    if Normalize(item.Name) == "" {
        continue // blackhole: no subject, no claims, no page
    }
    subject, isNew, err := s.plannedSubject(ctx, knownByNorm, item)
    ...
}
```

Because the item is skipped before `plannedSubject`, no `norm_name = ""` row is inserted (the D3 `NOT NULL UNIQUE` constraint is never exercised with an empty key), its claim bodies are never added to the plan, and no page is compiled. Other extracted subjects in the same job integrate normally.

This is the **only** place empty-normalization is handled: `Normalize` stays a pure function returning `""` (D3, no empty special-case), and `GetByPath` independently rejects an empty path token (D11). The integrate path hosting this guard is the single-worker, atomic commit of D4/D14.

**Rejected.**
- **Insert the subject with `norm_name = ""` and suppress only the page.** Persists an identity-less subject, the empty key collides under `UNIQUE(norm_name)` once a second appears, and it leaves orphan claims.
- **Hard-error the whole ingest job on an empty result.** One pathological name would discard a whole document's valid subjects. Blackholing the single item fails the bad unit, not the batch.
- **A placeholder/sentinel `norm_name`.** Invents fake identity and groups unrelated blackholed content under one subject.

**Verification.**
- R-Z5JL-2IBS — integrating an extraction containing an item whose `Name` normalizes to `""` (e.g. `"???"`) creates **no** subject for it: after the job, no subject row exists with `norm_name = ""` and the blackholed name is absent from the subject registry.
- R-Z6RH-GA2H — the claims carried by that blackholed item are **not persisted**: none of its claim bodies appear attached to any subject after integration (content discarded, not reattached).
- R-Z7ZD-U1T6 — in the same extraction, a sibling item with a non-empty normalization is unaffected: its subject is created and its page compiled — proving the blackhole drops only the empty-normalizing item, not the batch.
