# Decision 42 — Web read surface: foundation (routes, base-href, package shape, seam injection)

> **Render substrate superseded by D52.** `internal/web` stays the read-surface package (route table, base-href, injected seams) exactly as below, but its templates and static assets load from `wiki/share/www` through the chassis-provided `*appkit/web.Site` (`rt.WWW()`), not `//go:embed`; the `GET /static/` mount is chassis-owned. The R-ids below are re-proven over the on-disk tree.

**Decision.** The placeholder landing page (D39) becomes a small **read-only human web surface** — home/ask, subject pages — grown inside the existing `internal/web` package. This Decision fixes the **foundation** every page state builds on: the route table, the mount-relative link strategy, the package shape, and how handlers obtain data (injected seams). The per-state rendering lives in D43 (home), D44 (ask result), D45 (subject page), D46 (orphan index); the nginx gating restructure lives in D47.

The web surface adds **no domain logic** — it is a styled human window onto already-built service capabilities: `ask` (D9), alias-aware page-by-path (D29) + read-time link projection (D12). It mutates nothing.

## Routes (in-process; nginx strips the `/srv/wiki/` mount)

A single `http.Handler` (a `*http.ServeMux` built by `web.NewHandler`) mounted at the composition root, replacing today's two bare `rt.HandleFunc("GET /{$}", …)` + `rt.Handle("GET /static/", …)` lines:

```go
// NewHandler builds the read-surface mux. service/version feed the name+version
// footer (D43); mount is the external prefix used for <base href>. The three
// data seams are injected (see "Seams" below); each page state reaches exactly one.
func NewHandler(service, version, mount string, opts ...Option) http.Handler
```

| In-process pattern | External URL | Serves |
|---|---|---|
| `GET /{$}` | `/srv/wiki/` (± `?q=`) | **Home** (no `q`) or **ask result** (non-empty `q`) — D43/D44 |
| `GET /subject/{type}/{slug}` | `/srv/wiki/subject/entity/acme-corp` | **Subject page** — D45 |
| `GET /static/` | `/srv/wiki/static/tokens.css` | Embedded Carbon assets (unchanged from D39) |

- **One route for home and ask.** `{$}` matches the exact root path `/`; a query string is not part of the path, so `GET /{$}` serves both, and the handler dispatches on `r.URL.Query().Get("q")`: empty → home, non-empty → ask. This reuses the existing exact-root session gate (D47) — ask needs no new nginx location.
- **`/subject/{type}/{slug}` is collision-proof by construction.** It is a three-segment pattern under a literal `/subject/` prefix, so it cannot shadow `/mcp`, `/health`, `/feed`, `/static/…`, or the PRM doc (none of which begin with `/subject/`), and does not rely on ServeMux specificity tie-breaks. The `{type}`/`{slug}` wildcards are rejoined to the public path `"<type>/<slug>"` (D11) and handed to the `PageFinder` seam (D45).

## Mount-relative linking — one `<base href>`

Every page emits `<base href="{{.Mount}}">` (e.g. `/srv/wiki/`) in `<head>`, and **all** intra-surface links are written **relative, no leading slash**:

- subject link → `subject/entity/acme-corp` · asset → `static/tokens.css` · "ask another question" → `.` (resolves to the base) · search form `action=""` `method="get"`.

With `<base>`, a relative link resolves against the mount root **regardless of the current page's depth**, so a link from `/srv/wiki/subject/entity/acme-corp` to `subject/event/foo` correctly yields `/srv/wiki/subject/event/foo` (a bare relative link would resolve wrong, and a root-absolute `/subject/…` would drop the `/srv/wiki/` mount nginx requires). `mount` is injected from the `wiki.Mount` constant (`internal/wiki/wiki.go`) at the composition root — never hardcoded in a template — so the surface is correct under any mount.

## Seams — injected, mirroring the MCP `With…Service` pattern

The web handler depends on three small interfaces, owned by `internal/web` and satisfied by adapter types in `cmd/wiki/main.go` (the same shape as the MCP handler's injected services). This keeps the web layer unit-testable against stubs; the real `asker`/DB are wired only at the composition root and exercised by the per-state integration ids (D44/D45).

```go
// Asker answers a question (D9). owner is the nginx-supplied X-Owner-Email.
type Asker interface {
    Ask(ctx context.Context, owner, question string) (ask.Answer, error)
}

// PageFinder resolves a public type/slug path to a rendered subject view (D29 + D12).
type PageFinder interface {
    PageByPath(ctx context.Context, path string) (SubjectView, error) // ErrNotFound on miss
}

// OrphanLister lists subjects with zero inbound mentions, for the home index (D46).
type OrphanLister interface {
    Orphans(ctx context.Context) ([]Ref, error)
}

// Ref is a web link target: the mount-relative href tail + display text.
//   Href is the path AFTER <base> — e.g. "subject/entity/acme-corp".
type Ref struct { Href, Name string }

func WithAsker(a Asker) Option
func WithPageFinder(p PageFinder) Option
func WithOrphanLister(o OrphanLister) Option
```

`SubjectView` (D45) carries the subject's title, prose body, and its outbound/inbound `Ref` slices (projected from D12's `Mentions`/`MentionedBy`, whose `Ref{Path,Name}` paths are turned into `subject/<path>` hrefs by the adapter).

## Shared layout

One base template (`layout.tmpl`) holds `<!doctype>`, `<head>` (the `<base>`, embedded `tokens.css` link, Carbon fonts), and a `{{block "main"}}` the three page templates fill. Carbon rules (D39 / `design/carbon.md`): blue `#2563EB` the only signal color, mono name+version footer, 4px grid, no JS. Assets stay embedded (D39 `//go:embed`), so the binary remains one static file.

**Rejected.**
- **A dedicated `/ask` path for the search result.** Would need its own session-gated nginx location and complicate the "ask another question" reset target. Serving ask on the root via `?q=` reuses the exact-root gate and makes reset a link to the bare base. Rejected.
- **Encoding the subject as `?p=type/slug` on a flat `/subject` path.** Keeps every page at depth-1 (so relative links work without `<base>`), but yields ugly, non-RESTful URLs and still needs query parsing; `<base href>` solves the depth problem cleanly and `/subject/{type}/{slug}` reads as the subject's identity. Rejected.
- **Root-absolute (`/subject/…`) or bare-relative links instead of `<base>`.** Root-absolute drops the `/srv/wiki/` mount (nginx strips it); bare-relative resolves against the current depth and breaks on subject pages. A single injected `<base>` is the one strategy correct at every depth and under any mount. Rejected.
- **The web package calling `wiki.Service` / `asker` concrete types directly.** Couples the UI to construction details and forces real DB/LLM into every handler test. Injected seams (the established MCP pattern) keep handlers stub-testable. Rejected.
- **A shared web mux in appkit.** Same reasoning as D39: each app's surface diverges; wiki owns its handlers/templates/assets so divergence is a local edit. Rejected.

**Verification.** All httptest against `web.NewHandler` with recording stub seams — no DB/LLM/identity (the real-dependency proofs live in D44/D45). Foundation behaviors only:

- R-WC29-XALJ — **Subject route is mounted and collision-free.** `GET /subject/entity/acme-corp` is dispatched to the subject handler (the injected `PageFinder` is reached — see R-WGXV-GDKB), while `GET /subject/onlyoneseg` (a single segment under `/subject/`) is **not** matched by `GET /subject/{type}/{slug}` (404 from the mux), and the `/subject/` route serves nothing for `/mcp`, `/health`, `/feed`, or `/static/x` — proving the new route adds no shadow over the existing surfaces.
- R-WDA6-B2C8 — **`<base href>` is the injected mount, not a literal.** A handler built with a **non-default** mount (`/srv/zzz/`) renders `<base href="/srv/zzz/">` on the home page (and the subject page), proving the base is threaded from the injected `mount`, not a hardcoded `/srv/wiki/`.
- R-WFPZ-2LTM — **Root dispatches home vs ask on `q`.** `GET /?q=who%20is%20acme` invokes the injected `Asker.Ask` exactly once with the decoded question `"who is acme"`; `GET /` (no `q`) and `GET /?q=` (empty) invoke `Asker.Ask` **zero** times — the single root route splits on a non-empty query, nothing else.
- R-WGXV-GDKB — **Subject route reconstructs the public path.** `GET /subject/entity/acme-corp` invokes the injected `PageFinder.PageByPath` exactly once with `"entity/acme-corp"` (the `{type}` and `{slug}` segments rejoined with a single `/`, D11) — proving the wildcard capture maps to the canonical path the resolver expects.
