# Decision 53 — MCP surface over `appkit/mcp`: `internal/mcp` becomes the tool table

**Problem.** wiki's `internal/mcp` is a ~1500-line hand-rolled JSON-RPC
transport — `ServeHTTP`, the `initialize` / `notifications/initialized` /
`tools/list` / `tools/call` dispatch, `writeError`/`writeResult`, the
result/error-envelope helpers — plus its own `health` and `reflection` tool
implementations, wrapping **13 domain tools** (`ingest`, `status`, `abort`,
`rerun`, `jobs`, `jobs_count`, `merge`, `merges`, `ask`, `subjects`, `claims`,
`page`, `llm_calls`). The chassis now owns the transport (appkit D8) and the two
standard tools (appkit D9), consumed here as a fixed external contract. What is
genuinely wiki's is the 13 domain tools — their descriptors, JSON schemas,
argument decoding, identity gating, not-found/validation mapping, and the
reflect-based public-result shaping.

**Decision.** `internal/mcp` is reshaped from a transport into a **declaration**
over `appkit/mcp`:

- **`const Instructions`** — a concise wiki MCP instructions string (wiki served
  none before; the chassis `initialize` always carries `instructions`). The
  pinned text:

  > wiki is a knowledge base built from ingested source text. Call ingest to
  > queue text for extraction; the pipeline distills subjects
  > (entity/event/concept) and claims and compiles a cited page per subject. Use
  > ask for a grounded, cited answer over the owner's wiki; subjects, claims, and
  > page to read the compiled knowledge by type/slug path; jobs and status to
  > track ingestion; and merge to fold a duplicate subject into another. health
  > and reflection report service status and the event graph.

- **`Tools(...) []mcp.Tool`** returns wiki's 13 domain tools, gated by which
  domain services are injected. The current `Option`/`WithXxxService` injection
  surface is **retained** (a real, tested surface) — but each enabled service
  now yields an `mcp.Tool{Name, Description, InputSchema, Handler}` instead of a
  branch in a hand-rolled dispatcher and a `tools()` append. Every tool's
  `Name`, `Description`, and `InputSchema` are the current byte-for-byte values
  (the `*Tool()` descriptor funcs and the `objectSchema`/`listSchema` helpers are
  reused verbatim, including the empty-`required` omission).
- **`NewHandler(rt *appkit.Router, opts ...Option) (http.Handler, error)`**
  assembles `appkit/mcp.New` over `Options{Service: rt.Service(), Version:
  rt.Version(), Instructions, Tools: <enabled>, Health: rt.Health(), Events:
  rt.Events(), Publishes: rt.Publishes(), Subscriptions: rt.Subscriptions()}`.
  wiki is neither producer nor consumer, so `Events` is empty and
  `Subscriptions` nil — reflection reports empty `publishes`/`subscribes`, as
  today.

- **Deleted:** the hand-rolled transport (`ServeHTTP`, `handleToolCall`,
  `dispatchTool`, `tools()`, the `request` type, `writeResult`/`writeError`/
  `idOrNull`/`toolText`/`writeJSONTextResult`), and the local `health`/
  `reflection` implementations (`handleHealthCall`, `handleReflectionCall`,
  `healthTool`, `reflectionTool`) — the chassis registers those, and declaring
  `health`/`reflection` in wiki's table fails `New`'s reserved-name check.
- **Kept:** every domain tool's handler **body** — the arg struct + `decodeArgs`,
  the `strings.TrimSpace` required-field checks, the identity gating (now reading
  the `server.Identity` handler **parameter** instead of `appkit.IdentityFrom`
  off the context), the `sql.ErrNoRows` → `notFound(...)` mapping, and the
  reflect-based `publicXxxResult`/`answerFields`/`pathField` shaping. Each
  `handleXxxCall` becomes a `Tool.Handler func(ctx, args, id) (map[string]any,
  error)`: every path that currently `writeResult(w, req.ID, X)` becomes
  `return X, nil`; every `toolError(...)` / `notFound(...)` domain outcome
  becomes `return mcp.ErrorResult(...)` / `return mcp.JSONResult(notFound(...))`
  with a **nil** Go error — so a domain failure stays an `isError` **result**
  (HTTP 200), **not** a JSON-RPC `-32602` error. Only a genuinely unexpected
  marshal failure returns a non-nil Go error.

- **Wire surface.** The 13 domain tools' names, descriptions, schemas,
  `tools/call` results, and `isError` envelopes are **byte-for-byte unchanged**.
  The transport **metadata** becomes the chassis standard — the deliberate
  de-forking D8/D9 exist for: `initialize.serverInfo.name` is now `rt.Service()`
  (`"wiki"`, was the literal `"Wiki"`), `initialize.instructions` now carries
  `Instructions` (was absent), and the two reserved tools become the chassis
  ones (health gains `owner_email`/`client_id`; reflection uses the chassis index
  shape and the standard descriptions). The existing domain-tool behavioral tests
  (D10/D16/D27/…) keep their assertions and swap only the handler constructor.

**Rejected.**
- **Keeping the local transport** — the status quo the D8 audit indicts; a
  protocol fix would still be a many-service edit.
- **Declaring health/reflection in wiki's table** — the chassis reserves the
  names (D9); their inputs (Envelope, Registry, Subscriptions) are chassis-held.
- **Preserving `initialize.serverInfo.name = "Wiki"` and the instruction-less
  header** — the transport metadata is chassis-owned once the transport is;
  forking it back is exactly the drift D8/D9 remove. The tool **surface** (what
  an agent lists and calls) is what stays byte-for-byte.
- **Dissolving `internal/mcp` into `cmd/wiki`** — the 13 tools' schemas,
  decoders, and reflect-based result shaping with their test harness are a real
  package.

**Verification** (through the composition-root seam — the `appkit/mcp` handler
built by `NewHandler` over the wired domain services).
- R-JKMR-5MV1 — `tools/list` through the assembled handler returns **exactly
  fifteen** tools: wiki's 13 domain tools (`ingest`, `status`, `abort`, `rerun`,
  `jobs`, `jobs_count`, `merge`, `merges`, `ask`, `subjects`, `claims`, `page`,
  `llm_calls`) declared by wiki **plus** `health` and `reflection` supplied by
  the chassis — proving the table/chassis partition (a table that still declares
  its own `health`/`reflection` fails `New`'s reserved-name check; a surface
  missing them fails the count).
