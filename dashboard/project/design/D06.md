# Decision 6 — Purge the stale "single hybrid page / don't split" doc rule

**Decision.** The dashboard's standing scope rule that the apex `/` is a **single
hybrid page** that must **not** be split into a separate IAM console is now
**false** — this change splits it into login, landing, and profile. The rule is
**purged from `dashboard/AGENTS.md`** and replaced, in place, with the three-page
truth. (`dashboard/CLAUDE.md` is a symlink to `AGENTS.md`; editing `AGENTS.md`
updates both.)

- **Remove** the bullet currently in `AGENTS.md` (the "Scope" section) that reads,
  in substance: *"Keep the apex `/` a single hybrid page. … Do not split it into a
  separate IAM console — that's product-scale breadth this scope rules out."*
- **Replace** it with a bullet stating the current truth: the apex serves **three
  pages** — the logged-out **login** page, the logged-in **landing/home** page
  (connect-your-agent install instructions + the clickable service list), and the
  session-gated **profile** page that holds personal-access-token and OAuth-grant
  administration (reached by clicking the owner's email). State that token/grant
  management lives on the profile page, **not** the landing.
- **Doctrine:** docs state current truth — purge what the change falsified, rewrite
  in place, no archaeology, no "previously…" footnote. Touch only the one stale
  bullet; leave the rest of the Scope section (bounded-breadth / production-depth,
  ≤3 services) intact, as those are still true.
- This is a **build-loop step**, performed in the phase that realizes this Decision
  — it is not done while authoring the design/plan docs. Code and docs evolve
  together: this phase runs after the three pages exist, so the doc never describes
  a state that isn't built yet.

**Rejected.**
- **Leaving the old rule and appending a contradicting note** — produces a doc that
  asserts both "single hybrid page" and "three pages"; the doctrine is purge +
  rewrite, not stack.
- **Editing `CLAUDE.md` directly** — it is a symlink to `AGENTS.md`; the write goes
  through `AGENTS.md` (the refusal to write the symlink is expected).
- **Rewriting the whole Scope section** — only the falsified bullet changes; the
  surrounding bounded-breadth posture is unaffected and must be preserved.

**Verification.**
- R-DB16-DOCS — `dashboard/AGENTS.md` no longer contains the "single hybrid page" /
  "do not split … IAM console" rule, and instead states the three-page truth (login
  / landing / profile, with token+grant management on the profile page). A text
  search for the stale phrasing finds nothing; a search for the three-page
  statement finds it.
