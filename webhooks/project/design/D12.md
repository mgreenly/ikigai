# Decision 12 — MCP surface over `appkit/mcp`: `internal/mcp` becomes the tool table

## Problem.

webhooks's `internal/mcp` carries the copy-pasted JSON-RPC transport
(`ServeHTTP`, the `initialize` / `tools/list` / `tools/call` dispatch, the
`jsonRPCRequest` type, the JSON-RPC error/result writers, the `idOrNull` helper,
the local `Identity` type) plus its own `health` and `reflection` tool
implementations and result-envelope helpers (`toolResultText`/`toolResultErr`/
`toolResultJSON`) — the same near-identical plumbing every service duplicated.
The chassis now owns all of it (appkit D8 transport, D9 standard tools, consumed
here as a fixed external contract). What is genuinely webhooks's is the four
owner tools — `create` / `list` / `delete` / `rotate` — their descriptors,
schemas, owner scoping, URL minting, and sentinel→envelope error mapping.

The `serverInfo` hardcodes `{"name":"Webhooks","version":"1"}` and the local
`reflection`/`health` tool descriptions are webhooks-worded; the chassis renders
the real `{service, version}` and the suite-standard health/reflection
descriptions. Adopting the chassis surface means those three chassis-owned
details take their **standard** form — this is the intended uniformity, not a
regression (no webhooks test pins them today; see below).

## Decision.

`internal/mcp` is reshaped from a transport into a **declaration** over
`appkit/mcp`:

```go
// internal/mcp — what remains.

// Instructions is webhooks's MCP initialize instructions string (the exact
// bytes the local transport returned before conversion).
const Instructions = "Owner-scoped inbound webhooks. create mints a named hook " +
    "and a show-once signing secret; list shows your hooks; rotate issues a " +
    "fresh secret; delete removes one. POST to a hook's trigger_url to fire it."

// Tools declares webhooks's four DOMAIN tools over the Service and the public
// base URL (create/list/delete/rotate — descriptors, schemas, owner scoping,
// URL minting, and error envelope unchanged in wire content). health and
// reflection are NOT declared — the chassis registers them (declaring either
// locally fails appkit/mcp.New's reserved-name check).
func Tools(svc *webhooks.Service, baseURL string) []appkitmcp.Tool

// NewHandler assembles the POST /mcp http.Handler at the composition root:
// appkit/mcp.New over Instructions + Tools(svc, baseURL) + the Router-threaded
// Service/Version/Health/Events. baseURL is derived here from the Router.
func NewHandler(svc *webhooks.Service, rt *appkit.Router) (http.Handler, error)
```

- **`baseURL` derivation moves into `NewHandler`.** It is
  `strings.TrimSuffix(rt.ResourceID(), "mcp")` — the trailing-slash mount root
  (`https://<domain>/srv/webhooks/`) that D6 specifies, from which
  `trigger_url = baseURL + "in/" + name`. The tool handlers close over a small
  `toolHandlers{ svc, baseURL }` struct, so the four verbs render `trigger_url`
  through one site exactly as today. (Locating the trim inside `NewHandler`,
  keyed off `ResourceID`, is the single source of truth for the base — see
  Rejected.)
- **Deleted:** the JSON-RPC plumbing (`jsonRPCRequest`, `ServeHTTP`, the
  `initialize`/`notifications/initialized`/`tools/list`/`tools/call` dispatch,
  `writeJSONRPCResult`/`writeJSONRPCError`/`idOrNull`), the local `Identity`
  type, the local `toolHealth`/`toolReflection`/`reflectionUnknownTypeError`
  implementations, and the local result-envelope helpers (`toolResultText`/
  `toolResultErr`/`toolResultJSON` → the chassis `appkitmcp.TextResult`/
  `ErrorResult`/`JSONResult`).
- **Kept, wire-unchanged:** the descriptor/schema helpers (`desc`/`obj`/`typ`/
  `descTyp`, now producing `appkitmcp.Tool` values), the four tools' names,
  descriptions, and input schemas; the owner scoping (identity read from
  `server.Identity` supplied to each handler by the chassis); the show-once
  secret paths; the `webhookView` projection; and the sentinel→envelope mapping
  (`errorEnvelope`: `ErrNameTaken`→`duplicate`, `ErrInvalidName`→`validation`+
  `field:"name"`, `ErrNotFound`→`not_found`, else `internal`). The four domain
  tools' wire bytes agents see are unchanged.
- **Handler signature at the tool boundary.** Each `appkitmcp.Tool.Handler` is
  `func(ctx, args json.RawMessage, id server.Identity) (map[string]any, error)`;
  the verbs read `id.OwnerEmail` from that identity (the chassis fills it from the
  nginx-injected `X-Owner-Email`), replacing the local `Identity` struct.
- **The wire surface partition:** four tools declared by webhooks (`create`,
  `list`, `delete`, `rotate`) **plus** `health` and `reflection` supplied by the
  chassis = **six** tools. Same four domain descriptions/schemas, same
  `Instructions`, same domain error envelopes.

**Composition-root wiring** (`cmd/webhooks/main.go`, inside `Handlers`):

```go
handler, err := mcp.NewHandler(svc, rt)
if err != nil {
    return err
}
rt.Handle("POST /mcp", rt.RequireIdentity(handler))
```

The pre-existing `create`/`list`/`delete`/`rotate` behavioral tests
(`tools_test.go`) keep every assertion and swap only the handler constructor:
they build a real `appkit/server` Router via `server.New(server.Options{…,
ResourceID: "https://int.ikigenba.com/srv/webhooks/mcp", Version, Service,
Events: webhooks.Events, DB: conn, Register: …})` whose `Register` hook calls
`mcp.NewHandler(svc, rt)`, then drive JSON-RPC over the returned handler's
`ServeHTTP` with the `X-Owner-Email` header set (mirroring notify's
`newHandlerWithClient`). Because `ResourceID` ends in `mcp`, the derived
`baseURL` is `https://int.ikigenba.com/srv/webhooks/` — the tests' existing
`trigger_url == baseURL + "in/" + name` assertions hold unchanged.

## Rejected.

- **Keeping the local transport.** The status quo; a protocol fix would still be a
  many-service edit.
- **Declaring `health`/`reflection` in webhooks's table.** The chassis reserves
  the names (appkit D8) precisely so services cannot fork the standard tools;
  their inputs (health reporter, event registry, subscriptions) are chassis-held
  already. Declaring either fails `New`.
- **Passing `baseURL` in from `main` as a parameter.** The pre-conversion
  composition root passed `rt.ResourceID()` **raw** to the old `NewHandler`,
  while D6 and the tool tests assume the trailing-slash mount root
  (`TrimSuffix(…, "mcp")`) — a latent mismatch that only the direct-construction
  unit tests (which supply the trimmed base themselves) papered over. Folding the
  `TrimSuffix` into `NewHandler`, keyed off `ResourceID`, makes the mount root the
  one derivation site and removes the raw/trimmed ambiguity for good.
- **Dissolving `internal/mcp` into `cmd/webhooks`.** The four tools' schemas,
  owner scoping, URL minting, and error mapping with their test harness are a real
  package, unlike the post-D11 web remnant.

## Verification.

(Through the composition-root seam — the `appkit/mcp` handler built by
`NewHandler` over a real-SQLite `Service`. D6's R-5Z8J/R-60GG/R-61OC/R-62W9/
R-6445/R-65C1 continue to pin the four tools' behaviors through the assembled
handler.)

- R-0JBF-690N — `tools/list` through the assembled handler returns **exactly six**
  tools: `create`, `list`, `delete`, `rotate` declared by webhooks **plus**
  `health` and `reflection` supplied by the chassis — proving the table/chassis
  partition. (A table that still declared its own `health`/`reflection` fails
  `New`'s reserved-name check before serving; a surface missing them fails the
  count of six.)
