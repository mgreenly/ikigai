# Decision 9 — Human landing page (`share/www` template & Carbon assets)

## Decision.

webhooks serves a minimal human **landing page** at the mount root, matching the
suite-wide v1 landing every other app serves. The page and its assets live **on
disk** under `webhooks/share/www/` and are served through the appkit chassis WWW
mechanism (`Spec.WWW`, `Router.WWW()`, chassis `GET /static/` mount — D11); there
is no service-side web package and no embedding. The landing route is wired at
the composition root — no domain code, no MCP, no token logic. The browser-session
gate lives in the nginx fragment (D7); in-process the route is **ungated**.

**On-disk site `share/www/`** (mirrors the cron/crm canonical shape):

- The chassis loads the site at serve from the resolved www root
  (`share/current/www` on the box; `./share/www` or `WEBHOOKS_WWW_PATH` in dev)
  and exposes it as `rt.WWW()` — an `appkit/web.Site` whose
  `Render(w, "landing.html", data)` renders the template and whose `Static()` the
  chassis mounts at `GET /static/`. webhooks registers **no** static handler of
  its own. The `.woff2` mime type is registered by `appkit/web`.
- `share/www/landing.html` — **byte-conformed to the cron canonical template**
  (`cron/internal/web/landing.html`): the same `<head>` (relative
  `href="static/tokens.css"`, two `<link rel="preload" as="font" type="font/woff2"
  crossorigin>` for the display and body fonts), the same CSS, the same
  `<main>` with the top-left `Home` link (`href="/"`) and the
  `eyebrow / h1 / p / dl(Service, Version, API)` structure. Only the
  **per-service data** differs from the canonical: the eyebrow, the one-line
  description, and the `<title>` — everything else is identical markup. Content:
  - eyebrow: `Inbound Webhooks`
  - description: *"Webhooks receives inbound HTTP from third parties on
    per-webhook secret-guarded endpoints and publishes each delivery as a typed
    event to the event plane."*
  - the API detail cell shows `POST /mcp` (matching crm; the per-webhook public
    ingress URL is returned by the MCP tools, not advertised on the page).
- `static/tokens.css` + `static/fonts/*.woff2` — **verbatim copies of the cron
  canonical assets** (byte-identical `tokens.css`, md5
  `63cb5c83f0db30a9646ea40ba5bb469e`; the shared woff2 set). `tokens.css` already
  carries `font-display: optional` and self-served `local`/relative `src`, so the
  FOUT is eliminated by copying the finished asset, not by re-deriving it.

**Route wiring** (`cmd/webhooks/main.go`, inside the `Spec.Handlers` hook,
**outside** `RequireIdentity` — the session gate is nginx's job — with
`Spec.WWW: true`):

```go
rt.Handle("GET /{$}", landingHandler(rt.WWW(), rt.Service(), rt.Version()))
// GET /static/ is chassis-mounted from share/www — webhooks registers no static route.

func landingHandler(site *web.Site, service, version string) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        _ = site.Render(w, "landing.html",
            struct{ Service, Version string }{service, version})
    })
}
```

The values come from the chassis (`rt.Service()` / `rt.Version()`); the page adds
no new data source. `GET /{$}` (the exact root) sits beside `POST /mcp`, `/in/`,
`/feed`, and the PRM well-known and shadows none of them; `GET /static/` is owned
by the chassis (D11).

## Rejected.

- **A shared/suite-level landing handler.** The suite decision is that each app
  owns its own page so it can later diverge; webhooks ships its own copy of the
  template and assets under `share/www` and depends on no other service's assets
  at runtime.
- **Gating the routes in-process (behind `RequireIdentity`).** The gate is coarse
  and browser-session-based; it belongs in nginx (D7). Bearer-gating in-process
  would make the page unreachable to a browser.
- **Surfacing the public `/in/<name>` ingress URL on the page.** The page sits
  behind the owner's dashboard session and the ingress URL is per-webhook (handed
  back by the MCP `create`/`list` tools); advertising a path here adds nothing and
  breaks uniformity with the canonical. The API cell matches crm (`POST /mcp`).
- **Re-authoring the markup/tokens instead of copying the canonical.** Would risk
  visual drift and re-introduce the FOUT the canonical already fixed; copying the
  byte-canonical template + assets is the whole point of the suite-wide template.

## Verification.

The page content is proven over the **repo-real `webhooks/share/www` tree**
(resolved relative to the test package in `cmd/webhooks`, loaded via
`appkit/web.Load` and the chassis static mount — the exact files that ship), no
stub. The markup/asset claims are content assertions over the on-disk template
and `tokens.css`. (D11's R-0GVM-EPJ9 / R-0I3I-SH9Y additionally pin that the
source is the shipped tree, not an embed.)

- R-TMJH-V1NP — the landing route (`GET /{$}`, rendering `landing.html` through
  `rt.WWW()`) returns `200` with `Content-Type: text/html; charset=utf-8` and a
  body containing the service name `webhooks` and the exact injected version
  string. (A render that dropped the name or version, or the wrong content-type,
  fails it.)
- R-TNRE-8TEE — a non-root path (e.g. `GET /nope`) does **not** render the
  landing: it is not matched by the exact-root `GET /{$}` binding and yields no
  landing body. (Proves the landing is bound to the exact root only and does not
  swallow other paths.)
- R-TOZA-ML53 — the chassis static mount over `share/www` serves `GET
  /static/tokens.css` (`200`, non-empty, `text/css`) and a `.woff2` font asset
  with `Content-Type: font/woff2`. (Proves the assets ship on disk and the woff2
  mime type is registered — a missing file or unregistered mime fails it.)
- R-TQ77-0CVS — the shipped `share/www/landing.html` carries the cron-canonical
  structure: a relative `href="static/tokens.css"` stylesheet link, the top-left
  `Home` link to `/`, and the `eyebrow / <h1>{{.Service}}</h1> / description /
  dl(Service, Version, API `POST /mcp`)` markup with the webhooks-specific eyebrow
  and description. (A page that hard-coded an absolute asset path, dropped the
  Home link, or diverged from the canonical structure fails it.)
- R-TRF3-E4MH — the FOUT is eliminated in the shipped assets: `share/www/static/
  tokens.css` declares `font-display: optional` and self-served
  (`local(...)`/relative `url(...)`) `src` for its faces, and
  `share/www/landing.html`'s `<head>` preloads the display and body fonts with
  `<link rel="preload" as="font" type="font/woff2" crossorigin>`. (An asset set
  that shipped `font-display: swap`, remote font `src`, or dropped the preloads
  fails it.)
