# Decision 13 — Delete the `internal/db` shim, normalize the composition root, and true up the doctrine

## Decision.

Three cleanups that land the reference chassis shape whole, mirroring crm's and
notify's final structural pass. All are **behavior-preserving**: no wire surface,
no schema, no route, and no test *assertion* changes — only the shape of the code
that produces them.

**1. `internal/db` shrinks to its legitimate content.** The `Open` and `Migrate`
wrapper functions in `internal/db/db.go` are deleted — appkit owns DB opening and
the migration run (`Spec.Migrations: db.FS`, applied by appkit's forward-only
runner + downgrade guard). The blank `_ "modernc.org/sqlite"` driver import goes
with them (appkit/db registers the driver). What **remains** in the package is
what genuinely must live app-side:

- the embedded migration set (`//go:embed migrations/*.sql` → the exported `FS`;
  `go:embed` cannot cross modules);
- the webhooks **domain store** (`store.go` — `Store`, `NewStore`, `Insert`,
  `GetByName`, `ListByOwner`, `Delete`, `UpdateSecret`, `TouchLastTriggered`,
  `Webhook`); this is domain code, not a chassis shim, and stays;
- the byte-equality guard (`migrations_outbox_test.go`) that `003_outbox.sql`
  matches `outbox.SchemaSQL`, and the store tests (`store_test.go`).

The test harnesses that called the local wrappers call **appkit's db package
directly**, exactly as crm's harnesses do — no test *assertion* changes, only
harness plumbing:

```go
import appkitdb "appkit/db"

conn, _ := appkitdb.Open(path)
migs, _ := appkitdb.LoadMigrations(db.FS, "migrations")   // db.FS from webhooks/internal/db
_ = appkitdb.Migrate(ctx, conn, migs)
```

The call sites to rewire are every current `db.Open` / `db.Migrate` caller:
`internal/db/store_test.go` (in-package — uses the local `FS`),
`internal/mcp/tools_test.go`, `internal/webhooks/events_test.go`,
`internal/webhooks/secret_test.go`, `internal/webhooks/ingress_test.go`, and
`internal/e2e/e2e_test.go`. The `internal/db/db.go` package comment is trued up
to describe the embed set + domain store + outbox guard (dropping the "thin
migrate helper" and "Open" language).

**2. The composition root is normalized to one fully-formed inline `Spec`.**
Today `cmd/webhooks/main.go` declares a base `spec := webhooksSpec()` and then
**mutates** `spec.Handlers` and `spec.Producer` after construction, over a
`var svc *webhooks.Service` declared in `main`. The reference shape (crm, notify)
is a single fully-formed `appkit.Spec` with no post-construction field
assignment. Fold the two hooks into `webhooksSpec()`'s inline literal, moving the
shared `var svc *webhooks.Service` inside `webhooksSpec()` where both closures
capture it:

```go
func main() { appkit.Main(webhooksSpec()) }

func webhooksSpec() appkit.Spec {
    var svc *webhooks.Service
    return appkit.Spec{
        App:   "webhooks",
        Mount: "/srv/webhooks/",
        Port:  registry.MustPort("webhooks"),   // D10
        MCP:   true,
        WWW:   true,                             // D11
        Feed:  "/feed",
        Migrations: db.FS,
        Events:     webhooks.Events,
        ManifestExtras: []appkit.ManifestKV{
            {Key: "OUTBOX_RETENTION_DAYS", Value: "7"},
            {Key: "OUTBOX_RETENTION_MAX_ROWS", Value: "1000000"},
        },
        Handlers: func(rt *appkit.Router) error {
            conn := rt.DB()
            if conn == nil {
                return fmt.Errorf("webhooks: no DB handle on router")
            }
            svc = webhooks.NewService(conn, webhooks.RealClock{})
            handler, err := mcp.NewHandler(svc, rt)       // D12
            if err != nil {
                return err
            }
            rt.Handle("POST /mcp", rt.RequireIdentity(handler))
            rt.Handle("/in/", webhooks.NewIngressHandler(svc, rt.Logger())) // bare — self-guarded
            rt.Handle("GET /{$}", landingHandler(rt.WWW(), rt.Service(), rt.Version())) // D11
            return nil
        },
        Producer: func(ob *outbox.Outbox) error {
            if svc == nil {
                return fmt.Errorf("webhooks: Producer called before Handlers built the Service")
            }
            svc.Outbox = ob
            return nil
        },
    }
}
```

The `Producer` hook is a real chassis seam (webhooks is a producer; appkit calls
it after `Handlers` so the outbox appends on the domain write's transaction) and
stays — it is not post-construction mutation of the Spec, it is a Spec **field**.
The `main.go` package doc comment is trued up: drop any claim of a local MCP
JSON-RPC transport and of embedded web assets; the surface is now the
`appkit/mcp` tool table (D12) and the `share/www` web surface (D11).

**3. Doctrine truth-up.** webhooks has **no** service-level `AGENTS.md` /
`CLAUDE.md` (unlike notify), so there is no separate doctrine file to purge and
rewrite. The "docs state the converted truth" doctrine is discharged **in the
design tree**: D1, D6, and D9 are rewritten in place (by their respective phases)
to describe the converted shape — the `registry` port, the `appkit/mcp` tool
table, the `share/www` web surface, and the `internal/db` = embed-set + domain
store — with every falsified claim (the hardcoded `3006`, the local JSON-RPC
transport and local `NewHandler` signature, the embedded landing/`internal/web`,
the "test Open helper") removed, no archaeology.

## Rejected.

- **Keeping the `Open`/`Migrate` wrappers for import brevity.** A per-service shim
  that can drift, for two saved lines per call site; crm's and notify's final
  passes already made this call. The domain store is different — it is app-side
  behavior, not chassis duplication — and stays.
- **Deleting `internal/db` entirely.** The migration embed must live in a webhooks
  package, the outbox byte-equality guard is genuinely app-side, and the domain
  `Store` is webhooks's persistence layer. The package shrinks to its legitimate
  content instead.
- **Keeping the `spec := …; spec.Handlers = …` post-construction form.** It exists
  only because the shared `svc` predates a settled composition root; a `var svc`
  inside `webhooksSpec()` captured by both hooks gives the same sharing with the
  reference inline shape. No behavior depends on the mutation form.
- **Authoring a new `webhooks/AGENTS.md` to mirror notify's.** The conversion
  falsifies no such file (there is none); creating one is new scope, not a
  truth-up. The design tree already carries the current truth.

## Verification.

Structural — **no `R-XXXX-XXXX` ids**. The behaviors are already pinned by the
existing domain / ingress / event / MCP suites (D2–D6) and the D10–D12 ids; this
Decision changes only code shape. Realized by a plan phase whose done bar is
mechanical:

- `grep -n "func Open\|func Migrate" webhooks/internal/db/db.go` returns no
  matches, while `grep -n "go:embed migrations" webhooks/internal/db/db.go` still
  matches, `webhooks/internal/db/store.go` still declares `func NewStore`, and
  `webhooks/internal/db/migrations_outbox_test.go` still exists;
- `grep -rn "spec.Handlers =\|spec.Producer =" webhooks/cmd/webhooks/main.go`
  returns no matches (the Spec is one inline literal), while
  `grep -n "Producer:" webhooks/cmd/webhooks/main.go` matches (the producer seam
  stays as a Spec field);
- the suite stays green (design Conventions commands) with **no assertion
  changes** — only harness plumbing and composition shape moved.
