# Decision 6 — MCP tool surface (the four owner tools)

## Decision.

The owner-facing management surface is JSON-RPC MCP over the gated `POST /mcp`
route (wrapped in `rt.RequireIdentity` — D1). The **transport** is the shared
`appkit/mcp` chassis (D12): `internal/mcp` declares only webhooks's four domain
tools and assembles the chassis handler; it owns no JSON-RPC plumbing and no
`health`/`reflection` implementation (the chassis registers those). Tool names
are bare verbs (`toolPrefix == ""` across the suite; the `ikigenba_webhooks_*`
namespacing is client-side).

```go
// internal/mcp — a declaration over appkit/mcp (D12).
const Instructions = /* webhooks's MCP instructions string */

// Tools declares webhooks's four DOMAIN tools over the Service and the public
// mount-root base URL; each Tool.Handler is
// func(ctx, args json.RawMessage, id server.Identity) (map[string]any, error).
func Tools(svc *webhooks.Service, baseURL string) []appkitmcp.Tool

// NewHandler assembles the POST /mcp handler: appkit/mcp.New over
// Instructions + Tools(svc, baseURL) + the Router-threaded Service/Version/
// Health/Events. baseURL is derived here as TrimSuffix(rt.ResourceID(), "mcp")
// == "https://<domain>/srv/webhooks/" (trailing slash).
func NewHandler(svc *webhooks.Service, rt *appkit.Router) (http.Handler, error)
```

Four domain tools, each scoped to the authenticated `OwnerEmail` (from
`server.Identity` the chassis fills from `X-Owner-Email`, guaranteed present by
`RequireIdentity`), plus the chassis-registered `health` and `reflection` tools:

| tool | input | result | errors |
|---|---|---|---|
| `create` | `{name?: string}` | `{name, trigger_url, secret}` — secret show-once | `duplicate`, `validation` (`field:"name"`) |
| `list`   | `{}` | `[{name, created_at, last_triggered_at}]` — no secret/hash | — |
| `delete` | `{name: string}` | `{deleted: true}` | `not_found` |
| `rotate` | `{name: string}` | `{name, trigger_url, secret}` — new secret show-once | `not_found` |

- **Trigger URL.** `trigger_url = baseURL + "in/" + name` →
  `https://<domain>/srv/webhooks/in/<name>`. `baseURL` carries a trailing slash;
  it is `TrimSuffix(rt.ResourceID(), "mcp")` derived once inside `NewHandler`
  (the single derivation site — D12), threaded into `Tools` and closed over by
  each verb's `webhookView`/`triggerURL`.
- **Owner scoping.** Every call threads `id.OwnerEmail` into the `Service`/`Store`
  methods. `delete`/`rotate` of a name the caller does not own return `not_found`,
  indistinguishable from a truly-missing name, and act on nothing.
- **Error envelope.** Domain sentinels (D3) map to the suite's closed-vocabulary
  envelope (crm's `errorEnvelope` pattern): `ErrNameTaken` → `duplicate`,
  `ErrInvalidName` → `validation` + `field:"name"`, `ErrNotFound` → `not_found`,
  anything else → `internal` (opaque message).
- **Result/error shapes** reuse the chassis helpers (`appkitmcp.JSONResult`/
  `TextResult`/`ErrorResult`, D12): success →
  `{content:[{type:"text", text:<json>}]}`; domain error →
  `{isError:true, content:[…]}` carrying the envelope JSON.

## Rejected.

- **Reading owner from tool arguments.** Spoofable; the authoritative owner is the
  authenticated identity nginx established. Tool args never carry identity.
- **Returning the secret (or hash) in `list`, or a `get`-secret tool.** Breaks the
  show-once promise. There is deliberately no way to re-read a secret; the recovery
  path is `rotate`.
- **A separate `get` tool.** `list` covers v1; nothing requires single-fetch.
  Reject speculative surface.
- **Distinguishing not-owned from not-found in `delete`/`rotate`.** A distinct
  "exists but not yours" response would leak another owner's name existence to an
  authenticated prober; one uniform `not_found` preserves the same
  enumeration-resistance posture as the public endpoint.

## Verification.

(All via the real `Handler` driven over `httptest` with JSON-RPC `tools/call`,
backed by a real-SQLite `Service` and the authenticated identity supplied as
`X-Owner-Email` — the owner scoping, the persistence, and the URL minting are the
real substrate.)

- R-5Z8J-Y0YP — `create` with no `name` returns a result whose `trigger_url`
  equals `baseURL + "in/" + <generated name>` and whose `secret` begins `ms_wh_`,
  and the webhook is persisted owned by the authenticated caller. (Real handler+DB
  — pins the minted URL form and the one-time secret, and that ownership comes from
  identity.)
- R-60GG-BSPE — `list` returns exactly the authenticated caller's webhooks — each
  with `name`, `created_at`, `last_triggered_at` and no `secret`/`secret_hash`
  field — and excludes a different caller's webhooks. (Real handler+DB — owner
  scoping at the tool layer plus no-secret-in-list.)
- R-61OC-PKG3 — `delete{name}` issued by the owning caller removes the webhook: it
  no longer appears in that owner's `list`. (Real handler+DB.)
- R-62W9-3C6S — `rotate{name}` returns a new one-time secret with the same
  `trigger_url` (name unchanged), and the new secret differs from the secret
  originally returned by `create`. (Real handler+DB — rotation preserves the URL.)
- R-6445-H3XH — `delete{name}` and `rotate{name}` issued by a caller who does NOT
  own `name` each return a `not_found` error envelope and mutate nothing: the
  webhook still exists and its original secret still verifies. (Real handler+DB —
  tool-layer owner scoping is falsifiable against a two-owner setup.)
- R-65C1-UVO6 — A `create` for a name already in use returns an error envelope with
  `code == "duplicate"`; a `create` with an invalid name returns an error envelope
  with `code == "validation"` and `field == "name"`. (Real handler+DB — the
  sentinel→envelope mapping.)
