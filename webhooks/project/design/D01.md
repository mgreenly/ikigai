# Decision 1 — Service skeleton, seams & composition root

## Decision.

The webhooks service is a new appkit-based deployable Go binary. It follows the
suite chassis pattern exactly: the entry point collapses to one
`appkit.Main(appkit.Spec{…})` call, the domain is a concrete `Service` over the
appkit-owned `*sql.DB`, and the two HTTP surfaces (gated MCP, ungated ingress)
mount through the `Handlers` hook.

**Package layout** under `webhooks/`:

```
cmd/webhooks/main.go            # composition root: appkit.Main(Spec{…})
internal/db/db.go               # //go:embed migrations/*.sql → exported FS + test Open helper
internal/db/migrations/*.sql    # ordered immutable SQL (D2)
internal/webhooks/service.go    # domain Service over *sql.DB + injected Clock + Outbox
internal/webhooks/store.go      # SQL CRUD (concrete, over *sql.DB)
internal/webhooks/ids.go        # name/secret generation + verifier (D3)
internal/webhooks/events.go     # event registry + payload builder (D5)
internal/webhooks/ingress.go    # the public /in/ handler (D4)
internal/mcp/mcp.go, tools.go   # MCP surface (D6)
etc/{manifest.env,deploy.env,nginx.conf}, VERSION, go.mod, Makefile
```

**Composition root** (`cmd/webhooks/main.go`, illustrative — not full code):

```go
func main() {
    var svc *webhooks.Service
    appkit.Main(appkit.Spec{
        App: "webhooks", Mount: "/srv/webhooks/", Port: 3006,
        MCP: true, Feed: "/feed",
        Migrations: db.FS, Events: webhooks.Events,
        ManifestExtras: []appkit.ManifestKV{
            {Key: "OUTBOX_RETENTION_DAYS", Value: "7"},
            {Key: "OUTBOX_RETENTION_MAX_ROWS", Value: "1000000"},
        },
        Handlers: func(rt *appkit.Router) error {
            svc = webhooks.NewService(rt.DB(), webhooks.RealClock{})
            rt.Handle("POST /mcp", rt.RequireIdentity(mcp.NewHandler(svc, rt)))
            rt.Handle("/in/", webhooks.NewIngressHandler(svc, rt.Logger())) // NO RequireIdentity
            return nil
        },
        Producer: func(ob *outbox.Outbox) error { svc.Outbox = ob; return nil },
    })
}
```

The chassis owns the fixed verbs (`serve|version|manifest|migrate|schema`), env config, the migration runner + downgrade guard, the loopback
bind, the PRM/`/health` routes, and (because `Feed != ""`) the outbox
construction + retention + `/feed` mount. The service owns only its domain and
its two routes.

**Domain seam** — concrete, over `*sql.DB`, with an injectable clock and the
outbox injected after `Handlers` builds the Service:

```go
type Clock interface{ Now() time.Time }     // seam: deterministic time in tests
type RealClock struct{}
func (RealClock) Now() time.Time { return time.Now().UTC() }

type Service struct {
    db     *sql.DB
    clock  Clock
    Outbox *outbox.Outbox                    // injected by the Producer hook
}
func NewService(db *sql.DB, clock Clock) *Service
```

**Seams that make the service testable:**

1. `rt.DB()` hands the domain the same single-writer `*sql.DB` appkit opened and
   migrated, so tests run the domain over a real temp-file SQLite (`db.Open`).
2. `Clock` makes `created_at` / `last_triggered_at` deterministic under test.
3. The two routes are plain `http.Handler`s exercised via `httptest`: the MCP
   handler wrapped in `rt.RequireIdentity` (gated), the ingress handler mounted
   bare (it self-guards — see D4).

This Decision is largely **structural**; it carries only the two chassis-level
behaviors below that are specific to this service (manifest fidelity and a clean
boot). The HTTP behaviors live in D4 (ingress), D5 (events), and D6 (MCP).

## Rejected.

- **A `Store` interface for mocking.** The suite tests domains against real
  SQLite (crm, sites), and this service's load-bearing claims (the unique-name
  constraint in D2, the durable transactional append in D5) can only be falsified
  by a real DB — a mock store would accept whatever it is handed and prove
  nothing. A concrete store over a real temp-file DB is both the convention and
  the correct substrate.
- **Folding ingress into the MCP package.** The ingress route is unauthenticated
  and structurally distinct from the gated MCP surface; a separate `ingress.go`
  keeps the trust boundary legible and prevents accidental reuse of identity-gated
  helpers on the public path.
- **A second binary for ingress.** One binary, two routes, is the suite norm
  (sites runs four tiers in one binary); a split would duplicate the chassis and
  the DB handle for no benefit.

## Verification.

- R-IC14-FKIK — Running the built `webhooks` binary's `manifest` verb emits output
  that **byte-equals** the committed `etc/manifest.env`, and that file declares
  `APP=webhooks`, `MOUNT=/srv/webhooks/`, `PORT=3006`, `MCP=true`, and
  `FEED=/feed`. (Exercise the real binary's `manifest` verb and diff against the
  committed file — not a mock; a wrong port/flag or drift fails it.)
- R-ID90-TC99 — Running `serve` against a clean, empty temp-file SQLite applies
  all migrations and the loopback server answers `/health` with a 200 health
  envelope reporting service `webhooks`. (Exercise against a real temp-file DB and
  a real loopback/`httptest` server — proves the chassis boots and migrates this
  service end to end, not merely that config was set.)
