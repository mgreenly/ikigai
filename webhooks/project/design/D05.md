# Decision 5 — Event production (durable-before-ack)

## Decision.

The webhooks service is an event-plane producer. A successful trigger publishes a
**single** registered event kind, `received`, **addressed by the hook name as its
routing subject** — the canonical key is `webhooks:received/<hook name>` (the
kind/subject choice, the family registry, and the outbox schema conversion are
D15's; this Decision owns the durable emission path). The payload struct, the
kind constant, and the `Events` registry live in `internal/webhooks/events.go`.

```go
const kindReceived = "received" // key: webhooks:received/<hook name> (D15)

type webhookReceivedPayload struct {
    Name        string `json:"name"`         // which webhook fired (detail record; the subject routes)
    Owner       string `json:"owner"`        // stored owner_email — the for-whom
    ReceivedAt  string `json:"received_at"`  // RFC3339Nano UTC (from the injected clock)
    ContentType string `json:"content_type"` // request Content-Type, "" if absent
    Body        string `json:"body"`         // base64 (std encoding) of the raw request bytes
}

var Events = outbox.Registry{{
    Kind:        kindReceived,
    Subject:     "/<hook name>",
    Description: "An authenticated external call to a webhook was received and durably recorded.",
    Sample:      webhookReceivedPayload{ /* filled exemplar */ },
}}
```

`Service.Record` performs the durable append and the `last_triggered_at` touch in
**one transaction**, then rings (illustrative — not full code):

```go
func (s *Service) Record(ctx context.Context, wh Webhook, contentType string, body []byte) error {
    now := s.clock.Now().UTC()
    raw, _ := json.Marshal(webhookReceivedPayload{
        Name:        wh.Name,
        Owner:       wh.OwnerEmail,
        ReceivedAt:  now.Format(time.RFC3339Nano),
        ContentType: contentType,
        Body:        base64.StdEncoding.EncodeToString(body),
    })
    tx, err := s.db.BeginTx(ctx, nil)
    // append (lib mints ULID + emit time), then touch last_triggered_at, then commit:
    //   s.Outbox.Append(tx, outbox.Event{Kind: kindReceived, Subject: "/" + wh.Name, Payload: raw})
    //   s.Store.TouchLastTriggered(tx, wh.Name, now)
    //   tx.Commit()  (rollback on any error)
    s.Outbox.Ring() // best-effort wake AFTER commit; the durable row is the source of truth
    return nil
}
```

The ingress handler (D4) returns `202` **only after `Record` returns nil**, so a
`202` means the outbox row is committed — that is the durable-before-ack
guarantee. `Ring()` after commit is best-effort: a missed ring loses nothing
because the keepalive re-poll and the durable row cover it.

**Body encoding.** The body is base64-encoded into a string field alongside its
`content_type`. Webhook bodies are arbitrary bytes (JSON, form-encoded, XML,
binary, possibly non-UTF8); base64 round-trips anything and keeps the event
envelope always-valid JSON, at the cost of ~33% size inflation and lower feed
readability.

## Rejected.

- **Polymorphic body (raw JSON when it parses, else a string).** Makes the `body`
  field string-or-object, forcing every consumer to branch on the shape. A uniform
  base64 string is simpler and total.
- **Multiple event kinds (one per webhook name).** Per-name kinds would be a
  dynamic set that defeats the outbox's Append-time family validation; the routing
  revision puts the per-name precision in the **subject** (D15), where it belongs.
- **Storing the body outside the event (blob table referenced by the event).**
  Unnecessary for v1 and breaks the self-contained event the feed promises.
- **Returning `202` before commit (fire-and-forget).** Would violate
  durable-before-ack — an accepted call could vanish on restart.

## Verification.

(All against the real `eventplane/outbox` over a real temp-file SQLite, driven
through `Service.Record` — the append, the registry validation, and the commit
are the real substrate. Addressing conformance itself — kind/subject values read
back by SQL, the family reflection, the wire framing — is pinned by D15's ids;
these pin the durable emission behaviors.)

- R-GTUZ-AIGW — After one `Record`, exactly one outbox row exists for the
  `received` kind whose payload carries the webhook's `name`, the stored
  `owner`, the `content_type`, and a `body` that base64-decodes to the
  **exact** submitted bytes — verified with a non-UTF8/binary body recovered
  byte-for-byte. (Real DB — pins lossless round-trip, not merely "a body field
  exists".)
- R-GV2V-OA7L — The event payload's `owner` equals the webhook's stored
  `owner_email` even when the call that triggered `Record` carried a different or
  absent owner value. (Real DB — the for-whom guarantee; a wrong implementation
  that echoed caller input would fail.)
- R-GWAS-21YA — After `Record` returns, the committed outbox row is readable
  through a **freshly opened** database connection to the same file. (Real SQLite
  file reopened in the test — proves durability survives connection/process
  restart, i.e. durable-before-ack, not just a value held on the live handle.)
- R-GXIO-FTOZ — A single `Record` produces exactly one outbox row, and that
  webhook's `last_triggered_at` equals the event payload's `received_at` (append
  and touch committed together in one transaction). (Real DB — atomic, no
  duplicate, timestamps tied.)
- R-GYQK-TLFO — The `Events` registry declares exactly the one `received`
  family, and an `Append` of any kind not in the registry returns an error (no
  row written). (Real DB — guards the single-kind contract via the library's
  Append-time validation.)
