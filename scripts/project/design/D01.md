# Decision 1 ŌĆö The landing handler and its v1 content (service name + version)

> **Substrate note (D12).** The landing page originally lived in a
> `scripts/internal/web/` package that embedded its template via `//go:embed` and
> exposed a `web.LandingHandler(service, version)` constructor. **D12 de-embedded
> it:** the template now ships on disk at `scripts/share/www/landing.html` and is
> rendered through the chassis (`Spec.WWW` / `rt.WWW().Render`). This Decision is
> rewritten to that substrate; its *content contract* (name + version, one centered
> Carbon card, `text/html`) and its `R-LAND-*` ids are unchanged ŌĆö they are proven
> against the shipped `share/www` tree.

**Decision.** scripts owns its landing page. Its template lives on disk at
`scripts/share/www/landing.html` (shipped in the release `share` tier) and renders
the v1 landing page: a Carbon-styled, centered card showing the **service name**
(display type) and the **running version** (mono label) and nothing else. There is
**no shared landing handler** across apps; each app owns its page. The composition
root mounts `GET /{$}` (D2) with a few-line handler over
`rt.WWW().Render(w, "landing.html", struct{ Service, Version string }{ŌĆ”})`, supplying
the two values from the chassis (`rt.Service()` ŌåÆ `"scripts"`, `rt.Version()` ŌåÆ the
build version) ŌĆö so the page is fully determined by those two inputs and the shipped
template, needing no DB, clock, or network at test time.

**Rendering.** The page is an on-disk template rendered per request through the
chassis site loader (`appkit/web`); the observable contract is the output:

- The HTML is loaded from `scripts/share/www/landing.html` by the chassis
  (`Spec.WWW: true` ŌåÆ `rt.WWW()`), not embedded in the binary. The service name and
  version are the only dynamic substitutions; everything else is static markup.
- Rendering goes through `appkit/web`'s `html/template`-backed `Site.Render` (the
  two values are trusted, but `html/template` is the correct HTML default).
- The response sets `Content-Type: text/html; charset=utf-8` explicitly and
  returns `200 OK` for the page.
- The markup links the app's **own** `tokens.css` from `share/www/static` and
  applies the Carbon type scale (Space Grotesk display for the name, IBM Plex Mono
  for the version label) and neutral palette. The asset wiring is D3; this Decision
  owns that the page is *rendered with name + version* and is a single centered card.

**Why name + version as template data, not a chassis dependency.** The page is a
pure function of the two strings the composition root pulls from `rt`
(`rt.Service()`/`rt.Version()`), so its tests load the repo-real `share/www` tree
via `appkit/web` and drive it with fixed values ŌĆö no chassis internals, no DB.

**Rejected.**
- **A shared cross-app landing handler** (e.g. in appkit) ŌĆö the brief's locked
  design: each app's page will diverge to serve that app's purpose, so each app
  owns its page. A shared handler would have to be un-shared at the first
  divergence. v1's uniformity is a content coincidence, not a shared component.
- **Rendering anything beyond name + version** in v1 (owner email, script counts,
  recent-run status, links) ŌĆö out of product scope; v1 is exactly name + version.
- **Pulling the values from env or a new config knob** ŌĆö the chassis already
  exposes the authoritative service name and build version; a second source could
  drift from what `version`/`/health` report.
- **Reading the identity header (`X-Owner-Email`) to personalize the page** ŌĆö the
  gate is coarse (any logged-in user); v1 shows no per-user content, so the
  handler ignores identity entirely.

**Verification** (over an `appkit/web` Site loaded from the repo-real
`scripts/share/www` tree, driven with httptest from `cmd/scripts`).
- R-LAND-7Q3D ŌĆö the landing route rendering `scripts/share/www/landing.html` for a
  `GET /` returns HTTP `200`.
- R-LAND-9R5F ŌĆö the rendered body contains the service name passed in (the page
  shows `"scripts"`).
- R-LAND-1S7G ŌĆö the rendered body contains the version string passed in (e.g. a
  test version like `"9.9.9-test"` appears verbatim), proving the page reflects
  the running version rather than a hardcoded constant.
- R-LAND-3T9H ŌĆö the landing response sets `Content-Type: text/html; charset=utf-8`.
