# Decision 14 — Delete the `internal/db` shim and true up the doctrine

**Decision.** Two cleanups that land the reference shape whole, mirroring crm's and
notify's shim deletion:

1. **`internal/db` shrinks to its legitimate content.** The `Open` and `Migrate`
   wrapper functions in `scripts/internal/db/db.go` are deleted — appkit owns DB
   opening and the migration run (`Spec.Migrations: db.FS`, applied by the chassis;
   the domain never opens the DB itself). What remains is what genuinely must live
   in a scripts package: the embedded migration set (`FS` over
   `//go:embed migrations/*.sql`; `go:embed` cannot cross modules) and the
   `migrations_load_test.go` guard (asserting the real set loads through
   `appkit/db.LoadMigrations`). scripts keeps **no** second DB handle and no domain
   store in `internal/db` (the store is `internal/script`), so nothing else stays.
   The four test harnesses that called the wrappers —
   `internal/runner/runner_test.go`, `internal/script/store_test.go`, and the two in
   `internal/mcp/tools_test.go` (the latter reshaped by D13) — call appkit's db
   package directly (`appkitdb.Open` + `appkitdb.LoadMigrations` +
   `appkitdb.Migrate`, exactly as crm/notify test harnesses do). **No test
   *assertion* changes — harness plumbing only.**
2. **scripts' doctrine states the converted truth.** scripts has **no root
   `AGENTS.md`/`CLAUDE.md`** (D5 records this), so — unlike notify — there is no
   root doctrine file to purge. The doctrine that this conversion falsifies lives in
   the **design spine and the landing Decisions**, and it is corrected **in place**
   by the author as part of this work (the D1/D2/D3/D6/D8 substrate edits and the
   `design.md` Conventions/Layout/Scope/Testing edits; INDEX regenerated). This
   Decision's *build-loop* half is therefore a no-op on service prose: it exists to
   keep the phase sequence honest (the doc truth-up is authored, not code) and to
   pin the shim deletion. If scripts ever grows a root `AGENTS.md`/`CLAUDE.md`, the
   same rule applies — state the `Spec.Consumers` declaration, the `share/www` web
   surface, the `appkit/mcp` tool table, and chassis-owned db, editing `AGENTS.md`
   (the `CLAUDE.md` symlink refusal is expected).

**Rejected.**
- **Keeping the wrappers for import brevity** — a per-service shim that can drift,
  for two saved lines per call site; crm/notify already made this call.
- **Deleting `internal/db` entirely** — the migration embed must live in a scripts
  package and the load guard is genuinely app-side; the package shrinks to its
  legitimate content instead.
- **Deferring the design-doc truth-up to "later"** — the spine and landing Decisions
  would assert an embedded/hand-rolled shape the same tree no longer has; docs move
  with the code, so the author edits them in this change set.

**Verification.** Structural — no `R-XXXX-XXXX` ids (shim deletion + doc truth; the
behaviors are already pinned by the retained domain/MCP/consumer suites and the
D11–D13 ids). Realized by a plan phase whose done bar is:

- `grep -n "func Open\|func Migrate" scripts/internal/db/db.go` returns no matches,
  while `grep -n "go:embed migrations" scripts/internal/db/db.go` still matches and
  `scripts/internal/db/migrations_load_test.go` still exists.
- `grep -rn "db\.Open\|db\.Migrate" scripts --include=*.go` returns no matches
  (all harnesses call `appkitdb` directly).
- The suite stays green (design Conventions commands) with no domain assertion
  changes.
