# Decision 5 — Docs state current truth: state the landing-page surface in scripts's doctrine

**Decision.** scripts's service doctrine — its "what this app is" description in
`scripts/project/notes/README.md` (and `scripts/project/notes/ARCHITECTURE.md`) —
currently describes scripts as a loopback service that serves **only machine
surfaces** (the RFC 9728 PRM bootstrap, `/health`, the bearer-gated `/mcp`, and
the loopback `/feed`), framed as "non-interactive background work" whose only
front door is MCP. That picture is now **incomplete**: scripts also serves a
human web landing page (D1–D4). Per the suite doctrine — *docs state current
truth; purge what the change falsified, rewrite in place, no archaeology, no
footnotes* — this Decision **rewrites the doctrine in place** to state the
landing-page truth.

> **Note (no `AGENTS.md` to purge).** Unlike crm/ledger/notify/dropbox/wiki,
> scripts has **no root `AGENTS.md`/`CLAUDE.md`** and carries **no literal
> "no UI" line** today — so there is no false claim to delete. The doctrine
> change here is therefore purely **additive**: state that scripts now also
> serves a human landing page. If scripts later grows a root `AGENTS.md`/`CLAUDE.md`
> (a symlinked pair), the same rule applies — purge any "no UI" / "machine-only"
> overreach there and state the landing-page truth, editing `AGENTS.md` (the
> refusal to write through the `CLAUDE.md` symlink is expected).

**What changes (doc edits only — no code):**

- In `scripts/project/notes/README.md` (the "What scripts is" framing) and, where
  it lists the served surfaces, `scripts/project/notes/ARCHITECTURE.md`: add the
  current truth, in scripts's own voice — scripts is a loopback-only service that
  serves an **MCP surface for agents** (bearer-gated) **and a human web landing
  page** (dashboard-session-cookie-gated) under `/srv/scripts/`, alongside its
  event-plane producer/consumer feeds; it still runs **no token logic** — nginx
  remains the sole trust boundary for both doors.
- Purge any phrasing the change actually **falsifies** (e.g. an enumeration of
  served surfaces that implies "machine-only / no human page"). Leave true
  framing intact: scripts runs **non-interactive background work** and you still
  *delegate* a script over MCP rather than chat with a run — the static
  name+version landing page does not change that, so that line stays.
- Do **not** add a changelog, a "previously…", or a footnote. State the present
  as if it were always so.

**Why this is its own phase, performed by the build loop — not by the doc
author.** The doc author (this design+plan workspace) deliberately does **not**
edit the doctrine now: code and docs must evolve **together**, so the doctrine
update is sequenced as a plan phase that the build loop performs **after** the
landing page exists (D1–D4 shipped). Doing it earlier would make the docs claim a
UI the code does not yet serve; doing it never would leave an out-of-date surface
list on `main`.

**Scope guard.** Only the served-surface picture is brought current. scripts's
other doctrine — the deterministic `python3`-exec runner, the script/run data
model, the event-plane producer (`scripts.succeeded`/`scripts.failed`) and
multi-upstream consumer, the header-trust model, "no token logic," the loopback
bind, the deploy/manifest facts — is **unchanged and still true**; this Decision
does not touch it.

**Verification.** This is a **documentation** Decision with no Go behavior, so it
mints no `R-XXXX-XXXX` ids; it is realized by a **structural** plan phase whose
done bar is a content check rather than an id-tagged test:

- scripts's doctrine (`scripts/project/notes/README.md`, and
  `ARCHITECTURE.md` where it lists surfaces) states the landing-page truth: it
  mentions that scripts serves a human web **landing page** under `/srv/scripts/`
  alongside its MCP surface.
- No served-surface enumeration in those docs still implies scripts serves no
  human page (`grep -i "no UI" scripts/project/notes/*.md` finds nothing, and the
  surface list no longer reads as machine-only).
- The suite stays green (`bin/check-migrations scripts` and
  `cd scripts && go build ./...` unaffected — this is a docs-only change).
