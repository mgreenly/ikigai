# Decision 8 — Self-serve the landing page's fonts and eliminate the FOUT

> **Substrate note (D12).** This Decision's mechanism — the document-relative
> stylesheet link, `font-display: optional`, the service-relative `@font-face`
> `src`, the crossorigin preloads, and the session-gated nginx `/srv/scripts/static/`
> location — is unchanged. What moved is the substrate: the files are now
> `scripts/share/www/landing.html` and `scripts/share/www/static/tokens.css` (D12's
> de-embed), and the assets are served by the **chassis static mount** (`Spec.WWW`),
> not a service-side `StaticHandler`. The `R-M*` ids below are unchanged; their
> tests read the shipped `share/www` tree / the chassis mount. Read every
> `internal/web/…` / `StaticHandler` mention below as its `share/www` / chassis
> equivalent.

**Problem.** The landing page flashes: it paints with the fallback system font,
then visibly **swaps** to the web fonts once they load. Worse, the swap is driven
by the **dashboard's** assets, not scripts's own — which also violates the
suite-wide requirement that every service be 100% independent.

The cause is a chain of apex-absolute references:

1. `landing.html` links the stylesheet origin-absolute:
   `<link rel="stylesheet" href="/static/tokens.css">`. A leading-slash URL
   resolves against the site root, so a browser on `/srv/scripts/` requests
   `https://<host>/static/tokens.css` — the `/srv/scripts/` mount is gone. nginx
   routes apex `/static/` to the **dashboard** (`MOUNT=/`), so the page renders
   with the **dashboard's** tokens.css. scripts's own embedded copy (served at
   `/srv/scripts/static/tokens.css` by `StaticHandler`, wired at `GET /static/`
   in `cmd/scripts/main.go`) is never requested.
2. The `@font-face` `src` in scripts's `tokens.css` is likewise origin-absolute
   (`url('/static/fonts/X.woff2')`) and resolves to the apex.
3. Every `@font-face` declares `font-display: swap`, which paints fallback and
   swaps to the web font after load — i.e. after first paint.
4. Nothing preloads the fonts.

Because the stylesheet itself loads from the apex, fixing only scripts's embedded
`tokens.css` would be inert — the browser never fetches it. The link must point
at scripts's own copy first.

**Decision.** Make the landing page load scripts's **own** embedded CSS and fonts,
swap-free. Four coordinated changes; no new dependency, no Go logic beyond the
existing `StaticHandler` (already wired). The change set spans
`internal/web/landing.html`, `internal/web/static/tokens.css`, and
`etc/nginx.conf`.

1. **`landing.html`: relativize the stylesheet link.**
   `href="/static/tokens.css"` → `href="static/tokens.css"`. Document-relative,
   it resolves against the mount-root document URL (`/srv/scripts/`, canonical
   trailing slash) to `/srv/scripts/static/tokens.css` — scripts's own copy. (No
   `<base href>` is introduced; `landing.html` carries no `Mount`, and the
   document-relative form needs none.)

2. **`tokens.css`: self-serve the fonts and stop the swap.** On all four
   `@font-face` blocks (Space Grotesk, IBM Plex Sans, IBM Plex Mono 400, IBM Plex
   Mono 500): `font-display: swap` → `font-display: optional` (`optional` forbids
   a post-first-paint swap — this removes the jar), and
   `src: url('/static/fonts/X.woff2')` → `url('fonts/X.woff2')` (relative to the
   now-self-served stylesheet → `/srv/scripts/static/fonts/X.woff2`).

3. **`landing.html`: preload the above-the-fold fonts** in `<head>`, beside the
   stylesheet link:

   ```html
   <link rel="preload" as="font" type="font/woff2" crossorigin
         href="static/fonts/space-grotesk.woff2">
   <link rel="preload" as="font" type="font/woff2" crossorigin
         href="static/fonts/ibm-plex-sans.woff2">
   ```

   The **display** (`Space Grotesk`) and **body** (`IBM Plex Sans`) families,
   painted immediately. `crossorigin` is **mandatory** (fonts fetch in CORS mode;
   without it the preload double-fetches and is wasted). The document-relative
   `href` resolves to the same `/srv/scripts/static/fonts/…` URL as the new
   `@font-face` `src`, so the preloaded bytes satisfy the `@font-face` request and
   first paint uses the real font. The mono family (IBM Plex Mono 400/500) is
   **not** preloaded (minor / below the fold; `optional` already prevents its jar).

4. **`etc/nginx.conf`: session-gate scripts's own static path.** Today only the
   exact landing root (`= /srv/scripts/`) is session-gated; everything else under
   the mount — including `/srv/scripts/static/…` — falls to the bearer-gated prefix
   (`location /srv/scripts/` → `auth_request /_authn`). A browser carries the
   dashboard **session cookie**, not a bearer token, so it would get **401** on
   the self-served CSS/fonts and the page would be unstyled. Add a session-gated
   static location mirroring the dashboard's own and wiki's precedent:

   ```nginx
   location /srv/scripts/static/ {
       auth_request /_session-authn;
       proxy_pass http://127.0.0.1:3003/static/;
       proxy_set_header Host $host;
       proxy_set_header X-Forwarded-Proto $scheme;
       proxy_http_version 1.1;
   }
   ```

   The prefix `/srv/scripts/static/` is replaced by upstream `/static/`, which
   scripts's `StaticHandler` serves. The exact `= /srv/scripts/` landing location,
   the bearer-gated `/srv/scripts/` prefix (MCP, its rate-limit `@prompts_authn_500`
   handling), the `= /srv/scripts/feed` denial, and the PRM bootstrap location are
   unchanged.

**Supersession.** D8 corrects the apex-borrow that scripts's D3-area web tests
currently pin in its buggy form:
- **D3's R-ASST-7Y1N** (`TestLandingHandlerLinksOnlyAppLocalStaticAssets`) asserts
  `landing.html` links `href="/static/tokens.css"`. Its intent ("links scripts's
  *local* tokens.css") is fulfilled — only the path form changes to the
  service-relative truth. The build updates that assertion to
  `href="static/tokens.css"`; the id is retained. (Its `forbidden` list —
  `dashboard`, `https://`, `http://` — stays satisfied: the new preloads use a
  document-relative `href` and `crossorigin`, no scheme.)
- The un-idded `tokens.css` font-src assertions
  (`TestTokensCSSDeclaresEmbeddedFontFaces` asserts `url('/static/fonts/…')`) and
  the un-idded canonical-structure assertion
  (`TestLandingHandlerUsesCronCanonicalStructureForScripts` asserts
  `<link rel="stylesheet" href="/static/tokens.css">`) are likewise updated to the
  relative form by the build loop; they carry no `R-` id to retain.
- **D3's R-ASST-5X8M / R-ASST-9Z3P** (`TestStaticHandlerServesTokensAndFonts` —
  `StaticHandler` serves `/static/tokens.css` and the four `/static/fonts/*.woff2`
  with correct content types) are **unchanged** and remain the real-substrate proof
  that scripts serves its own assets at its own paths.

**Boundary.** D8 owns the landing page's asset *loading* (the link, the
`@font-face` rules, the preload) and the nginx static gate. It does not change the
design-token values, the landing markup/copy (D6), or the landing handler/route
(D1/D2). It corrects D3's reasoning, which assumed an origin-absolute `/static/…`
is rewritten by nginx back to `/srv/scripts/static/…` — nginx strips inbound, it
does not rewrite outbound CSS, so the apex-borrow was real.

**Rejected.**
- **`font-display: block` / `fallback`.** `block` is FOIT (invisible text then
  pop-in); `fallback` still permits a late swap in its window. `optional` is the
  only value that forbids a post-paint swap. Rejected.
- **Keep the apex-absolute link, fix only the dashboard.** Would fix the flash but
  leaves scripts dependent on the dashboard serving its CSS — violating the
  independence requirement. Rejected.
- **Introduce `<base href="{{.Mount}}">`.** Would require threading `Mount` into
  the landing template's data; the document-relative link/preload need no base and
  resolve correctly at the canonical trailing-slash mount root. Rejected.
- **Origin-absolute preload `href` to match the old `src`.** Would re-pin the
  apex-borrow; the whole point is to self-serve. Rejected.

**Verification** (httptest against `LandingHandler` and `StaticHandler`, and a
fragment-grep against `etc/nginx.conf`, mirroring the existing R-NGNX tests).

- R-M59W-5CAW — **No `@font-face` declares `font-display: swap`; all use
  `optional`.** scripts's embedded `tokens.css` contains `font-display: optional` in
  every `@font-face` block and **no** `font-display: swap` occurrence — a block
  left as `swap` (the jar-causing value) fails this. *(served `GET /static/tokens.css`)*
- R-M6HS-J41L — **Font `src` is service-relative, not origin-absolute.** scripts's
  `tokens.css` contains **no** `url('/static/fonts/` occurrence and each
  `@font-face` `src` uses `url('fonts/…woff2')` — so it resolves to scripts's own
  `/srv/scripts/static/fonts/…`. *(served `GET /static/tokens.css`)*
- R-M8XL-ANIZ — **The landing page links its own tokens.css service-relative.**
  `GET /` renders a `<head>` containing `href="static/tokens.css"` and **no**
  origin-absolute `href="/static/tokens.css"` — so the browser fetches scripts's own
  copy, not the apex. *(httptest `LandingHandler`)*
- R-MA5H-OF9O — **The rendered `<head>` preloads the display and body fonts with
  `crossorigin`.** `GET /` returns a `<head>` containing a
  `<link rel="preload" as="font" type="font/woff2" crossorigin …>` for both
  `space-grotesk.woff2` and `ibm-plex-sans.woff2`, each `href` document-relative
  (`static/fonts/…`) and resolving to the same path as the `@font-face` `src`. A
  missing `crossorigin`, a missing family, or an origin-absolute href fails this.
  *(httptest `LandingHandler`)*
- R-MBDE-270D — **nginx session-gates scripts's own static path.** `etc/nginx.conf`
  contains a `location /srv/scripts/static/` block whose body carries
  `auth_request /_session-authn;` and `proxy_pass http://127.0.0.1:3003/static/;`
  — so a browser session can fetch the self-served CSS/fonts. A static path left
  bearer-gated (only the inherited `/srv/scripts/` prefix) fails this. *(fragment grep
  against `etc/nginx.conf`)*
