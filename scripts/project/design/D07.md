# Decision 7 — A top-left Home link to the dashboard landing page

**Decision.** Scripts's landing page gains one navigation affordance: a
**top-left Home link** that returns the viewer to the **dashboard landing page**.
It is a plain anchor to the apex root — `<a class="home" href="/">Home</a>` —
pinned to the page's top-left corner and styled with the existing Carbon label
tokens. This is scripts's own page control, defined and built entirely within
scripts, independent of any other service.

**Why `href="/"`.** scripts's page is served under `/srv/scripts/`; nginx is the front
door and the apex root `/` is the **dashboard** (the `DEFAULT` app that owns the
install/landing surface). A root-absolute link to `/` therefore resolves —
through nginx — to the dashboard landing page. No `/srv/...` prefix and no dynamic
value: the target is the bare apex root, the same for every viewer.

**Markup (added to `scripts/internal/web/landing.html`):**

- A new anchor as the **first child of `<main>`** — inside the centered column —
  so it leads the tab order:

  ```html
  <a class="home" href="/">Home</a>
  ```

- A `.home` rule in the inline `<style>`, beside the other label-style selectors:

  ```css
  .home {
    position: absolute;
    top: var(--space-8);
    left: 0;
    font-family: var(--font-mono);
    font-size: var(--text-label-size);
    line-height: var(--text-label-lh);
    font-weight: var(--text-label-weight);
    text-transform: uppercase;
    color: var(--color-text-muted);
    text-decoration: none;
  }

  .home:hover,
  .home:focus-visible {
    color: var(--color-text);
  }
  ```

  `main` carries `position: relative` and the link is absolutely positioned to its
  top-left — inside the centered column, not at the page edge. It reuses the
  mono-label scale and muted ink of the
  existing `.eyebrow` / `dt` labels, brightening to full-strength text on
  hover/focus — no new token and no new asset.

**Link text is `Home`, deliberately not `Dashboard`.** scripts's landing page guards
against external/cross-service asset URLs by forbidding telltale substrings
(`http://`, `https://`, `dashboard`) in the rendered body. Labeling the control
`Home` keeps that guard green while still conveying the destination.

**Scope guard.** This Decision adds **only** the home link — one anchor and one
style rule. The landing handler, its routes, the `dl`, the eyebrow / `h1` /
description, the embedded assets, and the nginx fragment are unchanged. The link
is static markup, identical in every rendered response (no new dynamic data).

**Rejected.**
- **Linking to `/srv/dashboard/` or a computed dashboard URL** — the dashboard is
  the apex `DEFAULT`; its landing page is `/`. A `/srv/...` path would be wrong.
- **A back-button styled as a large CTA / header bar** — out of proportion for a
  name+version page; a quiet corner link matches the minimal Carbon aesthetic.
- **Labeling it `Dashboard` / `← Dashboard`** — trips the body's `dashboard`
  asset guard; `Home` conveys the same intent without weakening the test.

**Verification.**
- R-HOME-8R2V — the rendered landing body contains a home link to the dashboard apex:
  an anchor whose `href` is exactly `/` (the body contains `href="/"`), distinct
  from the page's `/static/...` asset references.
