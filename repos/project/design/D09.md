# Decision 9 — State layout & retention

**Decision.** Everything durable lives under the service state root (so
opsctl's S3 snapshots cover it for free):

```
state/
  repos.db                       # the D2 database (appkit-derived name)
  repos/<name>/                  # canonical clone (D4) — removed by `delete`
  sessions/<session_id>/
    instructions.md              # pinned input — durable
    output.jsonl                 # transcript — durable, survives everything
    check.log                    # check output — durable (when a check ran)
    worktree/                    # the run — pruned per the rules below
```

**Worktree lifecycle** (`internal/repos/reaper.go` + runner hooks):

- **Success prunes immediately**: `WorktreeRemove` at the end of the success
  path — commits are pushed; the worktree has no residual value.
- **Failure keeps the crime scene**: the worktree stays until (a) a later
  session on the same `(repo, issue)` **succeeds** (supersession — the
  runner prunes every older failed worktree for that issue), or (b) the age
  sweep reclaims it.
- **The age sweep**: a small ticker in the dispatcher Worker removes
  worktrees of terminal sessions older than `REPOS_WORKTREE_TTL_DAYS`
  (default 14) — worktrees only; `instructions.md`, `output.jsonl`, and
  `check.log` are never swept.
- **Repo deletion** (D4/D7): removes `state/repos/<name>` and prunes
  worktrees of that repo's sessions (they are unusable without the canonical
  repo), but session rows and their transcript files remain readable via
  `session_get`/`session_output` forever.
- Cancelled sessions follow the failure rules (inspectable, swept by age).

Transcript durability is the product promise: no code path deletes
`output.jsonl`.

**Rejected.**
- **Pruning failed worktrees immediately** — the pushed branch shows *what*
  was produced, but the worktree shows the *uncommitted* state (test
  droppings, half-edits) that explains failures.
- **A tombstoned repo keeping its clone** — clone-only v1 makes every clone
  recoverable from GitHub; disk is the scarce thing.
- **Sweeping transcripts with the worktrees** — transcripts are the audit
  record and are cheap text; explicitly exempt.

**Verification** (real substrate: real git worktrees on fixture repos, real
files, injected clock for the sweep).

- R-FWST-R7DG — a succeeded session's `worktree/` is gone at terminal state
  while its `instructions.md`, `output.jsonl`, and `check.log` remain; a
  failed session's `worktree/` remains.
- R-FY0Q-4Z45 — supersession: after a failed attempt and a later succeeded
  attempt on the same issue, the failed attempt's worktree is pruned and its
  transcript remains; the sweep with a clock advanced past the TTL removes a
  lone failed session's worktree (and only the worktree), and leaves
  younger ones untouched.
- R-G0GI-WILJ — after repo deletion, `state/repos/<name>` is gone, that
  repo's session worktrees are gone, and `session_get`/`session_output` for
  its past sessions still return the full record and transcript lines.
