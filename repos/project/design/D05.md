# Decision 5 — The session engine: worktree-per-session, queue, and the confined agent

**Decision.** The engine is **copied** from prompts' runner/tools pattern
(`prompts/internal/{runner,sandbox,tools}`), not extracted into a shared
library — extraction is reconsidered once both runners are stable, mandatory
at a third copy; the known cost (shared-shape fixes land twice) is accepted.
Two deliberate divergences from prompts: the workspace is a **git worktree**
with its own lifecycle instead of a durable sandbox, and sessions are
**queued and capped** instead of unbounded.

`internal/runner/runner.go`:

```go
type Runner struct {
    store   *repos.Store
    git     *repos.Git            // D4
    proto   *repos.Protocol       // D6 — GitHub reporting + check gate + PR
    ttl     time.Duration         // REPOS_SESSION_TTL, default 30m
    maxRun  int                   // REPOS_MAX_SESSIONS, default 2
    model   ModelConfig           // REPOS_PROVIDER/REPOS_MODEL (validated at boot)
    mu      sync.Mutex
    cancels map[string]context.CancelFunc
    userCancelled map[string]bool
    wake    chan struct{}         // dispatcher doorbell
}
func (r *Runner) Enqueue(ctx, req SessionRequest) (repos.Session, error)
func (r *Runner) Cancel(sessionID string) bool
func (r *Runner) Dispatch(ctx context.Context) error   // the Spec.Workers loop
func (r *Runner) Recover(ctx context.Context) (int, error)
```

**Enqueue is the single session-construction point.** Every session — whether
started by the MCP tool surface (D7) or by webhook intake (D3) — is built by
`Enqueue` and nowhere else. `Enqueue` alone: generates the id, computes the
branch and attempt (`ikibot/session-<id>` for a manual run;
`ikibot/issue-<N>` / `ikibot/issue-<N>.<attempt>` for an issue, advancing the
attempt past any `BranchExists` on local **or** remote), sets
`LogPath = state/sessions/<id>/output.jsonl`, posts the D6 `queued` comment
when the session is queued behind an active one on the same repo, inserts the
`queued` row, and rings the `wake` doorbell. Because `LogPath` and the doorbell
are set here, a session built any other way (e.g. a hand-rolled
`InsertSession`) is malformed: an empty `LogPath` breaks the transcript open in
`run`, and a session inserted without ringing `wake` sits `queued` until the
next sweep tick rather than being admitted promptly. Intake (D3) lives in
`internal/repos`, which **cannot** import `internal/runner` (the dependency
runs runner→repos), so intake reaches `Enqueue` through an **`Enqueuer`
interface defined in `internal/repos`**, wired to the runner at the composition
root; `SessionRequest` moves into `internal/repos` and `internal/runner`
aliases it (exactly as it already aliases `IssueContent`), so the seam carries
one shared request type with no translation.

**Queue & concurrency.** `Enqueue` inserts a `queued` session row and rings
`wake`. The `Dispatch` worker admits the oldest queued session whenever both
gates pass: global `CountRunning() < maxRun` **and** no running session on
the same repo (one active session per repo — merge semantics, not git, is the
constraint; a queued-behind session triggers the D6 "queued" comment when it
is issue-driven). Each admitted session runs in its own goroutine with
`context.WithTimeout(ttl)`. `Cancel` flags and cancels; terminal
classification mirrors prompts (user cancel > TTL (`"session TTL exceeded"`)
> engine error > success). `Recover` runs at Worker start: orphaned `running`
rows → `failed` (`"interrupted by restart"`); orphaned `queued` rows stay
queued and re-dispatch.

**Session setup (runner-owned, deterministic).** For session S on repo R,
issue N: `Freshen(R)`; branch `ikibot/issue-<N>` for attempt 1, else
`ikibot/issue-<N>.<attempt>` (never force-push over an inspected failure;
`BranchExists` on local+remote advances the attempt); manual sessions use
`ikibot/session-<id>`. `WorktreeAdd` at `state/sessions/<id>/worktree` off
`origin/<default>`. The pinned `instructions.md` (issue title/body + full
comment thread via the github peer, or the MCP caller's instructions) is
written to `state/sessions/<id>/instructions.md` **before** the agent starts,
and its content is injected as the conversation's user turn (the file sits
outside the worktree, so it never pollutes the repo). Never touches the
default branch.

**The confined agent.** One agentkit `Conversation`, one `Send`, drained to
completion — exactly prompts' drive pattern, transcript (`output.jsonl`,
agentkit stream-json) at `state/sessions/<id>/output.jsonl`. The toolset is
`internal/tools`: **`Bash`, `Read`, `Write`, `Edit`, `Glob`, `Grep` only** —
cwd/confinement root = the session worktree, path confinement copied from
prompts (`confinePath`/`resolveExisting`). **No Fetch, no File* share tools,
no MCP, no deferred tools, no discovery** — the agent holds no credentials
and no suite surface; GitHub I/O and the event plane are runner-side (D6,
D8). The framing prompt states the contract: work in the worktree, commit
what you produce, `.ikibot/check` is the gate, honor `AGENTS.md` if present,
final message = summary for the PR body.

**Model config.** Env-configured service-wide (not per-record):
`REPOS_PROVIDER`/`REPOS_MODEL`, default **`anthropic` / `claude-opus-4-8`**,
validated at boot against agentkit's pricing table
(`provider.Pricing(model)`) plus a non-empty provider API key — an invalid
pair refuses to serve. Per-repo/per-issue overrides are deferred post-v1.

**Rejected.**
- **Extracting a shared engine library now** — two consumers is too early to
  freeze the seam; divergences (worktree lifecycle, queueing) are exactly
  where a premature abstraction would fight us.
- **Concurrent sessions per repo on separate branches** — git allows it;
  merge semantics don't. Two agents solving two issues on one tree produce
  conflicting PRs; serialize per repo, revisit post-v1.
- **Unbounded global concurrency (prompts' posture)** — a session is a
  full agent working a repo for up to 30 minutes; two is the box's budget
  (env-overridable), revisit post-v1.
- **Giving the agent Fetch/File*/MCP tools** — every removed surface is
  removed exfiltration and prompt-injection blast radius; issue text is
  untrusted input that talks directly to this agent.
- **Per-issue model overrides in v1** — settled as deferred.

**Verification** (real substrates: real git fixture remotes; a fake agentkit
provider that scripts tool calls; real SQLite).

- R-F4R4-YHBH — an admitted issue-session builds its worktree at
  `state/sessions/<id>/worktree` on branch `ikibot/issue-41` branched from
  the freshened `origin/<default>` tip (SHA-asserted), and the canonical
  clone's checked-out branch is unchanged; after a recorded failed attempt,
  the next session for the same issue gets `ikibot/issue-41.2`.
- R-F5Z1-C926 — the toolset handed to the conversation is exactly
  {Bash, Read, Write, Edit, Glob, Grep}; Bash executes with cwd = the
  worktree; a file tool given an absolute path outside the worktree or a
  `..` escape is refused (path-confinement copied behavior).
- R-F76X-Q0SV — `instructions.md` for an issue-session contains the issue
  title, body, and every thread comment fetched through the github peer
  (recorded stub), pinned to disk before the conversation starts; a manual
  session pins the caller's instructions verbatim.
- R-F8EU-3SJK — with `maxRun=2` and three enqueued sessions on three repos,
  exactly two run concurrently and the third starts only after a terminal
  state; two sessions enqueued on the **same** repo never run concurrently
  regardless of the global gate; dispatch order is FIFO by `created_at`.
- R-F9MQ-HKA9 — a session whose scripted engine outlives a short test TTL
  ends `failed` with error `session TTL exceeded`; `Cancel` mid-run ends it
  `cancelled`; both write `ended_at` and release their repo for the next
  queued session.
- R-FAUM-VC0Y — boot recovery: rows left `running` become `failed`
  (`interrupted by restart`) while `queued` rows survive and are dispatched;
  the sweep count is returned.
- R-FC2J-93RN — boot-time model validation: `REPOS_PROVIDER=anthropic` with
  an unknown `REPOS_MODEL` (or an empty provider API key) fails startup with
  an error naming the bad pair, before any route is served; the default pair
  passes against agentkit's real pricing table.
- R-2U0F-NNXH — the **webhook intake path** produces a runnable, promptly
  dispatched session: driving `Intake.Handle` with a recorded
  `issues`/`labeled`/`execute` delivery (through the `Enqueuer` seam wired to
  the runner) yields a `queued` row whose `LogPath` is
  `state/sessions/<id>/output.jsonl` (non-empty), and `Dispatch` admits it on
  the doorbell — without waiting for the sweep ticker — after which a scripted
  fake engine drives it to completion and the transcript is opened at that
  `LogPath`. Substrate: real migrated SQLite, a real git fixture remote, a
  scripted fake agentkit provider, and a recording github peer — not the MCP
  `Enqueue` entry point. (A hand-rolled session insert leaves `LogPath` empty
  and never rings the doorbell, so this test fails on either regression.)
