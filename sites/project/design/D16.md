# Decision 16 — Filesystem layout: files live at the served path; delete the working tree and symlink indirection

**Decision.** A site's files live **directly** at the location they are served
from — there is no separate editable "working tree" and no published "served
tree" of symlinks. `internal/sites/layout.go` collapses to a single helper:

```go
const DefaultRoot = "/opt/sites/state/www" // SITES_ROOT default (unchanged)

const (
    PublicSeg  = "public"
    PrivateSeg = "private"
)

// SiteDir is a site's on-disk directory for its visibility:
//   public  → <root>/public/<slug>
//   private → <root>/private/<slug>
func (l Layout) SiteDir(public bool, slug string) string

// SiteBase is <root>/<public|private> — the parent of a visibility's site dirs.
func (l Layout) SiteBase(public bool) string
```

**Deleted** from `layout.go` / `publish.go`: `WorkingSeg`, `WorkingDir`,
`WorkingBase`, `ServedSeg`, `ServedDir`, `ServedBase`, `ServedTierBase`,
`symlinkTarget`, and the entire `publish.go` symlink body (`Publish`/`Unpublish`'s
`os.Symlink`/`os.Remove` machinery). `internal/sites/publish.go` is removed.

**Visibility flip moves the directory in lockstep with the flag.** Flipping a
site public↔private renames its directory between the two parents so that content
and flag never disagree; it is never present under both at once:

```go
// Move relocates a site's directory between the private and public parents to
// match a new visibility. Called by the MCP set_visibility path alongside
// Store.SetVisibility. A no-op when already at the target. Tolerates a missing
// source dir (a site with no files yet).
func (l Layout) Move(slug string, toPublic bool) error
```

The database `public` flag (D15) is authoritative; `SiteDir`/`Move` keep the
filesystem a faithful mirror. The MCP tools are the only writer (D20), so the two
never drift.

**Why this is safe without a read-time symlink guard.** The served tree now holds
**real files only** — the intentional publish-symlink (`<tier>/<slug>` →
`working/<slug>`) that previously had to be followed is gone. The only remaining
confinement concern is ordinary path traversal *within* a site dir, which the
static server confines per request via `internal/files.ConfinePath` (D17), and
which the file tools already confine on write (D10/D11). There is no symlink layer
to defend.

**Rejected.**
- **Keeping a `working/` tree edited separately from the served tree.** That split
  existed only to stage edits before a publish step; with no publish step and
  "content that exists is served", edits go straight to the live directory. The
  operator explicitly wants edits immediately live.
- **Keeping the served tree as symlinks into a working tree.** The symlink tree
  was a filesystem re-encoding of DB state that only existed so nginx (which can't
  read the DB) could `alias` it. Once the sites *process* serves the bytes (D17),
  it reads the DB and serves the real folder — the symlinks are pure redundancy.
- **Copying files into a served tree on publish.** A copy is a second source of
  truth that drifts from the edit tree; serving the one real directory avoids it.

**Verification.**
- R-QV1H-JU04 — `Layout.SiteDir(true, "blog")` equals `<root>/public/blog` and
  `Layout.SiteDir(false, "blog")` equals `<root>/private/blog` (exact path
  formula, `<root>` = the Layout's root).
- R-QW9D-XLQT — after `Layout.Move("blog", true)` on a site whose files were at
  `<root>/private/blog`, the files exist at `<root>/public/blog` and
  `<root>/private/blog` no longer exists (and the reverse for `Move("blog", false)`);
  a `Move` to the current visibility is a no-op that leaves the files in place.
- R-QYP6-P587 — the working-tree/served-symlink machinery is gone: a scoped grep
  over `internal/` (excluding `project/`) for `symlinkTarget`, `os.Symlink`,
  `WorkingDir`, `ServedDir`, `ServedTierBase`, and `ServedBase` returns no matches,
  and `internal/sites/publish.go` does not exist.
