# Decision 14 — Delete the `internal/db` shim, normalize the composition root, and true up the doctrine doc

**Decision.** Three structural cleanups that land the reference shape whole,
mirroring crm's and notify's shim deletion and webhooks' composition-root
normalization:

1. **`internal/db` shrinks to its legitimate content.** The `Open` and `Migrate`
   wrapper functions in `sites/internal/db/fs.go` are deleted — appkit owns DB
   opening and the migration run (`Spec.Migrations: db.FS`, already wired). What
   remains is what genuinely must live in a sites package: the embedded migration
   set (`FS`; `go:embed` cannot cross modules) and its load guard
   (`migrations_load_test.go`, which asserts the real embedded set applies
   through `appkit/db`'s runner). The package's imports shrink accordingly (the
   `database/sql`, `context`, `appkit/db`, and `modernc.org/sqlite` blank-import
   used only by the wrappers go away; only `embed` remains). The three domain
   test harnesses that called the wrappers — `internal/mcp/tools_test.go`'s
   `newTestHandler`, `internal/sites/store_test.go`, and
   `internal/sites/publish_test.go` — call `appkit/db` directly (`appkitdb.Open`
   + `appkitdb.LoadMigrations(db.FS, "migrations")` + `appkitdb.Migrate`, a
   local helper apiece, exactly as notify's converted tests do). No test
   *assertion* changes — harness plumbing only.

2. **The composition root is normalized to the reference shape.** Today
   `cmd/sites/main.go` builds the Spec in two steps —
   `spec := sitesSpec(); spec.Handlers = func(rt) {…}; appkit.Main(spec)` — a
   **post-construction mutation** of `.Handlers`, the exact pattern the reference
   shape forbids (the pilots declare the `Spec` fully formed with no later field
   assignment). The fix: move the `Handlers` closure **inside** `sitesSpec()` so
   it returns a fully-formed `Spec` (the closure captures nothing from `main`, so
   this is a pure relocation), and `main()` shrinks to `appkit.Main(sitesSpec())`.
   The `sitesSpec()` **helper is kept** — `cmd/sites/main_test.go`'s
   manifest/registry tests (`TestManifestLibraryByteEqualsCommittedFile`,
   `TestSitesSpecUsesRegistryPort`, and the WWW/MCP field checks) call it to
   inspect the Spec without running the binary, exactly as notify's `notifySpec()`
   and cron/gmail's builders are used; a builder function is the sanctioned shape,
   only the post-construction `.Handlers` write is not. No behavior changes: the
   same Spec value reaches `appkit.Main`, and the `Handlers` closure body (the
   landing render, D12; the `POST /mcp` mount, D13; the layout/store/mirror
   wiring) is byte-identical, only relocated. There is no separate
   `internal/sitesapp` spec package to remove — sites never had one.

3. **`sites/AGENTS.md` (and its `CLAUDE.md` symlink — one file, edit
   `AGENTS.md`) states the converted truth**, per the suite doctrine (docs state
   current truth; purge what the change falsified; no archaeology). The falsified
   claims to purge: the embedded-landing/`internal/web` web surface, the
   `internal/mcp` JSON-RPC transport description, any `internal/db`
   `Open`/`Migrate` wrapper mention, and any hardcoded-port language superseded by
   the registry (D9). Replaced by the current truth: the `share/www` web surface
   through `Spec.WWW`/`rt.WWW()` (chassis-served landing + `/static/`),
   `internal/mcp` as the fourteen-domain-tool table over the `appkit/mcp` chassis
   transport with chassis-supplied `health`/`reflection`, and `internal/db` as
   the embedded migration set only. If `sites/AGENTS.md` does not exist yet, this
   item is satisfied structurally by the other docs; the phase's grep-based done
   bar is written to no-op cleanly in that case (see the phase).

**Rejected.**
- **Keeping the wrappers for import brevity** — a per-service shim that can drift,
  for two saved lines per call site; crm's and notify's D14 already made this
  call.
- **Deleting `internal/db` entirely** — the migrations embed must live in a sites
  package (a module-local `go:embed`) and the load guard is genuinely app-side;
  the package shrinks to its legitimate content instead.
- **Dropping the `sitesSpec()` helper to inline the whole `Spec` at the
  `appkit.Main(appkit.Spec{…})` call** (crm's exact form) — would force
  `cmd/sites/main_test.go`'s manifest/registry tests to reconstruct the Spec
  themselves or scrape it back out of a captured value; the builder function is
  the sanctioned reference shape (notify's `notifySpec()`, cron/gmail's
  builders), and the only thing (f) forbids is the *post-construction* `.Handlers`
  write — not the helper. Kept.
- **Leaving the post-construction `spec.Handlers = …` as-is** — it is the precise
  pattern the reference shape normalizes away for suite consistency; the closure
  captures nothing from `main`, so relocating it into the builder is a zero-risk
  move that makes sites match every converted sibling.
- **Leaving `AGENTS.md` for a later pass** — the file would assert wiring that no
  longer exists in the same tree that changed it; docs move with the code.

**Verification.** Structural — no `R-XXXX-XXXX` ids (shim deletion +
composition-root normalization + doc truth; the behaviors are already pinned by
the existing domain/MCP/web suites and the D12–D13 ids). Realized by a plan phase
whose done bar is:

- `grep -n "func Open\|func Migrate" sites/internal/db/*.go` returns no matches,
  while `grep -n "go:embed migrations" sites/internal/db/fs.go` still matches and
  `sites/internal/db/migrations_load_test.go` still exists.
- `grep -rn "db.Open\|db.Migrate" sites --include=*.go | grep -v internal/db/`
  returns no matches (every domain test harness moved to `appkit/db` directly).
- `grep -n "spec.Handlers\s*=\|\.Handlers\s*=\|\.Workers\s*=" sites/cmd/sites/main.go`
  returns no matches (no post-construction field mutation), while
  `grep -n "func sitesSpec()" sites/cmd/sites/main.go` still matches (the builder
  is kept) and `grep -n "Handlers:" sites/cmd/sites/main.go` matches inside the
  `sitesSpec()` literal (the closure was relocated, not deleted); `main()`'s body
  is `appkit.Main(sitesSpec())`.
- If `sites/AGENTS.md` exists:
  `grep -n "go:embed\|internal/web\|StaticHandler\|LandingHandler\|JSON-RPC" sites/AGENTS.md`
  returns no matches and `grep -c "share/www\|appkit/mcp" sites/AGENTS.md` is ≥ 1.
- The suite stays green (design Conventions commands) with no assertion changes.
