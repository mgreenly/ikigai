# Decision 8 — Self-serve the landing page's fonts and eliminate the FOUT

**Problem.** The landing page flashes: it paints with the fallback system font,
then visibly **swaps** to the web fonts once they load. Worse, the swap is driven
by the **dashboard's** assets, not sites's own — which also violates the
suite-wide requirement that every service be 100% independent.

The cause is a chain of apex-absolute references:

1. `landing.html` links the stylesheet origin-absolute:
   `<link rel="stylesheet" href="/static/tokens.css">`. A leading-slash URL
   resolves against the site root, so a browser on `/srv/sites/` requests
   `https://<host>/static/tokens.css` — the `/srv/sites/` mount is gone. nginx
   routes apex `/static/` to the **dashboard** (`MOUNT=/`), so the page renders
   with the **dashboard's** tokens.css. sites's own copy (served at
   `/srv/sites/static/tokens.css` by the chassis static mount over `share/www`,
   D12) is never requested.
2. The `@font-face` `src` in sites's `tokens.css` is likewise origin-absolute
   (`url('/static/fonts/X.woff2')`) and resolves to the apex.
3. Every `@font-face` declares `font-display: swap`, which paints fallback and
   swaps to the web font after load — i.e. after first paint.
4. Nothing preloads the fonts.

Because the stylesheet itself loads from the apex, fixing only sites's embedded
`tokens.css` would be inert — the browser never fetches it. The link must point
at sites's own copy first.

**Decision.** Make the landing page load sites's **own** CSS and fonts,
swap-free. Four coordinated changes; no new dependency, no Go logic (the chassis
static mount over `share/www` already serves `GET /static/`, D12). The change set
spans `share/www/landing.html`, `share/www/static/tokens.css`, and
`etc/nginx.conf`.

1. **`landing.html`: relativize the stylesheet link.**
   `href="/static/tokens.css"` → `href="static/tokens.css"`. Document-relative,
   it resolves against the mount-root document URL (`/srv/sites/`, canonical
   trailing slash) to `/srv/sites/static/tokens.css` — sites's own copy. (No
   `<base href>` is introduced; `landing.html` carries no `Mount`, and the
   document-relative form needs none.)

2. **`tokens.css`: self-serve the fonts and stop the swap.** On all four
   `@font-face` blocks: `font-display: swap` → `font-display: optional`
   (`optional` forbids a post-first-paint swap — this removes the jar), and
   `src: url('/static/fonts/X.woff2')` → `url('fonts/X.woff2')` (relative to the
   now-self-served stylesheet → `/srv/sites/static/fonts/X.woff2`).

3. **`landing.html`: preload the above-the-fold fonts** in `<head>`, beside the
   stylesheet link:

   ```html
   <link rel="preload" as="font" type="font/woff2" crossorigin
         href="static/fonts/space-grotesk.woff2">
   <link rel="preload" as="font" type="font/woff2" crossorigin
         href="static/fonts/ibm-plex-sans.woff2">
   ```

   The **display** (`Space Grotesk`) and **body** (`IBM Plex Sans`) families,
   painted immediately. `crossorigin` is **mandatory** (fonts fetch in CORS mode;
   without it the preload double-fetches and is wasted). The document-relative
   `href` resolves to the same `/srv/sites/static/fonts/…` URL as the new
   `@font-face` `src`, so the preloaded bytes satisfy the `@font-face` request and
   first paint uses the real font. The mono family is **not** preloaded (minor /
   below the fold; `optional` already prevents its jar).

4. **`etc/nginx.conf`: session-gate sites's own design-asset static path.** sites
   already session-gates two **disk-backed** tiers — `/srv/sites/public/` (no
   auth) and `/srv/sites/private/` (`auth_request /_session-authn`) — but both are
   served straight from disk via `alias /opt/sites/state/www/{public,private}/`
   (the customer's website mirror). **Neither covers the design assets.** Those
   ship in `share/www/static` and are served by the **chassis static mount** at
   upstream `/static/` (D12); the browser would request them at
   `/srv/sites/static/…`, which
   matches **no** existing location (there is no catch-all `/srv/sites/` prefix —
   the landing root is exact-match `= /srv/sites/`). So a browser session would not
   reach the self-served CSS/fonts at all. Add a session-gated static location that
   **proxies** to the upstream (not an `alias`), mirroring the dashboard's own
   precedent:

   ```nginx
   location /srv/sites/static/ {
       auth_request /_session-authn;
       proxy_pass http://127.0.0.1:3004/static/;
       proxy_set_header Host $host;
       proxy_set_header X-Forwarded-Proto $scheme;
       proxy_http_version 1.1;
   }
   ```

   The prefix `/srv/sites/static/` is replaced by upstream `/static/`, which the
   chassis static mount serves from `share/www/static`. This is distinct from the
   two disk-backed tiers: `/srv/sites/static/` is the service's **own** design
   assets (proxied to the process), `/srv/sites/{public,private}/` are the
   **user's** website files (disk `alias`).
   The exact `= /srv/sites/` landing location, the exact `= /srv/sites/mcp` bearer
   endpoint, the PRM bootstrap, the public/private disk tiers, and the
   `@sites_authn_500` re-emit location are all unchanged.

**Supersession.** D8 corrects the apex-borrow that two shipped D3 ids currently
pin in their buggy form:
- **D3's R-ASST-5K9Q** asserts `landing.html` references `/static/tokens.css`
  (origin-absolute). Its intent ("references sites's *own* embedded tokens.css")
  is fulfilled — only the path form changes to the service-relative truth. The
  build updates that assertion to `static/tokens.css`; the id is retained. (Its
  companion checks — no `dashboard` and no `://` cross-service URL — still hold:
  the new preload `href`s are document-relative `static/fonts/…`.)
- **D3's R-ASST-3H7N** asserts sites's `tokens.css` is served `text/css` and
  contains `url('/static/fonts/space-grotesk.woff2')`. The font-`src` substring is
  updated to the `url('fonts/…')` form; the id is retained. Its `text/css`
  content-type proof and the Carbon-token presence checks are **unchanged**.
- **D3's R-ASST-7M2S** (the chassis static mount serves the four
  `/static/fonts/*.woff2` with `font/woff2`) is **unchanged** and remains the
  real-substrate proof that sites
  serves its own font bytes.

**Byte-pin to cron (do not break conformance).** sites's `share/www` landing
files are **byte-pinned to cron's canonical**: the conformance tests (D6, now
living beside the relocated landing tests in `cmd/sites` after D12)
`TestTokensCSSMatchesCronCanonical` assert `bytes.Equal(sites tokens.css, cron
tokens.css)`, and `TestLandingTemplateConformsToCronCanonicalWithSitesCopy`
asserts sites's `landing.html` equals cron's with only the three sites name/copy
substitutions
(title `· cron`→`· sites`, eyebrow, and the description `<p>`). Consequence: this
FOUT change to `tokens.css` and `landing.html` must be **byte-identical** to
cron's same change — the identical `font-display: optional`, the identical
relative `src`, the identical two preload `<link>`s — or those two conformance
tests fail. **Therefore cron's FOUT change must land first**; sites then mirrors
it byte-for-byte. (The tokens.css header comment "vendored locally for the cron
landing page" stays as-is — it is part of the byte-pinned canonical.) This
build-order dependency is carried prominently in the phase doc.

**Boundary.** D8 owns the landing page's asset *loading* (the link, the
`@font-face` rules, the preload) and the new nginx static proxy gate. It does not
change the design-token values, the landing markup/copy (D6), or the landing
handler/route (D1/D2). It does not touch the public/private disk tiers (D4) — the
design-asset path `/srv/sites/static/` (served from `share/www/static`) is a
distinct, third static concern. It
corrects D3's reasoning, which assumed an origin-absolute `/static/…` is rewritten
by nginx back to `/srv/sites/static/…` — nginx strips inbound, it does not rewrite
outbound CSS, so the apex-borrow was real.

**Rejected.**
- **`font-display: block` / `fallback`.** `block` is FOIT (invisible text then
  pop-in); `fallback` still permits a late swap in its window. `optional` is the
  only value that forbids a post-paint swap. Rejected.
- **Keep the apex-absolute link, fix only the dashboard.** Would fix the flash but
  leaves sites dependent on the dashboard serving its CSS — violating the
  independence requirement. Rejected.
- **Reuse the disk-backed `/srv/sites/private/` tier for the embedded assets.**
  That tier `alias`es the customer's website mirror on disk, not the process's
  embedded `/static/` route — it cannot serve the in-binary tokens.css/fonts.
  Rejected; a distinct proxied `/srv/sites/static/` location is required.
- **Introduce `<base href="{{.Mount}}">`.** Would require threading `Mount` into
  the landing template's data; the document-relative link/preload need no base and
  resolve correctly at the canonical trailing-slash mount root. Rejected.
- **Origin-absolute preload `href` to match the old `src`.** Would re-pin the
  apex-borrow; the whole point is to self-serve. Rejected.

**Verification** (httptest over the `appkit/web` Site loaded from `share/www` and
the chassis static mount, and a fragment-grep against `etc/nginx.conf`, mirroring
the existing R-NGNX tests — D12).

- R-629P-84O5 — **No `@font-face` declares `font-display: swap`; all use
  `optional`.** sites's shipped `tokens.css` contains `font-display: optional` in
  every `@font-face` block and **no** `font-display: swap` occurrence — a block
  left as `swap` (the jar-causing value) fails this. *(served `GET /static/tokens.css`)*
- R-63HL-LWEU — **Font `src` is service-relative, not origin-absolute.** sites's
  `tokens.css` contains **no** `url('/static/fonts/` occurrence and each
  `@font-face` `src` uses `url('fonts/…woff2')` — so it resolves to sites's own
  `/srv/sites/static/fonts/…`. *(served `GET /static/tokens.css`)*
- R-64PH-ZO5J — **The landing page links its own tokens.css service-relative.**
  `GET /` renders a `<head>` containing `href="static/tokens.css"` and **no**
  origin-absolute `href="/static/tokens.css"` — so the browser fetches sites's own
  copy, not the apex. *(httptest over the `share/www` Site)*
- R-65XE-DFW8 — **The rendered `<head>` preloads the display and body fonts with
  `crossorigin`.** `GET /` returns a `<head>` containing a
  `<link rel="preload" as="font" type="font/woff2" crossorigin …>` for both
  `space-grotesk.woff2` and `ibm-plex-sans.woff2`, each `href` document-relative
  (`static/fonts/…`) and resolving to the same path as the `@font-face` `src`. A
  missing `crossorigin`, a missing family, or an origin-absolute href fails this.
  *(httptest over the `share/www` Site)*
- R-675A-R7MX — **nginx session-gates sites's own design-asset static path.**
  `etc/nginx.conf` contains a `location /srv/sites/static/` block whose body
  carries `auth_request /_session-authn;` and
  `proxy_pass http://127.0.0.1:3004/static/;` (proxied, not an `alias`) — so a
  browser session can fetch the self-served CSS/fonts. The existing exact landing,
  exact MCP, PRM, public/private disk tiers, and `@sites_authn_500` locations are
  unchanged. *(fragment grep against `etc/nginx.conf`)*
