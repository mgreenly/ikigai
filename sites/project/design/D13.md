# Decision 13 — MCP surface over `appkit/mcp`: `internal/mcp` becomes the tool table

**Problem.** sites's `internal/mcp` carries the copy-pasted JSON-RPC transport
(`ServeHTTP`, the `initialize`/`notifications/initialized`/`tools/list`/
`tools/call` dispatch, the JSON-RPC error writer, the `Identity` type, the
result-envelope helpers) plus its own `health` tool implementation — the same
near-identical plumbing every service duplicated. The chassis now owns all of it
(appkit D8 transport, D9 standard tools, consumed here as a fixed external
contract); what is genuinely sites's is its **fourteen domain tools** — the
lifecycle verbs (`describe`, `create`, `list`, `delete`, `mkdir`, `publish`,
`unpublish`, `sync`) and the six file tools (`file_write`, `file_read`,
`file_edit`, `file_glob`, `file_grep`, `file_list`) — with their descriptors,
schemas, validation, confinement, and the dropbox-mirror `sync` hop.

**Two consequences of adopting the chassis standard tools** (both are honest wire
changes forced by appkit D9's reserved names, and appkit is frozen — the service
adapts):

1. **`health` moves to the chassis.** sites's local `toolHealth` is deleted;
   `health` is chassis-registered. The chassis `health` renders the same
   `appkit.Envelope(version, service, details)` **plus** the caller's
   `owner_email`/`client_id` from the injected identity — byte-identical to
   sites's local health output (the chassis health carries the same auth-proof
   fields, verified against crm's converted seam). Its *description* string
   becomes the chassis canonical ("…for this service…" rather than "…for the
   sites service…"); no test pins that text.
2. **`reflection` is gained.** sites has no `reflection` tool today; the chassis
   always registers one. sites is neither a producer nor a consumer, so its
   reflection reports an **empty event graph** (`{"publishes":[],
   "subscribes":[]}`) — `rt.Events()` is an empty `outbox.Registry` and
   `rt.Subscriptions()` is nil, both nil-safe through the seam. The listed
   surface therefore **grows by exactly one tool** (`reflection`). This does not
   collide with sites's own `describe` tool: `describe` is a domain tool (the
   create→edit→publish lifecycle) and is **not** a reserved name, so it stays
   declared alongside the chassis `reflection` (the event graph). The two are
   distinct axes.

**Decision.** `internal/mcp` is reshaped from a transport into a **declaration**:

```go
// internal/mcp — what remains.
const Instructions = /* sites's MCP instructions string, wire-unchanged */

// Tools declares sites's fourteen domain tools over the store/layout/baseURL and
// the dropbox mirror client, returning []mcp.Tool. health and reflection are NOT
// declared — the chassis registers them (declaring either fails New's
// reserved-name check).
func Tools(store *sites.Store, layout sites.Layout, baseURL string, mirror sites.MirrorClient) []mcp.Tool

// NewHandler assembles the /mcp http.Handler at the composition root:
// appkit/mcp.New over Instructions + Tools(...) + the Router-threaded
// Service/Version/Health/Events/Publishes/Subscriptions.
func NewHandler(store *sites.Store, layout sites.Layout, baseURL string, mirror sites.MirrorClient, rt *appkit.Router) (http.Handler, error)
```

- **Deleted:** the JSON-RPC plumbing (`jsonRPCRequest`, the `ServeHTTP`
  dispatch, `writeJSONRPCResult`/`writeJSONRPCError`, `idOrNull`, the local
  `Identity` type, `handleToolCall`/`dispatchTool`, `toolDescriptors`), the
  local `toolHealth` implementation, and the private result-envelope helpers
  (`toolResultText`/`toolResultErr`) — replaced by the chassis result helpers
  (`mcp.TextResult`/`mcp.JSONResult`/`mcp.ErrorResult`).
- **Kept (wire content unchanged):** the descriptor/schema helpers
  (`desc`/`obj`/`descTyp`, now producing `mcp.Tool` values), every domain tool's
  description and schema, each handler's body (slug validation, working-tree
  mutation, `files.*` calls with `files.ConfinePath` confinement mapped to
  `path_escapes_working_dir`, the `sync` reconcile over the mirror client, the
  `renderSite`/`errResult` projections), and the closed error-envelope vocabulary
  — the wire bytes an agent sees for every domain tool are unchanged.
- **The mirror-client seam changes from field injection to a constructor
  parameter.** Today `NewHandler` returns a `*Handler` whose `client` field is
  set afterward via `SetMirrorClient`. In the declaration shape the tool handlers
  are closures built by `Tools(...)`, so the mirror client must be present at
  declaration time: it becomes a parameter to `Tools`/`NewHandler`, and
  `SetMirrorClient` is deleted. Behavior is unchanged (`sync` still enumerates
  `/list` and fetches `/content` through the same `sites.MirrorClient`); only the
  wiring order at the composition root and the test harness's construction move.

**The wire surface partition after conversion:** sites **declares** its 14 domain
tools; the chassis **supplies** `health` and `reflection`. `tools/list` therefore
returns **exactly 16** tools. This **supersedes D11's R-0KIG-2A5X**, which pinned
a 15-tool set with `health` declared locally and no `reflection`: the retained
id's expected set grows to include the chassis `reflection` and treats
`health`/`reflection` as chassis-supplied. The build updates that one assertion;
the id is retained (D11's file-tool behavioral ids R-0FMU-J775 / R-0GUQ-WYXU /
R-0I2N-AQOJ / R-0JAJ-OIF8 are unaffected and re-prove through the new seam).

**Rejected.**
- **Keeping the local transport** — the status quo; a protocol fix would still be
  a many-service edit.
- **Declaring `health`/`reflection` in sites's table to preserve the exact old
  surface** — the chassis reserves the names (appkit D8/D9) precisely so services
  cannot fork the standard tools; a declared `health` fails `New`. The gained
  `reflection` is the cost of the uniform surface, and is benign (empty graph for
  a neither-service) and consistent with every other converted service.
- **Renaming or dropping `describe` because `reflection` now exists** —
  `describe` is a real domain tool (the hosting lifecycle) with its own test and
  a distinct purpose from the event-graph `reflection`; the surface keeps both.
- **Dissolving `internal/mcp` into `cmd/sites`** — the fourteen tool
  descriptors, schemas, validation, and error mapping with their test harness are
  a real package.

**Verification** (through the composition-root seam — the `appkit/mcp` handler
built by `NewHandler` over a temp DB, temp `SITES_ROOT`, and a fake mirror
client; the existing domain-tool behavioral ids continue to pin the per-tool
behaviors through the new seam).
- R-0UUY-N97T — `tools/list` through the assembled handler returns **exactly
  sixteen** tools: the 14 domain tools sites declares (`describe`, `create`,
  `list`, `delete`, `mkdir`, `publish`, `unpublish`, `sync`, `file_write`,
  `file_read`, `file_edit`, `file_glob`, `file_grep`, `file_list`) **plus**
  `health` and `reflection` supplied by the chassis — proving the table/chassis
  partition (a table that declares its own `health`/`reflection` fails `New`'s
  reserved-name check; a surface missing them fails the count).
- R-P21E-0285 — `tools/call reflection` (no arguments) through the assembled
  handler for this neither-a-producer-nor-consumer service returns
  `{"publishes":[], "subscribes":[]}` (empty arrays, not a crash) — proving the
  chassis reflection is nil-safe over sites's empty `Events`/`Subscriptions` and
  that sites gains a working reflection tool without becoming an event-plane
  participant.
