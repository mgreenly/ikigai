# Decision 10 — `internal/files`: confined filesystem operations as native Go

**Problem.** sites's MCP file tools (`file_read`/`file_edit`/`file_glob`/
`file_grep`) are implemented by bridging each `tools/call` into the local
`agentkit` module: `internal/mcp/files.go` builds an `agentkit/wire.ToolUseBlock`
and calls `agentkit/tools.Dispatch(ctx, sandboxRoot, block)`, whose `confine.go`
enforces the per-site sandbox. But **agentkit exists to give services access to
LLMs** — its `tools` package is a *model-facing* tool registry meant to be driven
by an in-process agent generating JSON tool-use blocks. sites runs **no agent and
no model**; it only ever borrowed agentkit's confined `Read`/`Edit`/`Glob`/`Grep`
implementations as a filesystem toolkit. The dependency is a category mismatch,
and it forces a two-layer JSON round-trip (MCP args → wire block → agentkit's
typed decode) plus a second, subtly different confinement implementation living
beside sites's own (`internal/mcp/tools.go`'s `confinePath`, used by the native
`file_write`/`file_list`/`mkdir`).

The migration target is the remote `github.com/ikigenba/agentkit`, which **removed
the file-tool sub-library entirely** — it keeps only the LLM/orchestration surface
(`agentkit.Tool`, providers, embeddings). So there is nothing for sites to migrate
*to*: the reusable piece it depended on no longer exists. The service `prompts`,
which does run agents, re-implemented the six confined file operations itself in
`prompts/internal/tools/tools.go` and merely wraps each in `agentkit.NewTool` for
its orchestrator.

**Decision.** Lift the confined filesystem logic into a new **native Go package
`sites/internal/files`** — plain functions over Go types, no agent framing, no
JSON, no `agentkit`. The function *bodies* are the same substance as `prompts`'
workers (path confinement, `os.ReadFile` + line slicing, `strings.Replace`, a
segment-aware glob walk, the grep walk); the shaping differs because sites calls
them directly from an MCP handler rather than handing them to an LLM. This package
becomes the **single home for every confined filesystem operation** sites
performs and the **single trust boundary** for file access.

Public surface (illustrative signatures; full bodies are a build phase's job):

```go
package files

// ErrEscapes is the sentinel every confinement rejection wraps, so a caller can
// classify "path left the sandbox" distinctly from an I/O error.
var ErrEscapes = errors.New("path escapes the working dir")

// ConfinePath resolves p against root and verifies the result stays inside root,
// resolving symlinks. Absolute p is permitted when it still lands under root; a
// non-existent path under root resolves (write/mkdir create case). Any escape
// returns an error wrapping ErrEscapes.
func ConfinePath(root, p string) (string, error)

func Read(root, path string, offset, limit int) (string, error)
func Write(root, path, content string, append bool) error
func Edit(root, path, oldStr, newStr string, replaceAll bool) (replaced int, err error)
func Glob(root, pattern, path string) ([]string, error)
func Grep(root, pattern, path, glob string) ([]Match, error)
func List(root, scope string) ([]FileInfo, error)
func Mkdir(root, path string) error

type Match struct { Path string; Line int; Text string }
type FileInfo struct { Path string; Size int64; Md5 string }
```

Notes that pin the shape:

- **One confinement check, everywhere.** `ConfinePath` (ported from `prompts`,
  the symlink-resolving form) is the *only* confinement in sites. The native
  `file_write`/`file_list`/`mkdir` handlers switch from `internal/mcp`'s old
  `confinePath` to `files.ConfinePath`; that old helper is deleted. Consequence,
  accepted deliberately: an absolute path that resolves *inside* a site's tree is
  now permitted by every file tool (escapes are still rejected). The tool
  descriptions that say "absolute … rejected" are reworded to "confined".
- **Pure Go types, no JSON in the package.** `Glob` returns `[]string`, `Grep`
  returns `[]Match`, `List` returns `[]FileInfo` — the package marshals nothing.
  Serialization to the MCP result is the handler's job (D11). This is the point of
  making them functions, not tools.
- **`Glob` matches path-segment-wise, with `**` recursion.** The pattern is
  matched one `/`-separated segment at a time: `*`, `?`, and `[…]` behave as
  `filepath.Match` and **never cross `/`**, while `**` matches any run of segments
  including none — so `**/*.css` finds `.css` files at the search base *and* at
  any depth beneath it. `Glob` walks the confined search base and tests each
  base-relative path against the pattern rather than delegating to
  `filepath.Glob`, whose `*` cannot cross `/` and whose `**` collapses to a
  single-segment match — the reason recursive `**/…` patterns matched nothing.
  `filepath.WalkDir` does not follow symlinks, so a match can never leave the
  sandbox, and the results stay base-relative and sorted as before.
- **`Grep`/`Read` are the straightforward form**, not old agentkit's
  Claude-Code-mirroring surface: no `output_mode`/`-i`/`-A`/`-B`/`type`/`cat -n`.
  Nothing in sites depends on those (the file tools have **no** internal caller —
  they exist solely for a connected agent), so the minimal form is sufficient.
- **`Write` keeps `append`** (the one behavior sites's native write had that
  agentkit's lacked), folding the former native `toolFileWrite` body in.
- **No new dependency.** The package uses only the standard library.

`sync` (the Dropbox reconcile in `internal/sites`) keeps its own logic but routes
its writes through `files.Write`/`files.ConfinePath` so confinement has exactly
one implementation in the module.

**Rejected.**
- **Migrate the bridge to the remote `github.com/ikigenba/agentkit`.** The remote
  module deleted the file-tool sub-library; "migrating" would mean copying
  `prompts`' workers into sites *and still* importing agentkit only to wrap them
  in `agentkit.Tool` and immediately `.Call()` them — ceremony around functions
  sites calls directly, pulling an LLM-orchestration dependency into a service
  that runs no LLM. agentkit is for services that need an agent; sites doesn't.
- **Keep two confinement implementations** (native for write/list/mkdir, ported
  for read/edit/glob/grep). Two subtly different answers to "is this path safe" in
  one service is a latent security bug; unify on the stronger (symlink-resolving)
  one.
- **Return `string` from every function (as `prompts` does).** `prompts` returns
  strings because a tool result is fed back to a model as text. sites builds an
  MCP response; returning native Go types lets the handler serialize them the
  ordinary way and keeps the package free of JSON.
- **Port old agentkit's full Grep/Read surface.** Rich options with no caller and
  no client are dead surface; the minimal form is the honest scope.

**Verification** (in-package unit tests in `internal/files`, using `t.TempDir()`
as the sandbox root; no network, no running suite, no agent).

- R-027Y-BQ1I — **`ConfinePath` rejects an escape with `ErrEscapes`.** A relative
  `../…` that leaves root, and an absolute path outside root, both return an error
  for which `errors.Is(err, files.ErrEscapes)` is true. Falsifiable: a nil error,
  or an error not wrapping the sentinel, fails it.
- R-03FU-PHS7 — **`ConfinePath` accepts in-sandbox paths, existing or not.** A
  relative path under root, an absolute path that resolves under root, and a
  not-yet-existing path under root (the create case) all return a confined
  absolute path and no error. Falsifiable: rejecting an in-root path, or rejecting
  a non-existent-but-in-root path, fails it.
- R-04NR-39IW — **`ConfinePath` rejects a symlink that resolves outside root.** A
  symlink placed under root but pointing outside it is rejected (error wraps
  `ErrEscapes`), proving confinement resolves symlinks rather than trusting the
  lexical path. Falsifiable: returning the symlink's in-root lexical path fails it.
- R-05VN-H19L — **`Read` returns content and pages by offset/limit.** With
  `offset=0,limit=0` it returns the whole file; with `offset/limit` set it returns
  exactly that 1-based line window. Falsifiable: returning the whole file when a
  window was asked, or the wrong window, fails it.
- R-073J-UT0A — **`Edit` replaces and reports the count.** Replaces the first
  occurrence by default and every occurrence when `replaceAll`, returning the
  number replaced; a missing `oldStr` returns an error and does not write.
  Falsifiable: replacing all when `replaceAll` is false, a wrong count, or
  silently succeeding on a missing `oldStr`, fails it.
- R-08BG-8KQZ — **`Glob` returns confined relative matches.** Returns the matching
  paths (relative to the search base) as `[]string`; a pattern is resolved under
  root and never escapes it. Falsifiable: returning absolute paths, non-matches,
  or a match outside root, fails it.
- R-3ZP8-T0GP — **`Glob` matches `**` recursively across any depth, including
  zero.** With a tree holding a base-level `a.css` and a nested
  `assets/css/style.css`, `**/*.css` returns *both* (the base-level file at zero
  intermediate segments and the deeply-nested one), and a trailing `assets/**`
  returns every file beneath that subtree. Falsifiable: the pre-fix behavior —
  `**/*.css` returning `[]` (or omitting either the base-level or the nested
  match) — fails it.
- R-40X5-6S7E — **A single-segment wildcard never crosses `/`.** With files at
  the base and in a nested subdir, `*.css` matches only the `.css` files directly
  in the search base (or the `path`-scoped base), never one in a nested subdir,
  and its results stay base-relative. Falsifiable: `*.css` returning a nested
  `sub/x.css` — the `**` fix over-matching single-segment patterns — fails it.
- R-09JC-MCHO — **`Grep` returns confined `[]Match`.** Each match carries the
  relative path, 1-based line number, and matching line text; the search is
  confined under root. Falsifiable: a wrong line number, missing match, or a match
  from outside root, fails it.
- R-0AR9-048D — **`Write` truncates, appends, and creates parents.** Default write
  truncates; `append=true` appends to an existing file (and creates it if
  missing); missing parent directories are created. Falsifiable: appending when
  `append` is false, truncating when it is true, or failing on a missing parent
  dir, fails it.
- R-0D71-RNPR — **`List` returns per-file path/size/md5, scoped.** Walks the tree
  under root (or the confined `scope` subdir) and returns one `FileInfo` per
  regular file with its working-root-relative path, byte size, and md5 hex.
  Falsifiable: a wrong md5, wrong size, a directory entry, or a file outside the
  scope, fails it.
- R-0EEY-5FGG — **`Mkdir` creates the directory and parents, confined.** Creates
  the (possibly nested) directory under root; a path that escapes root returns an
  error wrapping `ErrEscapes`. Falsifiable: not creating parents, or creating a
  directory outside root, fails it.
