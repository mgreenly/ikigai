# Decision 17 — In-process static serving of hosted sites (`internal/serve`)

**Decision.** A new sites-owned package `internal/serve` serves the two site
trees from `SITES_ROOT` over the loopback HTTP server. It is the code that makes
"sites serves every byte under its mount" true.

```go
package serve

// Handler serves files under a single visibility root (root == SITES_ROOT/public
// or SITES_ROOT/private). urlPrefix is the request-path prefix nginx forwards
// under ("/public/" or "/private/"); the remainder after the prefix is the
// slug-plus-path, confined under root before any read.
func Handler(root, urlPrefix string) http.Handler
```

The composition root mounts one per visibility (D-wiring in main.go):
`rt.Handle("GET /public/", serve.Handler(layout.SiteBase(true), "/public/"))` and
`rt.Handle("GET /private/", serve.Handler(layout.SiteBase(false), "/private/"))`.
Both are mounted **ungated in-process** — nginx applies the visibility gate (D18).

**Serving semantics** (purpose-built, not the chassis `noDirFS` which 404s all
directories and so can't serve a site's `index.html`):

- A request resolving to a **regular file** returns `200` with a `Content-Type`
  from the file extension (`mime.TypeByExtension`, e.g. `.css` → `text/css`).
- A request resolving to a **directory that contains `index.html`** returns `200`
  serving that `index.html`.
- A request resolving to a **directory with no `index.html`** returns `404` — never
  a directory listing, never `403`. (Deliberate divergence from the old nginx
  `alias` tier's `403`; a directory index is never leaked.)
- A request whose path **escapes** the root via `..` or an absolute path is refused
  `404` — confined via `internal/files.ConfinePath` before any filesystem access.
- A request for a **missing** path returns `404`.
- A request for a **directory without a trailing slash** `301`-redirects to the
  trailing-slash form, so relative asset links resolve.

The handler serves **real files** — there is no symlink layer to follow (D16), so
no symlink-escape guard is needed beyond the per-request `ConfinePath`. It is
distinct from the chassis `GET /static/` mount, which serves the service's *own*
Carbon UI assets from `share/www` (D12) and is unrelated to hosted-site content.

**Rejected.**
- **Reusing the chassis `appkit/web` static handler (`noDirFS`).** `noDirFS.Open`
  returns `ErrNotExist` for directories to suppress listings, which also kills the
  `index.html`-at-directory-root behavior a website requires. Hosted sites need
  index mapping *and* no listing — different semantics, hence a purpose-built
  handler.
- **`http.FileServer(http.Dir(root))` unmodified.** It auto-generates directory
  listings (leaks the file inventory) and its default `404`/redirect behavior is
  fine but the listing is not — the handler must suppress listings while keeping
  index mapping.
- **Letting nginx serve the files off disk (`alias`).** The whole point of the
  change (D18): nginx serving off disk is the redundant-state path being removed.
- **A read-time symlink-resolution guard.** Unnecessary once the served tree holds
  real files (D16); `ConfinePath` on the request path is the only guard needed.

**Verification.** (Driven with `net/http/httptest` over a real tree under a
`t.TempDir()` root.)
- R-QZX3-2WYW — a `GET` for an existing file (e.g. `/public/blog/app.css` over a
  tree containing `blog/app.css`) returns `200` with a `Content-Type` derived from
  the extension (`text/css`), and the file's bytes as the body.
- R-R14Z-GOPL — a `GET` for a directory that contains `index.html` (e.g.
  `/public/blog/`) returns `200` serving that `index.html`'s bytes.
- R-R2CV-UGGA — a `GET` for a directory with **no** `index.html` returns `404`
  with no directory listing in the body (and specifically not `403`).
- R-R3KS-886Z — a `GET` whose path escapes the root via `..` (e.g.
  `/public/../../etc/passwd`) or an absolute traversal returns `404` and serves no
  file outside the root.
- R-R4SO-LZXO — a `GET` for an existing directory without a trailing slash (e.g.
  `/public/blog`) returns a `301` redirect to the trailing-slash form (`/public/blog/`).
- R-R60K-ZROD — a `GET` for a path that does not exist under the root returns `404`.
