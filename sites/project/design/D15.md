# Decision 15 — Data model: a `public` boolean and `created_by`, retiring the publish lifecycle

**Decision.** The `sites` row drops the publish-lifecycle state and becomes a
plain record of *what exists, at what visibility, created by whom*:

| column | type | meaning |
|---|---|---|
| `name` | TEXT PK | slug (unchanged grammar `^[a-z0-9][a-z0-9-]{0,62}$`, reserved-name guard) |
| `public` | INTEGER NOT NULL DEFAULT 0 | visibility boolean: `1` = public, `0` = private |
| `created_by` | TEXT NOT NULL DEFAULT '' | owner email that created the site |
| `source_path` | TEXT | dropbox-sync provenance (nullable; **unchanged**) |
| `created_at` | TEXT NOT NULL | UTC creation timestamp |
| `updated_at` | TEXT NOT NULL | UTC last-update timestamp |

The retired columns — `tier`, `published`, `published_at` — are **removed**. A
site's visibility is the single `public` boolean; there is no "unpublished" state.

**Schema change is two immutable migrations** (per Conventions; created with
`bin/create-migration sites <name>`, never hand-named):

1. An **additive** migration adds `public` and `created_by` (both `NOT NULL`
   with defaults so existing rows are valid), keeping the old columns in place so
   the codebase stays green while callers migrate.
2. A later **cleanup** migration drops `tier`, `published`, `published_at` (SQLite
   `ALTER TABLE … DROP COLUMN`, or a STRICT-preserving table rebuild), after no
   code reads them.

`002_sites.sql` and `20260609135943_add_source_path.sql` remain frozen.

**Store contract** (`internal/sites/store.go`). The `Site` struct and CRUD lose
the lifecycle fields and gain the new ones:

```go
type Site struct {
    Name       string
    Public     bool
    CreatedBy  string
    SourcePath string     // dropbox-sync provenance (unchanged)
    CreatedAt  time.Time
    UpdatedAt  time.Time
}

// Create inserts a fresh row: public defaults false (private), created_by is the
// caller-supplied owner email. Returns ErrExists on a duplicate slug.
func (s *Store) Create(ctx context.Context, name, createdBy string) (Site, error)

// SetVisibility flips a site's public flag (and bumps updated_at); the on-disk
// directory move is D16's concern, invoked in lockstep. Returns ErrNotFound.
func (s *Store) SetVisibility(ctx context.Context, name string, public bool) error
```

`Publish`, `Unpublish`, `ErrInvalidTier`, `PublishedAt`, `Tier`, and `Published`
are deleted. `Get`/`List`/`Delete`/`SetSourcePath` keep their shapes (their column
lists change to the new set). `List` stays ordered by `name`.

**Rejected.**
- **Keeping `tier TEXT` with values `'public'`/`'private'`.** The operator called
  it out: it is a binary, not a spectrum. A boolean is the honest type and can't
  hold a third accidental value.
- **Keeping a `published` boolean alongside `public`.** `published` was fully
  redundant with `tier != ''`; carrying any "exists but not served" flag
  reintroduces the draft state the model deliberately drops. A site that exists is
  served.
- **Deriving visibility from the folder location instead of a column.** The DB is
  hooked into the MCP surface and feeds the landing list; it must know visibility
  without walking the filesystem. The column is authoritative and the folder
  mirrors it (D16).
- **One migration that adds and drops together.** Splitting additive-then-cleanup
  is what lets every intermediate build stay green while callers move off the old
  columns.

**Verification.**
- R-QRDS-EIS1 — `Store.Create(ctx, name, createdBy)` persists `createdBy` into
  `created_by`; a subsequent `Get`/`List` returns a `Site` whose `CreatedBy`
  equals the value passed (proven against a real migrated SQLite DB).
- R-QSLO-SAIQ — a site created by `Store.Create` has `Public == false` by default
  (a new site is private, not public) until visibility is explicitly set.
- R-QTTL-629F — `Store.SetVisibility(ctx, name, true)` then `Get` returns
  `Public == true`, and `SetVisibility(ctx, name, false)` returns it to
  `Public == false`; `updated_at` advances; a missing slug yields `ErrNotFound`.
- R-QQ5W-0R1C — after the full migration set runs, `pragma table_info(sites)`
  reports columns including `public` and `created_by` and **excludes** `tier`,
  `published`, and `published_at` (proven against the real SQLite the runner uses).
