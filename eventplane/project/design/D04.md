# Decision 4 ‚Äî Consumer surface: routing fields on `consumer.Event`

**Decision.** The consumer engine's delivery semantics are untouched (baseline:
handler-return cursor gate, resync, backoff, structural-vs-transport). What
changes is the *shape of what a handler sees*: `consumer.Event` carries the
routing address, decomposed, plus the canonical key for matching:

```go
// Event is one event delivered to the handler, decomposed from the wire
// envelope. Kind and Subject come from the envelope JSON (the authoritative
// structure); the SSE event: line is the rendered key and is not parsed.
type Event struct {
    ID      string          // envelope "id" (unchanged)
    Source  string          // envelope "source" (unchanged)
    Time    string          // envelope "time" (unchanged)
    Kind    string          // envelope "kind"
    Subject string          // envelope "subject" ("" when subjectless)
    Payload json.RawMessage // opaque domain snapshot (unchanged)
}

// Key returns the event's canonical routing key,
// routing.Key(e.Source, e.Kind, e.Subject) ‚Äî what a handler-side filter
// matches against with routing.Match.
func (e Event) Key() string
```

The `Type` field is deleted (hard cutover, with D1). `parseEvent` reads
`kind`/`subject` from the envelope JSON instead of taking the type from the
`event:` line; an envelope that decodes with an **empty kind** is malformed
under the new contract and is handled by the engine's existing
unparseable-envelope path ‚Äî log, run no handler, advance the cursor (so a
poison frame never wedges the feed; this is the engine envelope-skip, not
`ErrSkip`).

*Filtering stays in the services.* `consumer.Config` has no `Filter` field
today (verified in research) and gains none: the engine still invokes the
handler for every event and the handler's return still gates the cursor. A
service filters by calling `routing.Match(pattern, ev.Key())` in its handler ‚Äî
the matcher is the exported seam, not an engine policy.

*Subscription follows the same cutover.* `consumer.Subscription` ‚Äî the
declarative `{Source, Filter, Description, Handler}` record that appkit
carries and renders in reflection ‚Äî keeps its shape, but its `Filter` field is
re-documented as a **glob over the canonical routing key** in the `routing`
dialect (e.g. `dropbox:create/bills/**/*.pdf`), no longer a `path.Match` glob
over the dotted type. The `Match(eventType string) bool` method is
**deleted**, and with it the package's `path` import: it was a second matcher
seam with divergent semantics (`path.Match` cannot express `**`, and the
method swallowed `ErrBadPattern` as a silent non-match), and it has exactly
one caller in the suite (notify's push handler), which moves to
`routing.Match(sub.Filter, ev.Key())` ‚Äî the same handler-side pattern as any
other filter. appkit only carries and renders `Filter` opaquely and never
calls `Match`. One matcher, one seam; `Subscription` stays pure declaration.

*Control-frame dispatch is safe by construction:* the reserved `event:` names
(`resync`, `caught-up`, `status`) contain no `:`, while every canonical key
contains one (`source:`‚Ä¶), so the existing literal switch in `handleFrame`
cannot confuse a key for a control frame.

**Rejected.**

- *A `Filter` field on `consumer.Config` (engine-side filtering)* ‚Äî the
  engine's contract is "deliver everything; the handler decides", and cursor
  semantics around filtered-out events (advance silently?) would complicate
  the baseline for no caller that needs it; services filter in the handler
  with the shared matcher.
- *Taking Kind/Subject by parsing the `event:` line* ‚Äî violates "nobody
  parses a key"; the envelope carries the structure.
- *Keeping `Type` alongside Kind/Subject during transition* ‚Äî no dual-format
  period (D1).
- *Revising `Subscription.Match` to wrap `routing.Match` over the key* ‚Äî a
  bool-returning wrapper must swallow the malformed-pattern error (the exact
  silent non-match D2 forbids), and it duplicates the one exported matcher
  seam for the sake of a single caller; deleting it is smaller and leaves
  exactly one way to match.

**Verification.**

- R-3VDM-JK38 ‚Äî end-to-end (real `FeedHandler` + `httptest` +
  `consumer.Run`): a handler receives an Event whose `Kind`, `Subject`,
  `ID`, `Source`, `Time`, and `Payload` equal what the producer appended;
  the `Event` struct has no `Type` field.
- R-3WLI-XBTX ‚Äî `Event.Key()` equals the canonical key: for the appended
  (source `dropbox`, kind `create`, subject `/bills/a.pdf`) it returns
  `dropbox:create/bills/a.pdf`, identical to the frame's `event:` line; for
  a subjectless event it returns `ledger:recorded`-shaped output.
- R-3XTF-B3KM ‚Äî with keyed `event:` lines in play, control frames are still
  dispatched as control end-to-end: a stream carrying `caught-up`, `status`,
  a keepalive, and a `resync` never invokes the handler for those frames and
  never corrupts the cursor (the baseline control-frame behavior holds under
  the new frame format).
- R-4098-2N20 ‚Äî an event frame whose envelope decodes with an empty `kind`
  is engine-skipped: the handler is not invoked, the cursor still advances
  past it, and the next event is delivered (real feed connection).
- R-95KP-1QIO ‚Äî filtering through the declared Subscription seam: a
  `Subscription` whose `Filter` is a canonical-key glob, with a handler that
  gates on `routing.Match(sub.Filter, ev.Key())`, acts end-to-end (real
  `FeedHandler` + `httptest` + `consumer.Run`) on exactly the events its
  declared `Filter` selects ‚Äî so the field reflection reports and the runtime
  match cannot drift. `Subscription` itself exposes no `Match` method and the
  `consumer` package no longer imports `path` (the deletion, checked
  structurally alongside this test).
- R-3Z1B-OVBB ‚Äî end-to-end filtering: with events
  `dropbox:create/bills/aws/1.pdf`, `dropbox:create/notes.txt`, and
  `dropbox:delete/bills/aws/1.pdf` in the outbox, a handler filtering with
  `routing.Match("dropbox:create/bills/**/*.pdf", ev.Key())` acts on exactly
  the first event, while the cursor advances past all three (delivery is
  unfiltered; selection is the handler's).
