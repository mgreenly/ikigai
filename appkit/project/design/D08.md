# Decision 8 — The `appkit/mcp` JSON-RPC transport over a declared tool table, with structured results

**Problem.** Two problems, one seam. First (built): eleven services carried a
copy-pasted ~150–190-line MCP transport — `ServeHTTP`, the `initialize` /
`notifications/initialized` / `tools/list` / `tools/call` dispatch,
`writeJSONRPCError`, the error envelope — differing only in server name,
instructions string, and tool table; a protocol fix had to be made eleven
times. Second (this revision): tool results are agent-shaped only — JSON
marshalled into a `text` content block by `JSONResult`, errors as bare
`ErrorResult(msg)` prose — so a deterministic caller (a scripts run, any
machine consumer under `docs/structured-mcp-design.md`) must parse prose and
string-match error messages. The suite decision is that MCP is the single verb
surface for agents *and* machines, so the transport must carry a machine
rendering.

**Decision.** The package `appkit/mcp` owns the transport and speaks MCP
revision **`2025-06-18`** (the lowest revision carrying structured tool
output; see `project/research/research.md`). A service declares its tools; the
chassis speaks JSON-RPC 2.0 over plain HTTP POST (no SSE/streaming):

```go
package mcp // import "appkit/mcp"

const protocolVersion = "2025-06-18"

// Tool is one declared MCP tool: its wire descriptor plus its handler.
type Tool struct {
    Name         string
    Description  string
    InputSchema  map[string]any // JSON Schema object
    // OutputSchema, when non-nil, is the JSON Schema of the tool's
    // structuredContent and is surfaced in tools/list. Nil for prose tools
    // (guide/describe/raw-content reads), whose results are text-only.
    OutputSchema map[string]any
    Handler      func(ctx context.Context, args json.RawMessage, id server.Identity) (map[string]any, error)
}

// Options is unchanged from the built revision (Service, Version,
// Instructions, Tools, Health, Events, Publishes, Subscriptions).

// ErrorCode is the suite's closed tool-error vocabulary. Adding a member is
// an explicit appkit change, never an ad-hoc service string.
type ErrorCode string

const (
    ErrValidation        ErrorCode = "validation"
    ErrNotFound          ErrorCode = "not_found"
    ErrConflict          ErrorCode = "conflict"
    ErrTooLarge          ErrorCode = "too_large"
    ErrSourceUnavailable ErrorCode = "source_unavailable"
    ErrInternal          ErrorCode = "internal"
)

// Result helpers:
func TextResult(text string) map[string]any
// StructuredResult replaces JSONResult (deleted, same move): the result
// carries structuredContent == v plus one text block whose bytes are the
// compact JSON marshal of v — the machine rendering and the agent rendering
// of the same value, from one call.
func StructuredResult(v any) (map[string]any, error)
// ErrorResult now requires a code (breaking signature change, converted
// suite-wide in the services' adoption phases; the suite deploys together):
// {content: [text msg], isError: true, structuredContent: {code, message}}.
func ErrorResult(code ErrorCode, msg string) map[string]any
```

Wire semantics:

- **`initialize`** → result with `protocolVersion` `"2025-06-18"`,
  `capabilities.tools`, `serverInfo{name: Service, version: Version}`, and
  `instructions`. The inbound `MCP-Protocol-Version` header and the client's
  proposed version are not validated — replying with the served version is
  the negotiation (research §MCP spec).
- **`notifications/initialized`** → acknowledged with no result body.
- **`tools/list`** → the standard tools (D9) plus every declared Tool, each as
  `{name, description, inputSchema}` plus `outputSchema` **when the Tool
  declares one**; the key is absent (never `null`/`{}`) for prose tools.
- **`tools/call`** → dispatch by name to the Tool's `Handler` with the raw
  `arguments` and the identity read from `X-Owner-Email` / `X-Client-Id`; the
  returned map is the JSON-RPC `result`, passed through verbatim (the
  transport never validates a result against `OutputSchema` at runtime —
  conformance is the declaring service's test suite's job).
- **Error taxonomy** — `-32700` malformed body; `-32601` unknown method;
  `-32602` invalid `tools/call` params **and** an undeclared tool name;
  **`-32603` for a Go error returned by a declared tool's Handler**. Under
  the structured contract every *domain* failure is an
  `ErrorResult(code, msg)` inside a successful response; a Go error escaping
  a handler now genuinely means "the service broke" and is labelled internal,
  not "invalid params".
- The transport carries **no token logic** — nginx is the trust boundary. A
  loopback machine caller asserting `X-Owner-Email`/`X-Client-Id` itself is
  indistinguishable from nginx by design (`docs/structured-mcp-design.md`,
  identity contract); the transport neither knows nor cares.

Mounting stays in `Spec.Handlers` (one line), gated:
`rt.Handle("POST /mcp", rt.RequireIdentity(h))`.

**Rejected.**
- **A separate per-service domain HTTP API mirroring the MCP verbs** — a
  permanent second surface on every service, a second route class to guard,
  and a mirroring discipline to police, in exchange for saving a machine
  caller one JSON-RPC envelope (ten lines of client code). The suite decision
  (`docs/structured-mcp-design.md`) rejects it; the byte-plane endpoints that
  already exist (content endpoints, dropbox's filesystem API) are not
  verb-shaped and stay.
- **Keeping `JSONResult` beside `StructuredResult`** — a deprecated twin
  invites drift and un-adopted stragglers; the suite deploys together, so the
  singular move (delete `JSONResult`, convert every call site in the
  services' adoption phases) is safe and leaves one way to do it.
- **Open-ended string error codes** — services minting their own codes is
  stringly-typed drift; the typed closed set makes the vocabulary
  compile-time visible and its growth an explicit appkit change.
- **Runtime validation of results against `OutputSchema`** — a JSON-Schema
  validator dependency and a per-call cost to re-prove what the declaring
  service's own tests already pin; the transport surfaces the schema and
  helps produce conforming results, nothing more.
- **A newer protocol revision (e.g. `2025-11-25`)** — adds capabilities
  (tasks, elicitation, richer transports) this deliberately minimal plain-POST
  transport does not implement; `2025-06-18` is the lowest revision carrying
  structured output.
- **Adopting an external MCP SDK / keeping the transport per-service /
  `any`-typed handler results / a `Spec.Tools` auto-mount field** — rejected
  in the built revision, unchanged: minimal transport, one copy, the
  `map[string]any` result convention, tool tables closing over the domain
  Service inside `Handlers`.

**Verification** (through the real `ServeHTTP` seam with a test tool table).
- R-MCJJ-NXJR — `initialize` returns `serverInfo.name`/`.version` equal to
  `Options.Service`/`.Version` and `instructions` equal to
  `Options.Instructions`.
- R-WPNN-6Q9E — `initialize` returns `protocolVersion` exactly
  `"2025-06-18"` — a build still answering `2025-03-26` fails this.
- R-MDRG-1PAG — `tools/list` includes every declared Tool with its exact name,
  description, and inputSchema.
- R-WQVJ-KI03 — `tools/list` surfaces a declared Tool's `outputSchema`
  verbatim, and a Tool with nil `OutputSchema` has **no** `outputSchema` key
  in its descriptor (key-absence asserted, not schema-emptiness).
- R-MEZC-FH15 — `tools/call` for a declared tool invokes its Handler with the
  request's raw `arguments` bytes and returns the handler's map as the
  JSON-RPC `result`.
- R-MG78-T8RU — the identity passed to the Handler carries the request's
  `X-Owner-Email` and `X-Client-Id` header values.
- R-WTBC-C1HH — `StructuredResult(v)` returns a result whose
  `structuredContent` unmarshals equal to `v` **and** whose single text block
  is the compact JSON marshal of the same `v` — the two renderings cannot
  disagree (asserted by comparing the parsed text block to
  `structuredContent`, not by string-comparing to a fixture).
- R-WUJ8-PT86 — `ErrorResult(code, msg)` returns `isError: true`, a
  `structuredContent` of exactly `{code: <code>, message: <msg>}`, and a text
  block equal to `msg`; the code round-trips as the typed constant's string
  (e.g. `ErrNotFound` → `"not_found"`).
- R-WVR5-3KYV — a declared tool whose Handler returns a Go error yields
  JSON-RPC error `-32603`, while a `tools/call` naming an undeclared tool
  yields `-32602` — the two are distinguishable on the wire (a build mapping
  both to `-32602` fails this).
- R-MHF5-70IJ — an unknown method returns JSON-RPC error `-32601`, and a
  `tools/call` naming an undeclared tool returns the error shape (not a `200`
  success result).
- R-MIN1-KS98 — a malformed (non-JSON) body returns JSON-RPC error `-32700`.
- R-MJUX-YJZX — `New` returns a non-nil error for a table with two tools of
  the same name, and for a tool named `health` or `reflection` (reserved by
  D9); it does not silently shadow.
