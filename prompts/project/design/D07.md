# Decision 7 — Runner

## Decision

`runner.go` replaces the `agent.Run` + `wire.Session` execution model with `agentkit.Conversation.Send` + stream drain.

**`execute` rewrite** (key steps, replacing the current body after the pinned-input read):

```go
// 1. Build the provider (D4)
provider, err := r.buildProvider(cfg, os.Getenv)

// 2. Build the Conversation
builtins := tools.All(r.sandbox.Root(run.ID), r.sourcePortAllowed) // D5/D21 — eager, always surfaced
suiteGroups := r.discover(ctx, run.OwnerEmail, run.PromptID) // D6 — deferred groups

conv := &agentkit.Conversation{
    Provider:          provider,
    Model:             cfg.Model,
    System:            buildSystem(sysPrompt),
    Log:               logFile,       // LogRecord JSONL written here
    Gen:               buildGenSettings(cfg),
    Retry:             buildRetryPolicy(cfg),
    MaxToolIterations: cfg.ToolLoopLimit, // 0 → agentkit default (1000)
    Tools:             builtins,          // sandbox tools only
    DeferredTools:     suiteGroups,       // suite tools, loaded on demand (D19)
}

// 3. Send the single user turn
userMsg := buildUserMessage(userPrompt, eventBytes)
stream := conv.Send(ctx, userMsg)

// 4. Drain — logging is a side-effect of the Log writer; we drain for
//    the error and usage
for range stream.Events() {}

runErr := stream.Err()
usage := stream.Usage()
cost := stream.Cost()
conv.Close() // emits the summary LogRecord

// 5. Serialize usage for the run row
usageJSON := serializeUsage(usage, cost)
```

**`buildSystem`**: concatenates `agent.FramingPrompt` (copied from the local `agentkit/agent/prompt.go` into `prompts/internal/runner/`) with the optional system prompt, separated by `"\n\n"`. Same logic as before.

**`buildUserMessage`**: for a manual run returns `userPrompt` directly. For an event-triggered run appends the event preamble + pretty-printed JSON as a second paragraph (same logic as the current `buildRequest`, but as a plain string rather than a `[]provider.Block` — `Conversation.Send` takes a single string).

**`buildGenSettings`**: maps Config fields to `agentkit.GenSettings`. Reasoning precedence (effort → thinking_level → thinking_budget → thinking, first non-zero wins). Duration strings parsed via `time.ParseDuration`.

**`buildRetryPolicy`**: maps `MaxAttempts`, `BaseDelay`, `MaxDelay`, `MaxElapsed`, `IgnoreRetryAfter` from Config to `agentkit.RetryPolicy`. Zero values leave each field at the agentkit default.

**`serializeUsage`**: replaces the old `captureUsage` file scanner. Directly marshals `agentkit.Usage` and `agentkit.Cost` into the `usage_json` column format. No file scanning.

**Run output format**: `logFile` receives the agentkit's native `LogRecord` JSONL. Event types include `turn_start`, `message`, `tool_use`, `tool_result`, `usage`, `turn_end`, `summary`. The `describe` tool is updated to reflect this format.

**What is removed**: `wire.NewSession`, `wire.Session`, `captureUsage` (file scanner), `buildRequest`, `model.Resolve`, `model.ModelContext`, `model.DefaultEffort`, `clientFactory`, `defaultClientFactory`, all imports of `agentkit/agent`, `agentkit/wire`, `agentkit/model`, `agentkit/provider`, `agentkit/tools`.

**Test seam**: `Runner.buildProvider` (injectable, D4) and `Runner.discover` (injectable, D6 — returning `[]agentkit.DeferredToolGroup`) are the two seams. Tests inject a fake `buildProvider` returning a stub `agentkit.Provider` (implements `RoundTrip`, `Name`, `Pricing`) and a fake `discover` returning an empty slice. The stub provider's `RoundTrip` returns a pre-canned `*agentkit.RoundTrip` (no events, `FinishStop`, no error) so the conversation completes in one turn.

```go
type fakeProvider struct{}
func (f *fakeProvider) Name() string { return "fake" }
func (f *fakeProvider) Pricing(model string) (agentkit.Pricing, bool) {
    return agentkit.Pricing{}, true // always OK
}
func (f *fakeProvider) RoundTrip(ctx context.Context, req *agentkit.Request) *agentkit.RoundTrip {
    msg := agentkit.Message{Role: agentkit.RoleAssistant, Blocks: []agentkit.Block{agentkit.TextBlock{Text: "done"}}}
    return agentkit.NewRoundTrip(msg, agentkit.FinishStop, agentkit.Usage{}, nil, nil)
}
```

## Rejected

**Keep `wire.Session` output** — rejected (product decision D format decision): LogRecord is the native format; a translation bridge would be maintenance debt with no consumer benefit.

**Multi-turn conversation** — prompts runs a single user turn per run (one `Send` call). Multi-turn is out of scope. The `Conversation` struct supports it but we do not use it.

**Stream events for real-time progress** — the published agentkit's stream no longer emits `TextDelta`/`ReasoningDelta` events. Real-time text is written to `Conversation.Log` (the LogRecord file) which is tail-readable via `run_output`.

## Verification

- R-K5I9-YGS9 — a run seeded with `provider: "fake"` uses the injected fake provider's `RoundTrip` (verified by injecting a provider that records calls); runs with different provider strings use different injected providers.
- R-K6Q6-C8IY — after a run completes, `output.jsonl` contains at least one line parseable as a `LogRecord` with `type: "turn_start"` and one with `type: "summary"`.
- R-K7Y2-Q09N — after a successful run, the run row's `usage_json` is a non-empty JSON object (not `""`).
- R-K95Z-3S0C — a run started with a TTL of 50ms against a fake provider that blocks until context cancellation is marked `failed` with `error: "run TTL exceeded"`.
