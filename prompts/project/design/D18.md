# Decision 18 — Delete the chassis shims (`internal/db` wrappers) and true up the doctrine doc

**Decision.** Two cleanups that land the reference shape whole, mirroring crm's
and notify's shim deletion:

1. **`internal/db` shrinks to its legitimate content.** The `Open` and `Migrate`
   wrapper functions are deleted — appkit owns DB opening and the migration run
   (`Spec.Migrations: db.FS`). What remains is what genuinely must live in a
   prompts package: the embedded migration set (`migrationsFS` / `FS`;
   `go:embed` cannot cross modules) and the app-side migration guard tests
   (`prompts_schema_test.go`, `migrations_feed_offset_test.go`,
   `migrations_outbox_test.go`, `migrations_provider_model_aliases_test.go`,
   `redesign_migration_test.go`). The wrapper-exercising tests in
   `internal/db/db_test.go` (`TestOpenAndMigrate`, `TestMigrate_IsIdempotent`,
   `TestMigrate_RefusesDowngrade`) are deleted with the functions they call —
   they proved appkit/db's runner through prompts' shim, and appkit/db owns
   those tests. Every other test harness that stood up a schema through the
   wrappers calls appkit's db package directly instead —
   `appkitdb.Open(path)` and
   `appkitdb.Migrate(ctx, conn, appkitdb.LoadMigrations(db.FS, "migrations"))` —
   with **no test assertion changes**, only harness plumbing. The harnesses
   affected: `internal/consume/smoke_test.go`,
   `internal/db/{migrations_provider_model_aliases_test.go,
   redesign_migration_test.go}`, `internal/prompt/{service_test.go,
   outcome_test.go, store_test.go}`, `internal/mcp/mcp_test.go`, and
   `internal/runner/runner_test.go`.

2. **The composition-root package doctrine is trued up.** prompts has **no**
   service-level `AGENTS.md`/`CLAUDE.md`; its authoritative surface-posture
   doctrine is the **package doc comment of `cmd/prompts/main.go`** (the D10/
   Phase-13 precedent). That comment currently asserts several things this
   round — and an earlier, never-trued-up event-plane change — falsified:
   - **"It is neither an event-plane producer nor a consumer — no `/feed`, no
     consumer loop, no background worker"** — already false and now doubly so:
     prompts **is** a producer (`Feed: "/feed"`, `Events`, `Producer` emitting
     `run.succeeded`/`run.failed`) and a **consumer** of six upstreams via
     `Spec.Consumers`. Replace with the producer + `Spec.Consumers` truth (D15).
   - **"the bare MCP surface"** — replace with the `appkit/mcp` tool table
     (`internal/mcp` declares the sixteen domain tools; the chassis serves the
     transport plus `health`/`reflection`) (D17).
   - **the landing page "served ungated in-process"** clause — restate it as the
     `share/www` surface served through the chassis `Spec.WWW` / `rt.WWW()`
     (still nginx-gated, in-process ungated) (D16).

   The "performs no token logic of its own" / "nginx is the trust boundary"
   posture stays true and stays stated. No archaeology — the comment states the
   converted current truth only.

**Rejected.**
- **Keeping the wrappers for import brevity** — a per-service shim that can
  drift, for two saved lines per call site; crm's and notify's D14 already made
  this call.
- **Deleting `internal/db` entirely** — the migrations embed must live in a
  prompts package and the migration guard tests are genuinely app-side; the
  package shrinks to its legitimate content instead.
- **Leaving the package doc comment for a later pass** — it would assert wiring
  that no longer exists (and already asserts a producer/consumer negative that is
  flatly wrong) in the same tree that changed it; docs move with the code.

**Verification.** Structural — no `R-XXXX-XXXX` ids (shim deletion + doc truth;
the behaviors are already pinned by the existing domain/MCP/migration suites and
the D15–D17 ids). Realized by a plan phase whose done bar is:

- `grep -n "func Open\|func Migrate" prompts/internal/db/db.go` returns no
  matches, while `grep -n "go:embed migrations" prompts/internal/db/db.go`
  still matches and `prompts/internal/db/prompts_schema_test.go` still exists.
- `grep -n "neither an event-plane producer nor a consumer\|the bare MCP surface"
  prompts/cmd/prompts/main.go` returns no matches, and
  `grep -c "Spec.Consumers\|Consumers" prompts/cmd/prompts/main.go` is ≥ 1.
- The suite stays green (design Conventions commands) with no assertion changes.
