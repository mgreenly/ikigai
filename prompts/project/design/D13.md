# Decision 13 — Self-serve the landing page's fonts and eliminate the FOUT

**Problem.** The landing page flashes: it paints with the fallback system font,
then visibly **swaps** to the web fonts once they load. Worse, the swap is driven
by the **dashboard's** assets, not prompts's own — which also violates the
suite-wide requirement that every service be 100% independent.

The cause is a chain of apex-absolute references:

1. `landing.html` links the stylesheet origin-absolute:
   `<link rel="stylesheet" href="/static/tokens.css">`. A leading-slash URL
   resolves against the site root, so a browser on `/srv/prompts/` requests
   `https://<host>/static/tokens.css` — the `/srv/prompts/` mount is gone. nginx
   routes apex `/static/` to the **dashboard** (`MOUNT=/`), so the page renders
   with the **dashboard's** tokens.css. prompts's own copy (served at
   `/srv/prompts/static/tokens.css` by the chassis static mount from `Spec.WWW`)
   is never requested.
2. The `@font-face` `src` in prompts's `tokens.css` is likewise origin-absolute
   (`url('/static/fonts/X.woff2')`) and resolves to the apex.
3. Every `@font-face` declares `font-display: swap`, which paints fallback and
   swaps to the web font after load — i.e. after first paint.
4. Nothing preloads the fonts.

Because the stylesheet itself loads from the apex, fixing only prompts's own
`tokens.css` would be inert — the browser never fetches it. The link must point
at prompts's own copy first.

`landing.html` is a Go `html/template` rendered by the chassis through `rt.WWW()`
(D16); the `<head>` the browser sees is the **rendered** output, so the
link/preload assertions verify the rendered `<head>`, not the raw template
source. The CSS itself is a static file served verbatim by the chassis static
mount (`GET /static/tokens.css`).

**Decision.** Make the landing page load prompts's **own** on-disk CSS and
fonts, swap-free. Four coordinated changes; no new dependency, no Go logic beyond
the chassis static mount (`Spec.WWW`, already serving `GET /static/`). The change
set spans `share/www/landing.html`, `share/www/static/tokens.css`, and
`etc/nginx.conf`.

1. **`landing.html`: relativize the stylesheet link.**
   `href="/static/tokens.css"` → `href="static/tokens.css"`. Document-relative,
   it resolves against the mount-root document URL (`/srv/prompts/`, canonical
   trailing slash) to `/srv/prompts/static/tokens.css` — prompts's own copy. (No
   `<base href>` is introduced; the template carries no `Mount` in its
   `landingData`, and the document-relative form needs none.)

2. **`tokens.css`: self-serve the fonts and stop the swap.** On all four
   `@font-face` blocks: `font-display: swap` → `font-display: optional`
   (`optional` forbids a post-first-paint swap — this removes the jar), and
   `src: url('/static/fonts/X.woff2')` → `url('fonts/X.woff2')` (relative to the
   now-self-served stylesheet → `/srv/prompts/static/fonts/X.woff2`).

3. **`landing.html`: preload the above-the-fold fonts** in `<head>`, beside the
   stylesheet link:

   ```html
   <link rel="preload" as="font" type="font/woff2" crossorigin
         href="static/fonts/space-grotesk.woff2">
   <link rel="preload" as="font" type="font/woff2" crossorigin
         href="static/fonts/ibm-plex-sans.woff2">
   ```

   The **display** (`Space Grotesk`) and **body** (`IBM Plex Sans`) families,
   painted immediately. `crossorigin` is **mandatory** (fonts fetch in CORS mode;
   without it the preload double-fetches and is wasted). The document-relative
   `href` resolves to the same `/srv/prompts/static/fonts/…` URL as the new
   `@font-face` `src`, so the preloaded bytes satisfy the `@font-face` request and
   first paint uses the real font. The mono family is **not** preloaded (minor /
   below the fold; `optional` already prevents its jar).

4. **`etc/nginx.conf`: session-gate prompts's own static path.** Today only the
   exact landing root (`= /srv/prompts/`) is session-gated; everything else under
   the mount — including `/srv/prompts/static/…` — falls to the bearer-gated
   prefix (`location /srv/prompts/` → `auth_request /_authn`). A browser carries
   the dashboard **session cookie**, not a bearer token, so it would get **401**
   on the self-served CSS/fonts and the page would be unstyled. Add a
   session-gated static location mirroring the dashboard's own static gate:

   ```nginx
   location /srv/prompts/static/ {
       auth_request /_session-authn;
       proxy_pass http://127.0.0.1:3002/static/;
       proxy_set_header Host $host;
       proxy_set_header X-Forwarded-Proto $scheme;
       proxy_http_version 1.1;
   }
   ```

   The prefix `/srv/prompts/static/` is replaced by upstream `/static/`, which
   the chassis static mount (`Spec.WWW`) serves. The exact `= /srv/prompts/` landing location,
   the bearer-gated `/srv/prompts/` prefix (MCP, PRM bootstrap), and the
   `= /srv/prompts/feed` 404 denial are unchanged.

**Supersession.** D13 corrects the apex-borrow that one shipped id currently pins
in its buggy form:
- **D10's R-LAND-CARB** asserts the rendered landing page references
  `/static/tokens.css` (origin-absolute). Its intent ("the page references
  prompts's *local* embedded tokens.css") is fulfilled — only the path form
  changes to the service-relative truth. The build updates that page-reference
  assertion to `href="static/tokens.css"`; the id is retained. The **other** half
  of R-LAND-CARB — that `tokens.css` ships on disk and the chassis static mount
  serves `GET /static/tokens.css` as `text/css`, plus the `*.woff2` fonts —
  is **unchanged** and remains the real-substrate proof that prompts serves its
  own assets.

**Boundary.** D13 owns the landing page's asset *loading* (the link, the
`@font-face` rules, the preload) and the nginx static gate. It does not change the
design-token values, the landing markup/copy (D11), the Home link (D12), or the
landing render/route (D10/D16). It corrects D10's reasoning, which assumed an
origin-absolute `/static/…` is rewritten by nginx back to `/srv/prompts/static/…`
— nginx strips inbound, it does not rewrite outbound CSS, so the apex-borrow was
real.

**Rejected.**
- **`font-display: block` / `fallback`.** `block` is FOIT (invisible text then
  pop-in); `fallback` still permits a late swap in its window. `optional` is the
  only value that forbids a post-paint swap. Rejected.
- **Keep the apex-absolute link, fix only the dashboard.** Would fix the flash but
  leaves prompts dependent on the dashboard serving its CSS — violating the
  independence requirement. Rejected.
- **Introduce `<base href="{{.Mount}}">`.** Would require threading `Mount` into
  the landing template's `landingData`; the document-relative link/preload need no
  base and resolve correctly at the canonical trailing-slash mount root. Rejected.
- **Origin-absolute preload `href` to match the old `src`.** Would re-pin the
  apex-borrow; the whole point is to self-serve. Rejected.

**Verification** (httptest against the chassis-rendered `landing.html` page and
the chassis static mount, both over the repo-real `prompts/share/www` tree, and a
fragment-grep against `etc/nginx.conf`).

- R-DFKP-IVZU — **No `@font-face` declares `font-display: swap`; all use
  `optional`.** prompts's embedded `tokens.css` contains `font-display: optional`
  in every `@font-face` block and **no** `font-display: swap` occurrence — a block
  left as `swap` (the jar-causing value) fails this. *(served `GET /static/tokens.css`)*
- R-DGSL-WNQJ — **Font `src` is service-relative, not origin-absolute.** prompts's
  `tokens.css` contains **no** `url('/static/fonts/` occurrence and each
  `@font-face` `src` uses `url('fonts/…woff2')` — so it resolves to prompts's own
  `/srv/prompts/static/fonts/…`. *(served `GET /static/tokens.css`)*
- R-DI0I-AFH8 — **The landing page links its own tokens.css service-relative.**
  `GET /` renders a `<head>` containing `href="static/tokens.css"` and **no**
  origin-absolute `href="/static/tokens.css"` — so the browser fetches prompts's
  own copy, not the apex. *(httptest, chassis-rendered `landing.html`, rendered `<head>`)*
- R-DJ8E-O77X — **The rendered `<head>` preloads the display and body fonts with
  `crossorigin`.** `GET /` returns a `<head>` containing a
  `<link rel="preload" as="font" type="font/woff2" crossorigin …>` for both
  `space-grotesk.woff2` and `ibm-plex-sans.woff2`, each `href` document-relative
  (`static/fonts/…`) and resolving to the same path as the `@font-face` `src`. A
  missing `crossorigin`, a missing family, or an origin-absolute href fails this.
  *(httptest, chassis-rendered `landing.html`, rendered `<head>`)*
- R-DKGB-1YYM — **nginx session-gates prompts's own static path.** `etc/nginx.conf`
  contains a `location /srv/prompts/static/` block whose body carries
  `auth_request /_session-authn;` and `proxy_pass http://127.0.0.1:3002/static/;`
  — so a browser session can fetch the self-served CSS/fonts. A static path left
  bearer-gated (only the inherited `/srv/prompts/` prefix) fails this. *(fragment
  grep against `etc/nginx.conf`)*
