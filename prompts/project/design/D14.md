# Decision 14 — Adopt the shared `registry` for all loopback addressing

## Decision

prompts stops writing loopback port numbers into its own source. Every port /
loopback URL it needs is resolved from the shared `registry` table **by service
name**, at the composition root, at startup. This is behavior-preserving: the
`registry` table pins the same numbers prompts hardcodes today (`prompts` → 3002,
`cron` → 3005, `crm` → 3100, `ledger` → 3101, `dropbox` → 3200, `scripts` →
3003), so no address changes; only the *source of truth* moves.

Three literal sites in `cmd/prompts/main.go` are the whole surface (a grep of
`prompts/` for `127.0.0.1:30` and the own-port default finds nothing else in
non-test source):

**1. The peer `feedDefaults` map (self + upstreams).** Today a package-level
literal map:

```go
var feedDefaults = map[string]string{
    "cron":    "http://127.0.0.1:3005/feed",
    "crm":     "http://127.0.0.1:3100/feed",
    "ledger":  "http://127.0.0.1:3101/feed",
    "dropbox": "http://127.0.0.1:3200/feed",
    "scripts": "http://127.0.0.1:3003/feed",
    "prompts": "http://127.0.0.1:3002/feed", // self-chaining (A12)
}
```

becomes derived from `registry.BaseURL`, one entry per `sources` name, so the
port literals vanish and the map cannot drift from `sources`:

```go
// feedDefaults is each upstream's loopback dev fallback (A11), derived once from
// the shared registry: registry owns the address, prompts appends the /feed path.
// Resolved at package init (composition-time), not per request.
var feedDefaults = func() map[string]string {
    m := make(map[string]string, len(sources))
    for _, src := range sources {
        m[src] = registry.BaseURL(src) + "/feed"
    }
    return m
}()
```

`registry.BaseURL(src)` returns `"http://127.0.0.1:<port>"` with **no path**; the
caller appends `"/feed"` (D03 — registry owns the address, not the routes). Each
value is byte-identical to the literal it replaces. `registry.BaseURL` panics on
an unknown name, which is correct here: every entry in `sources` is a
compile-time-known service, so an unknown name is a programming error, not a
runtime condition. This default still feeds the **override layer** unchanged:
`runConsumer` reads `config.EnvOr(os.Getenv, feedURLEnv(source), feedDefaults[source])`,
so `PROMPTS_<SRC>_FEED_URL` still wins where set; registry only supplies the
fallback the override lands on.

**2. The service's own port.** `appkit.Spec{… Port: 3002 …}` becomes
`Port: registry.MustPort("prompts")`. `MustPort` is the strict composition-root
form (D03): it panics if `"prompts"` is absent from the table, which at the apex
of `main` is exactly the loud failure we want over a silent zero. Any `PORT`
handling appkit itself layers on top of `Spec.Port` is unchanged — this Decision
only replaces the literal *default* the chassis receives.

**3. The dropbox import base URL default.** `config.EnvOr(os.Getenv,
"DROPBOX_BASE_URL", "http://127.0.0.1:3200")` becomes `config.EnvOr(os.Getenv,
"DROPBOX_BASE_URL", registry.BaseURL("dropbox"))`. `registry.BaseURL("dropbox")`
is exactly `"http://127.0.0.1:3200"` (D03), so the `DROPBOX_BASE_URL` override
keeps identical semantics; only the hardcoded fallback is now registry-sourced.

**Doc comments carry the literals too** (the package doc around the `feedDefaults`
block, and the `DROPBOX_BASE_URL` comment in `internal/prompt/dropbox.go`); those
`127.0.0.1:30xx` occurrences are scrubbed in the same change so the guard below is
honest and no stale literal implies a second source of truth.

**Wiring (structural, in scope, inside `prompts/`).** `prompts/go.mod` gains
`require registry v0.0.0` and a committed `replace registry => ../registry`,
mirroring the existing `eventplane` directive (a sibling source tree, never
tagged, keeping the build deterministic under `GOWORK=off`). `cmd/prompts/main.go`
adds `import "registry"`. `registry` is a zero-dependency leaf, so this adds
nothing to prompts's transitive module graph or `go.sum`.

**External preconditions (out of scope — NOT built here).** The `registry` module
at `../registry` and the repo-root `go.work use ./registry` entry live outside
`prompts/` and are owned at the repo root / by the registry project. They are
assumed satisfied; no phase in this plan creates, edits, or tests them. The
committed `replace` is what makes prompts build correctly under `GOWORK=off`
regardless of the workspace file.

This Decision is **structural at the module boundary** (the `require`/`replace`/
`import`) but carries **behavioral** proof for the three resolutions and a guard
that the literals are gone. Its acceptance is the green build plus the ids below.

## Rejected

- **Resolve per request / per consumer connect.** The table is compile-time data
  (a renumber means rebuilding the binaries anyway), so re-looking-up on every
  request or feed poll buys nothing and adds a hot-path call. Resolve once at the
  composition root and store the result — `feedDefaults` at package init,
  `Port`/`dropboxBase` in `main`/`registerRoutes` — matching how prompts already
  reads these exactly once.
- **Drop the env overrides now that registry supplies the address.**
  `PROMPTS_<SRC>_FEED_URL` and `DROPBOX_BASE_URL` are the deploy-time seam
  (pointing prompts at real peer feeds through nginx, or a test double); registry
  only replaces the *literal the override fell back to*. Keeping the override
  layer intact is required (product: overrides still win where set).
- **Use `registry.Port(name)` (checked form) at the composition root.** The
  checked `(int, bool)` form is for callers that want to handle a miss; at `main`
  a missing `"prompts"`/peer name is an unrecoverable build-time bug, so the
  panicking `MustPort`/`BaseURL` forms are the right fit (D03's two-tier policy).
- **Keep `feedDefaults` as a literal map but swap the values by hand.** That would
  leave the port numbers written in prompts's source (just relocated), missing the
  entire point; deriving the map from `registry.BaseURL` over `sources` removes the
  literals *and* couples the map's keys to `sources` so the two cannot drift.
- **Also route the `internal/web` nginx-fixture test (`web_test.go`) through
  registry.** That test asserts on committed `etc/nginx.conf` content — the nginx
  front door, an out-of-scope external concern — not on prompts's Go addressing.
  The no-literal guard is therefore scoped to non-test source; nginx keeps its own
  literal, which registry does not own.

## Verification

- R-RG01-PORT — **prompts's own port resolves from the registry to its unchanged
  value:** `registry.MustPort("prompts")` returns `3002` (the port prompts serves
  on today, now the value handed to `appkit.Spec.Port`). Falsifiable: any other
  value, or a panic for the known name `"prompts"`, fails it. Pins
  behavior-preservation of the own-port default, not merely "a port is set".
- R-RG02-FEED — **every peer feed default is the registry address plus `/feed`,
  byte-identical to the prior literal:** for each name in `sources`,
  `feedDefaults[name] == registry.BaseURL(name) + "/feed"`, and the concrete
  values are `cron`→`http://127.0.0.1:3005/feed`, `crm`→`…:3100/feed`,
  `ledger`→`…:3101/feed`, `dropbox`→`…:3200/feed`, `scripts`→`…:3003/feed`,
  `prompts`→`…:3002/feed` (self-chaining). Falsifiable: a wrong port, a missing
  source, a dropped/extra `/feed`, or drift between the map keys and `sources`
  fails it.
- R-RG03-DBOX — **the dropbox import base default is registry-sourced and
  unchanged:** with `DROPBOX_BASE_URL` unset, the base resolves to
  `registry.BaseURL("dropbox")` == `"http://127.0.0.1:3200"`; with
  `DROPBOX_BASE_URL` set, that override still wins. Falsifiable: a different
  default host/port, a trailing path/slash, or the override being ignored fails
  it.
- R-RG04-NLIT — **no bare loopback port literal remains in prompts's production
  source:** a guard test walking prompts's module `.go` files, excluding
  `_test.go`, finds no match for `127\.0\.0\.1:30` and no bare own-port literal
  `3002` used as an address/port — the `registry` table is the only place a
  prompts loopback port is written. Falsifiable: reintroducing any such literal
  (in code or a doc comment) fails it. Scope is non-test source by design (the
  nginx-fixture assertion in `web_test.go` is out of scope, per Rejected).
