# Decision 13 — MCP surface over `appkit/mcp`: `internal/mcp` becomes the tool table

**Problem.** notify's `internal/mcp` carries the copy-pasted JSON-RPC transport
(`ServeHTTP`, the `initialize`/`tools/list`/`tools/call` dispatch, the error
writer, the result envelopes) plus its own `health` and `reflection`
implementations — the same near-identical plumbing every service duplicated.
The chassis now owns all of it (appkit D8 transport, D9 standard tools,
consumed here as a fixed external contract); what is genuinely notify's is the
`send` tool — its descriptor, schema, validation, and the ntfy publish hop.

**Decision.** `internal/mcp` is reshaped from a transport into a
**declaration**:

```go
// internal/mcp — what remains.
const Instructions = /* notify's MCP instructions string */

// Tools declares notify's one domain tool over the push client: send
// (descriptor + schema + handler behavior preserved — priority mapping,
// tags, click validation; its result/error shapes are the structured-MCP
// contract of D17). health and reflection are NOT declared — the chassis
// registers them (their inputs all come from Router accessors).
func Tools(client *push.Client) []mcp.Tool

// NewHandler assembles the /mcp http.Handler at the composition root:
// appkit/mcp.New over Instructions + Tools(client) + the Router-threaded
// Service/Version/Health/Events/Subscriptions.
func NewHandler(client *push.Client, rt *appkit.Router) (http.Handler, error)
```

- **Deleted:** the JSON-RPC plumbing (`jsonRPCRequest`, the `ServeHTTP`
  dispatch, the JSON-RPC error writer, the local `Identity` type), the local
  `toolHealth`/`toolReflection`/`renderSubscriptions`/
  `reflectionUnknownTypeError` implementations, and the local result-envelope
  helpers (replaced by the chassis result helpers).
- **Kept:** the descriptor/schema helpers (`desc`/`obj`/`typ`/…, now producing
  `mcp.Tool` values), the `send` description and input schema, and the `send`
  handler's validate-and-publish body (priority→1..5 mapping, tags join, click
  validation, no secret ever in an error). The result/error *shapes* this body
  returns are not owned here — they are the structured-MCP contract of **D17**
  (`structuredContent` on success, a declared `outputSchema`, typed error codes
  over the closed vocabulary).
- **The tool partition is unchanged:** same three tools (`send` declared;
  `health`, `reflection` chassis-registered), same descriptions, same
  instructions. The `send` handler constructor moves to the new seam; the
  result/error wire shapes it produces are set by D17, not by this Decision.

**Rejected.**
- **Keeping the local transport** — the status quo; a protocol fix would still
  be a many-service edit.
- **Declaring health/reflection in notify's table** — the chassis reserves the
  names (appkit D8) precisely so services cannot fork the standard tools;
  their inputs (Envelope, Registry, Subscriptions) are chassis-held already.
- **Dissolving `internal/mcp` into `cmd/notify`** — the send tool's schema,
  validation, and error mapping with their test harness are a real package,
  unlike the post-D12 web remnant.

**Verification** (through the composition-root seam — the appkit/mcp handler
built by `NewHandler` over a mock-ntfy `push.Client`).
- R-4IBU-MT7Z — `tools/list` through the assembled handler returns **exactly
  three** tools: `send` declared by notify **plus** `health` and `reflection`
  supplied by the chassis — proving the table/chassis partition (a table that
  still declares its own health/reflection fails `New`'s reserved-name check;
  a surface missing them fails the count).
