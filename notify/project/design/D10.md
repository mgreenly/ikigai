# Decision 10 — Prove no loopback-port literal survives, and guard the deploy artifacts against registry drift

**Decision.** The adoption in D9 is only complete if it is **enforced by a test**,
not by good intentions (the suite's "enforce with systems" principle). Two guards,
both living entirely under `notify/` and running in the ordinary
`cd notify && go test ./...`:

1. **A source-scan guard** proving no bare loopback-address string literal of the
   form `127.0.0.1:30xx` remains anywhere in notify's Go source.
2. **A drift guard** proving the two static deploy artifacts that *must* carry the
   literal port — `etc/manifest.env` and `etc/nginx.conf` — agree with the
   authoritative `registry` value, so a future renumber fails a notify test
   instead of silently reaching deploy.

**Why two guards, and why this split.** After D9 there are two distinct kinds of
port literal left in the `notify/` tree:

- **Go-source loopback URLs** (the peer-feed literals D9 removes) — these are the
  drift vector D9 eliminates, and guard (1) proves they are gone and stay gone.
- **Non-Go deploy artifacts** — `etc/nginx.conf` is an nginx fragment (nginx
  cannot call Go) and `etc/manifest.env` is generated deploy metadata; both
  legitimately contain the literal `3201`. `registry` cannot *own* their contents
  without generating them, which is out of scope. What it *can* do is make notify's
  own tests assert those artifacts match `registry`, converting a silent drift into
  a red test. Guard (2) does exactly that.

**Guard (1) — source-scan (`R-RGNL-4E5P`).** A test in `cmd/notify` (or a small
new `internal` guard package) walks every `*.go` file under the `notify/` module
and fails if any file's source contains a bare loopback-port address literal
(`127.0.0.1:30` followed by two digits). Two constraints make it robust:

- **It must not match itself.** The test assembles the forbidden needle from
  fragments at runtime (e.g. `"127.0.0.1:" + "30"`) rather than embedding the full
  literal, and skips its own filename, so the guard's own source can never trip it.
- **It scans real Go source, not comments-only exemptions.** A doc comment that
  merely mentions a port in prose (no `127.0.0.1:30xx` address) does not trip it;
  the needle is the loopback *address* form, which is what drifts. After D9 +
  phase 08 there are zero such literals, so the scan is green; reintroducing a
  hardcoded peer URL turns it red.

This guard covers the **string-literal** loopback URLs. It deliberately does
**not** try to flag the bare integer `3201` (an int literal like `3201` is
ambiguous — it could be any number — and grepping for it is noisy). The own-port
integer literal is instead eliminated by D9's `registry.MustPort("notify")` change
and proven gone by guard (2), which emits the manifest from `registry` and matches
the committed file.

**Guard (2) — deploy-artifact drift (`R-RGDR-4F6Q`).** Two existing tests are
re-pointed at `registry` so their expected port is *derived*, not hardcoded:

- **`cmd/notify/main_test.go`** — the manifest byte-equality test currently emits
  `manifest.Emit(manifest.Fields{... Port: 3201 ...})` and compares to the
  committed `etc/manifest.env`. Change the literal to
  `Port: registry.MustPort("notify")`. The emitted `PORT=3201` still matches the
  committed file today; if `registry` renumbers notify, `manifest.Emit` produces a
  different `PORT` line and the byte-equality assertion fails — surfacing that the
  committed `manifest.env` is stale.
- **`internal/web/nginx_test.go`** — the exact-match and static-location tests
  currently assert the substring `proxy_pass http://127.0.0.1:3201/`. Build the
  expected `proxy_pass` origin from `registry.BaseURL("notify")` (==
  `"http://127.0.0.1:3201"`) and assert the fragment contains
  `"proxy_pass " + registry.BaseURL("notify") + "/"` (and the `/static/` variant).
  The assertion holds today; a renumber makes the still-literal `etc/nginx.conf`
  disagree with `registry.BaseURL("notify")` and the test goes red.

Both re-pointed tests keep their **existing** behavioral assertions (the session
gate, the exact-match vs prefix distinction, the manifest's non-port fields); only
the **port value** they compare against stops being a hardcoded number and becomes
a `registry` call. This is what removes the last `127.0.0.1:3201` / `3201`
literals from notify's Go source while turning the static artifacts into
drift-caught, not drift-prone.

**Scope guard.** These guards add tests only — no production code beyond D9, no
new package required (a single guard test file is enough), no schema change (**no
migration**), and no edit to `etc/nginx.conf` or `etc/manifest.env` themselves
(their literals stay; the tests now police them). Nothing outside `notify/` is
touched.

**Rejected.**
- **Regenerating `etc/nginx.conf` / `etc/manifest.env` from `registry`.** Out of
  scope and unnecessary: nginx config generation and the manifest artifact are
  owned elsewhere; a drift-catching test delivers the safety without a codegen
  step or a cross-module dependency.
- **An AST/int-literal scan for the bare `3201`.** Too noisy to be falsifiable —
  `3201` carries no context that marks it as a port. Deriving the manifest and
  nginx expectations from `registry` (guard 2) proves the own-port literal is gone
  far more precisely.
- **Leaving the nginx/manifest tests as literal `3201` assertions.** They would
  keep two bare `30xx` literals in notify's Go source (violating D9's end-state)
  and, worse, would keep *passing* after a `registry` renumber — the exact silent
  drift this whole adoption removes.

**Verification.**
- R-RGNL-4E5P — **no bare `127.0.0.1:30xx` loopback-address string literal exists
  in notify's Go source.** A guard test walks the module's `*.go` files (skipping
  itself, needle assembled at runtime) and fails on any such literal. Falsifiable:
  reintroducing a hardcoded peer feed URL like `"http://127.0.0.1:3100/feed"` makes
  it red; with D9 + phase 08 applied it is green.
- R-RGDR-4F6Q — **the committed `etc/manifest.env` and `etc/nginx.conf` ports agree
  with `registry`.** The manifest test emits with `registry.MustPort("notify")` and
  byte-matches the committed `manifest.env`; the nginx tests assert the fragment
  contains `registry.BaseURL("notify") + "/"` proxy targets. Falsifiable: a
  `registry` value that differs from the committed artifacts' `3201` fails these
  tests (the intended drift alarm); matching values pass.
