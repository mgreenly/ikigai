# Decision 8 — Self-serve the landing page's fonts and eliminate the FOUT

**Problem.** The landing page flashes: it paints with the fallback system font,
then visibly **swaps** to the web fonts once they load. Worse, the swap is driven
by the **dashboard's** assets, not ledger's own — which also violates the
suite-wide requirement that every service be 100% independent.

The cause is a chain of apex-absolute references:

1. `landing.html` links the stylesheet origin-absolute:
   `<link rel="stylesheet" href="/static/tokens.css">`. A leading-slash URL
   resolves against the site root, so a browser on `/srv/ledger/` requests
   `https://<host>/static/tokens.css` — the `/srv/ledger/` mount is gone. nginx
   routes apex `/static/` to the **dashboard** (`MOUNT=/`), so the page renders
   with the **dashboard's** tokens.css. ledger's own copy (served at
   `/srv/ledger/static/tokens.css` by the chassis static mount over
   `share/www/static`, D10) is never requested.
2. The `@font-face` `src` in ledger's `tokens.css` is likewise origin-absolute
   (`url('/static/fonts/X.woff2')`) and resolves to the apex.
3. Every `@font-face` declares `font-display: swap`, which paints fallback and
   swaps to the web font after load — i.e. after first paint.
4. Nothing preloads the fonts.

Because the stylesheet itself loads from the apex, fixing only ledger's embedded
`tokens.css` would be inert — the browser never fetches it. The link must point
at ledger's own copy first.

**Decision.** Make the landing page load ledger's **own** CSS and fonts,
swap-free. Four coordinated changes; no new dependency, no Go logic (the chassis
static mount serves `GET /static/`, D10). The change set spans
`share/www/landing.html`, `share/www/static/tokens.css`, and `etc/nginx.conf`.

1. **`landing.html`: relativize the stylesheet link.**
   `href="/static/tokens.css"` → `href="static/tokens.css"`. Document-relative,
   it resolves against the mount-root document URL (`/srv/ledger/`, canonical
   trailing slash) to `/srv/ledger/static/tokens.css` — ledger's own copy. (No
   `<base href>` is introduced; `landing.html` carries no `Mount`, and the
   document-relative form needs none.)

2. **`tokens.css`: self-serve the fonts and stop the swap.** On all four
   `@font-face` blocks: `font-display: swap` → `font-display: optional`
   (`optional` forbids a post-first-paint swap — this removes the jar), and
   `src: url('/static/fonts/X.woff2')` → `url('fonts/X.woff2')` (relative to the
   now-self-served stylesheet → `/srv/ledger/static/fonts/X.woff2`).

3. **`landing.html`: preload the above-the-fold fonts** in `<head>`, beside the
   stylesheet link:

   ```html
   <link rel="preload" as="font" type="font/woff2" crossorigin
         href="static/fonts/space-grotesk.woff2">
   <link rel="preload" as="font" type="font/woff2" crossorigin
         href="static/fonts/ibm-plex-sans.woff2">
   ```

   The **display** (`Space Grotesk`) and **body** (`IBM Plex Sans`) families,
   painted immediately. `crossorigin` is **mandatory** (fonts fetch in CORS mode;
   without it the preload double-fetches and is wasted). The document-relative
   `href` resolves to the same `/srv/ledger/static/fonts/…` URL as the new
   `@font-face` `src`, so the preloaded bytes satisfy the `@font-face` request and
   first paint uses the real font. The mono family (`IBM Plex Mono`, both 400 and
   500 weights) is **not** preloaded (minor / below the fold; `optional` already
   prevents its jar).

4. **`etc/nginx.conf`: session-gate ledger's own static path.** Today only the
   exact landing root (`= /srv/ledger/`) is session-gated; everything else under
   the mount — including `/srv/ledger/static/…` — falls to the bearer-gated prefix
   (`location /srv/ledger/` → `auth_request /_authn`). A browser carries the
   dashboard **session cookie**, not a bearer token, so it would get **401** on
   the self-served CSS/fonts and the page would be unstyled. Add a session-gated
   static location mirroring the dashboard's own and wiki's precedent:

   ```nginx
   location /srv/ledger/static/ {
       auth_request /_session-authn;
       proxy_pass http://127.0.0.1:3101/static/;
       proxy_set_header Host $host;
       proxy_set_header X-Forwarded-Proto $scheme;
       proxy_http_version 1.1;
   }
   ```

   The prefix `/srv/ledger/static/` is replaced by upstream `/static/`, which the
   chassis static mount serves (over `share/www/static`, D10). The exact
   `= /srv/ledger/` landing location, the bearer-gated
   `/srv/ledger/` prefix (MCP, `/feed`), the unauthenticated PRM bootstrap
   (`= /srv/ledger/.well-known/oauth-protected-resource`), and the
   `@ledger_authn_500` re-emit helper are unchanged.

**Supersession.** D8 corrects the apex-borrow that two shipped ids currently pin
in their buggy form:
- **D1's R-LAND-7G2H** asserts `landing.html` links `href="/static/tokens.css"`.
  Its intent ("links ledger's *local* tokens.css") is fulfilled — only the path
  form changes to the service-relative truth. The build updates that assertion to
  `href="static/tokens.css"`; the id is retained.
- **D3's R-ASST-3T7V** asserts `tokens.css` contains `url('/static/fonts/…')`.
  Same: the build updates those assertions to the `url('fonts/…')` form; the id is
  retained.
- **D2's R-ROUT-4P8Q** (`/static/tokens.css` served with `text/css`) and **D3's
  R-ASST-5W9X** (the four `/static/fonts/*.woff2` served with `font/woff2`) are
  **unchanged** in intent — after D10 the chassis static mount over
  `share/www/static` serves them — and remain the real-substrate proof that ledger
  serves its own assets.

**Boundary.** D8 owns the landing page's asset *loading* (the link, the
`@font-face` rules, the preload) and the nginx static gate. It does not change the
design-token values, the landing markup/copy (D6), or the landing handler/route
(D1/D2). It corrects D3's reasoning, which assumed an origin-absolute `/static/…`
is rewritten by nginx back to `/srv/ledger/static/…` — nginx strips inbound, it
does not rewrite outbound CSS, so the apex-borrow was real.

**Rejected.**
- **`font-display: block` / `fallback`.** `block` is FOIT (invisible text then
  pop-in); `fallback` still permits a late swap in its window. `optional` is the
  only value that forbids a post-paint swap. Rejected.
- **Keep the apex-absolute link, fix only the dashboard.** Would fix the flash but
  leaves ledger dependent on the dashboard serving its CSS — violating the
  independence requirement. Rejected.
- **Introduce `<base href="{{.Mount}}">`.** Would require threading `Mount` into
  the landing template's data; the document-relative link/preload need no base and
  resolve correctly at the canonical trailing-slash mount root. Rejected.
- **Origin-absolute preload `href` to match the old `src`.** Would re-pin the
  apex-borrow; the whole point is to self-serve. Rejected.

**Verification** (httptest over the landing handler and the chassis static mount
on the repo-real `share/www` tree, D10, and a fragment-grep against
`etc/nginx.conf`, mirroring the existing R-NGNX tests).

- R-7AW0-4QF8 — **No `@font-face` declares `font-display: swap`; all use
  `optional`.** ledger's embedded `tokens.css` contains `font-display: optional`
  in every `@font-face` block and **no** `font-display: swap` occurrence — a block
  left as `swap` (the jar-causing value) fails this. *(served `GET /static/tokens.css`)*
- R-7DBS-W9WM — **Font `src` is service-relative, not origin-absolute.** ledger's
  `tokens.css` contains **no** `url('/static/fonts/` occurrence and each
  `@font-face` `src` uses `url('fonts/…woff2')` — so it resolves to ledger's own
  `/srv/ledger/static/fonts/…`. *(served `GET /static/tokens.css`)*
- R-7EJP-A1NB — **The landing page links its own tokens.css service-relative.**
  `GET /` renders a `<head>` containing `href="static/tokens.css"` and **no**
  origin-absolute `href="/static/tokens.css"` — so the browser fetches ledger's own
  copy, not the apex. *(httptest `LandingHandler`)*
- R-7FRL-NTE0 — **The rendered `<head>` preloads the display and body fonts with
  `crossorigin`.** `GET /` returns a `<head>` containing a
  `<link rel="preload" as="font" type="font/woff2" crossorigin …>` for both
  `space-grotesk.woff2` and `ibm-plex-sans.woff2`, each `href` document-relative
  (`static/fonts/…`) and resolving to the same path as the `@font-face` `src`. A
  missing `crossorigin`, a missing family, or an origin-absolute href fails this.
  *(httptest landing handler)*
- R-7GZI-1L4P — **nginx session-gates ledger's own static path.** `etc/nginx.conf`
  contains a `location /srv/ledger/static/` block whose body carries
  `auth_request /_session-authn;` and `proxy_pass http://127.0.0.1:3101/static/;`
  — so a browser session can fetch the self-served CSS/fonts. A static path left
  bearer-gated (only the inherited `/srv/ledger/` prefix) fails this. *(fragment
  grep against `etc/nginx.conf`)*
