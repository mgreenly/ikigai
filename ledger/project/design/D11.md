# Decision 11 — MCP surface over `appkit/mcp`: `internal/mcp` becomes the tool table

**Problem.** ledger's `internal/mcp` carries the copy-pasted JSON-RPC transport
(`ServeHTTP`, the `initialize`/`tools/list`/`tools/call` dispatch, the JSON-RPC
error writer, the `jsonRPCRequest`/`toolCallParams` types, the result-envelope
helpers, the local `Identity` type) plus its own `health` and `reflection` tool
implementations (`toolHealth`, `toolReflection`, `renderSubscriptions`,
`reflectionUnknownTypeError`) — the same near-identical plumbing every service
duplicated. The chassis now owns all of it (appkit D8 transport, D9 standard
tools, consumed here as fixed external contracts). What is genuinely ledger's is
its **seven domain tools** — `record`, `reverse`, `reconcile`, `balance`,
`register`, `get`, `describe` — their descriptors, schemas, arg
validation/normalization (account canonicalization, date parsing, period
expansion, elision well-formedness), and the ledger-sentinel → wire-error mapping.

**Decision.** `internal/mcp` is reshaped from a transport into a **declaration**:

```go
// internal/mcp — what remains.
const Instructions = "Double-entry bookkeeping over an immutable journal. Call " +
    "describe first for the account model, then use record, balance, and register."

// Tools declares ledger's seven DOMAIN tools over the domain Service:
// record, reverse, reconcile, balance, register, get, describe — descriptors,
// schemas, and handler bodies unchanged in wire content. health and reflection
// are NOT declared — the chassis registers them (their inputs all come from
// Router accessors: Envelope/identity for health, Events/Subscriptions for
// reflection).
func Tools(svc *ledger.Service) []mcp.Tool

// NewHandler assembles the /mcp http.Handler at the composition root:
// appkit/mcp.New over Instructions + Tools(svc) + the Router-threaded
// Service/Version/Health/Events/Publishes/Subscriptions.
func NewHandler(svc *ledger.Service, rt *appkit.Router) (http.Handler, error)
```

- **Deleted** from `internal/mcp`: the JSON-RPC plumbing (`ServeHTTP`,
  `jsonRPCRequest`, `toolCallParams`, `handleToolCall`, `writeJSONRPCResult`/
  `writeJSONRPCError`, `idOrNull`), the local `Identity` type, the local
  result-envelope helpers (`toolResultText`/`toolResultErr`/`toolResultJSON`,
  replaced by the chassis `appkitmcp.TextResult`/`ErrorResult`/`JSONResult`), and
  the local `health`/`reflection` implementations (`toolHealth`, `toolReflection`,
  `renderSubscriptions`, `reflectionUnknownTypeError`) — the chassis renders those.
- **Kept:** the seven domain tool descriptors and schema helpers
  (`desc`/`obj`/`typ`/`typd`/`enumd`/`array`/`periodSchema`, now producing
  `mcp.Tool` values), each domain handler's validate-normalize-dispatch body
  (`toolRecord`/`toolReverse`/`toolReconcile`/`toolBalance`/`toolRegister`/
  `toolGet`/`toolDescribe`), the date/period parsing (`parseDay`, `parsePeriod`,
  `expandBucket`), the transaction JSON renderer (`transactionJSON`), and the
  ledger-sentinel → wire-error mapping (`translateLedgerError`, `jsonEscape`) —
  the error-envelope **content** contract is unchanged, so the wire bytes agents
  see for a domain error (`{"error":{"code":"unbalanced",…}}` etc.) are identical.
- **The `dispatchTool` switch collapses into per-tool `Handler` closures.** Each
  of the seven `mcp.Tool` entries wraps its existing `toolX(ctx, svc, args)`
  function in the chassis handler signature
  `func(ctx, args json.RawMessage, _ server.Identity) (map[string]any, error)`;
  the domain tools ignore identity (only `health`, now chassis-owned, reads it).
- **The wire surface for tested paths is unchanged:** `tools/list` returns the
  same nine tools (seven ledger-declared + chassis `health`/`reflection`), same
  descriptions and schemas; `tools/call` returns the same success and
  error-envelope bytes for every behavior the existing tests assert. The existing
  behavioral tests (`TestRecordGetReverseReconcile_EndToEnd`,
  `TestRecord_ErrorsSurfaceAsToolErrors`, `TestBalance_PeriodBucketAndRange`,
  `TestReflection`, `TestHealth`) keep their assertions and swap only how the
  handler is constructed — driving the assembled `appkit/mcp` handler through its
  real `ServeHTTP` seam (built over a migrated in-memory `ledger.Service`, mirroring
  notify's `server.New`/`Register` test harness).

**Two chassis-owned wire details change on untested paths (accepted).** Both are
consequences of adopting the shared transport uniformly, match every converted
service, and are covered by no existing ledger test:
- **`initialize` `serverInfo`.** The old transport hard-coded
  `{"name":"Ledger","version":"1"}`; the chassis emits `{"name": rt.Service()
  (== "ledger"), "version": rt.Version()}`. The `instructions` string is preserved
  verbatim via the `Instructions` const. No ledger test drives `initialize`.
- **Malformed-args error class.** The old `handleToolCall` rendered a handler's
  returned error (e.g. a `json.Unmarshal` failure on the `arguments` object) as an
  `isError:true` **tool result**; the chassis renders a returned error as a
  JSON-RPC **`-32602`** transport error. No ledger test passes malformed argument
  JSON, so no asserted path changes; the domain handlers keep returning their
  unmarshal error, now rendered by the chassis.

**Rejected.**
- **Keeping the local transport** — the status quo; a protocol fix would still be
  a many-service edit.
- **Declaring `health`/`reflection` in ledger's table** — the chassis reserves the
  names (appkit D8) precisely so services cannot fork the standard tools; their
  inputs (Envelope, Events registry, Subscriptions) are chassis-held already, and
  declaring either locally fails `appkitmcp.New`'s reserved-name check.
- **Preserving the old `serverInfo`/malformed-args wire bytes with a shim** — a
  per-service deviation from the shared transport for two untested edge cases;
  every converted service takes the chassis behavior, and re-forking it here is the
  drift the conversion removes.
- **Dissolving `internal/mcp` into `cmd/ledger`** — the seven domain tools'
  schemas, arg normalization (account canonicalization, period expansion), and
  sentinel→wire mapping with their test harness are a real package, unlike the
  post-D10 web remnant.

**Verification** (through the composition-root seam — the appkit/mcp handler built
by `NewHandler` over a migrated in-memory `ledger.Service`).
- R-52PA-OE6N — `tools/list` through the assembled handler returns **exactly nine**
  tools: the seven ledger domain tools (`record`, `reverse`, `reconcile`,
  `balance`, `register`, `get`, `describe`) declared by ledger **plus** `health`
  and `reflection` supplied by the chassis — proving the table/chassis partition
  (a table that still declares its own `health`/`reflection` fails `New`'s
  reserved-name check; a surface missing them fails the count).
