# Decision 14 — Delete the chassis shims (`internal/db` wrappers) and true up the doctrine doc

**Decision.** Two cleanups that land the reference shape whole, mirroring crm's
and notify's shim deletion:

1. **`internal/db` shrinks to its legitimate content.** The `Open` and `Migrate`
   wrapper functions in `gmail/internal/db/db.go` are deleted — appkit owns DB
   opening and the migration run (`Spec.Migrations: db.FS`). What remains is what
   genuinely must live in a gmail package: the embedded migration set (`FS`;
   `go:embed` cannot cross modules) and the outbox byte-equality guard
   (`migrations_outbox_test.go`, which asserts `003_outbox.sql` contains
   `outbox.SchemaSQL` verbatim) plus the load test (`migrations_load_test.go`,
   which already drives `appkitdb.LoadMigrations(FS, "migrations")`). The one
   caller of the wrapper — `internal/gmail/sync_test.go`'s `openTestDB`, which
   calls `db.Migrate(ctx, conn)` — calls appkit's db package directly instead:
   `migs, _ := appkitdb.LoadMigrations(db.FS, "migrations")` then
   `appkitdb.Migrate(ctx, conn, migs)`. No test *assertion* changes — only
   harness plumbing. (`sync_test.go` already opens its handle with a raw
   `sql.Open`, so the deleted `db.Open` has no caller.)

2. **`gmail/AGENTS.md` (and its `CLAUDE.md` symlink — one file, edit
   `AGENTS.md`) states the converted truth**, per the suite doctrine (docs state
   current truth; purge what the change falsified; no archaeology). The falsified
   claims to remove and replace:
   - the **embedded-assets** landing description ("bearer-gated MCP surface … and
     a session-cookie-gated human web landing page" is fine, but any implication
     the page/assets live in the binary is false) → the `share/www` web surface
     served through `Spec.WWW` and `rt.WWW()`, static auto-mounted by the chassis
     (D9);
   - the **local MCP transport** framing → `internal/mcp` as the ten-tool
     declaration (`Instructions` + `Tools(client)` + `NewHandler`) over the
     `appkit/mcp` chassis transport, with `health`/`reflection` chassis-owned
     (D10);
   - the **`internal/db` wrapper** framing → `internal/db` holds only the
     embedded migration set and the outbox/load guards; SQLite open and migration
     execution are appkit concerns (D14 item 1);
   - the **`internal/gmailapp` composition** framing (if present) → the
     `appkit.Spec` is declared inline at `cmd/gmail/main.go` as `gmailSpec()`
     (D13);
   - the **hardcoded port** framing → the loopback port resolves via
     `registry.MustPort("gmail")` at the composition root (D11), and the
     `etc/manifest.env`/`etc/nginx.conf` literals are `registry`-drift-guarded
     (D12).

   The current-truth AGENTS.md should also carry a short **Composition Root**
   note (like crm/notify) and keep the standing facts that remain true: gmail is
   a loopback-only Gmail connector; nginx is the sole trust boundary; it runs no
   token logic; its domain work is the normal-mailbox MCP surface, the Gmail
   History-API poll daemon (`Workers`), and the `mail.*` event-plane producer
   (`Producer`/`Feed`); the `cmd/consent` one-time OAuth CLI is unchanged; the
   three `GMAIL_*` secrets + `GMAIL_POLL_INTERVAL` reach the process only through
   the environment and are never read or logged by appkit.

**Rejected.**
- **Keeping the wrappers for import brevity** — a per-service shim that can drift,
  for two saved lines per call site; crm's and notify's D14 already made this
  call.
- **Deleting `internal/db` entirely** — the migrations embed must live in a gmail
  package and the outbox byte-equality guard is genuinely app-side; the package
  shrinks to its legitimate content instead.
- **Leaving `AGENTS.md` for a later pass** — the file would assert wiring that no
  longer exists in the same tree that changed it; docs move with the code.

**Verification.** Structural — no `R-XXXX-XXXX` ids (shim deletion + doc truth;
the behaviors are already pinned by the existing domain/MCP/producer suites and
the D9–D13 ids). Realized by a plan phase whose done bar is:

- `grep -n "func Open\|func Migrate" gmail/internal/db/db.go` returns no matches,
  while `grep -n "go:embed migrations" gmail/internal/db/db.go` still matches and
  `gmail/internal/db/migrations_outbox_test.go` still exists.
- `grep -rniE "go:embed|embedded (asset|template|landing)|json-rpc|jsonRPC|internal/gmailapp" gmail/AGENTS.md`
  returns no matches, and `grep -c "share/www\|registry\|appkit/mcp" gmail/AGENTS.md`
  reports at least 1.
- The suite stays green (design Conventions commands) with no assertion changes.
