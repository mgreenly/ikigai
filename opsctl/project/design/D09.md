# Decision 9 — restore reconstitutes the served tree's ownership to the service user

## Decision

`opsctl restore <app>` wipes `state/`, untars the archived `state/` back into
place, and (per D01) recreates `cache/` owned by the service user before the
deferred unit restart (`backup.go`). The restored `state/` returns with whatever
owner/group the snapshot tar captured — restore *replays archive metadata*, it
does not re-establish invariants. For the served `state/www` tree that is a
fidelity trap: a snapshot taken while `www` was mis-owned restores a tree the
sites process may not be able to read or write, even though the box is otherwise
healthy. Restore is precisely the recovery path where you cannot assume the
captured metadata was correct.

Restore already refuses to trust the archive for `cache/` ownership — it
re-establishes `cache/` as `<app>:<app>` rather than replaying it (D01). This
Decision extends that same "reconstitute to the invariant, don't replay" posture
to the served tree: **after `state/` is replaced (and alongside the existing
`cache/` recreate), reconstitute `state/www` ownership to the service user with a
guarded `ChownTree(app, app, l.WWWDir())`**, so the restored box's sites process
owns its tree regardless of the snapshot's captured metadata:

```go
// (*Opsctl).Restore, after replaceStateFromArchive succeeds and near the cache
// recreate/chown (D01), before the deferred unit restart. Guarded on existence so
// restore of a no-served-tree app is a no-op:
if _, err := os.Stat(l.WWWDir()); err == nil {
    if err := o.System.ChownTree(ctx, app, app, l.WWWDir()); err != nil {
        return fmt.Errorf("restore: chown served tree: %w", err)
    }
} else if !os.IsNotExist(err) {
    return fmt.Errorf("restore: stat served tree: %w", err)
}
```

There is **no** `web` group and **no** setgid to re-apply (the web-group model is
gone, per D7): the served tree's
only correctness property is that the service user owns it, so a plain recursive
`ChownTree(app, app, …)` is the whole reconstitution. The recreate happens before
the deferred restart, so the sites process sees the correct tree the moment the
unit comes back.

**Boundary.** D9 replaces the retired `ensureWWWPerms` re-assert in the restore
path with a guarded service-user chown of the served tree, after
`replaceStateFromArchive`. It does not change the state wipe/untar, the `cache/`
recreate (D01), the generation-sidecar removal, the cert restore (dashboard), or
the deferred restart.

## Rejected

- **Trust the archive's ownership for `state/www` (do nothing on restore).** Only
  correct if every snapshot was captured from a healthy tree — the one thing a
  recovery operation cannot assume. A snapshot taken mid-incident would restore a
  mis-owned tree. Rejected.
- **Re-apply a `web` group / setgid on restore (the prior model).** The `web`
  web-group model is gone (D7); nginx does not read the served tree. The only
  ownership that matters is the service user's, so reconstituting `<app>:<app>` is
  the complete fix. Rejected.
- **Chown the entire restored `state/` tree to `<app>:<app>` (as deploy does).**
  Restore deliberately trusts the archive for the rest of `state/` (its DB and
  sidecars were captured `<app>:<app>` from a running box); only `cache/` (D01)
  and now the served tree are reconstituted. A blanket state chown here would be a
  broader behavior change than this recovery fix warrants. Rejected.
- **Rely on the next `deploy` (D8) to fix ownership post-restore.** Leaves the
  freshly-restored box potentially unservable until the operator happens to
  deploy. Restore must stand on its own. Rejected.

## Verification

- R-3NXM-O0S2 — **Restore reconstitutes the served tree to the service user after
  replacing state.** After `Restore` of an app whose `state/www` exists, the fake
  `System` records a `ChownTree(app, app, WWWDir())` ordered **after** the state
  replacement and **before** the deferred unit restart, in addition to the existing
  `cache/` chown (D01) — and records **no** `ChownTree(_, "web", _)` and **no**
  `Chmod`. Falsifiable: the prior restore recorded `ChownTree(app, "web", WWWDir())`
  + tier-dir `Chmod(02750)`. *(unit: temp `OPSCTL_ROOT` + fake `System`)*
- R-B0E0-BRXO — **Real-substrate (live box `int.ikigenba.com`).** After a real
  `opsctl restore sites <key>` on the box, an anonymous
  `GET https://int.ikigenba.com/srv/sites/public/<published-site>/` returns **200**
  — the restored tree is owned by and servable through the sites process regardless
  of the snapshot's captured metadata. Cannot be falsified by the fake `System`;
  requires a real restore + live HTTP fetch. Operator-verified out-of-loop.
