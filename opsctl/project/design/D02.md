# Decision 2 — Stage unpacks into a temp dir on the OPSCTL_ROOT filesystem

## Decision

`opsctl stage <app> <version> --artifact <bundle>` unpacks the release bundle
into a scratch directory, preflights the bundled binary, then moves the three
tiers (`libexec/<app>-<version>`, `etc/<version>`, `share/<version>`) into the
app tree under `OPSCTL_ROOT`. The move is a rename: `replacePath`
(`internal/opsctl/deploy.go`) calls `os.Rename(src, dst)` to place each tier.

Today the scratch directory is created with `os.MkdirTemp("", "opsctl-stage-*")`
(`deploy.go:40`). The empty first argument means the OS temp dir — `$TMPDIR`,
else `/tmp`. On the real box `/tmp` and `OPSCTL_ROOT` (`/opt`) are **different
filesystems**, so `os.Rename` from the scratch tree into the app tree fails with
`EXDEV` (cross-device link) and every `stage` aborts.

This Decision fixes the seam: **the stage scratch directory is created on the
same filesystem as `OPSCTL_ROOT`**, so the tier rename is always intra-device.
The staging parent is `Layout.Root` (the configured `OPSCTL_ROOT`, default
`/opt`), which every stage destination already lives under. A small helper names
the seam so it is directly testable:

```go
// stagingParent returns the directory the stage scratch tree is created in.
// It is Layout.Root so os.Rename of a placed tier into the app tree never
// crosses a filesystem boundary. NEVER "" (the OS temp dir), which is the EXDEV
// bug this replaces.
func (l Layout) stagingParent() string { return l.Root }

// in (*Opsctl).Stage, replacing os.MkdirTemp("", "opsctl-stage-*"):
tmp, err := os.MkdirTemp(l.stagingParent(), "opsctl-stage-*")
```

`Layout.Root` is guaranteed non-empty by `NewLayoutSys` (it defaults `""` to
`/opt`), so the staging parent is always a concrete path on the install
filesystem. The `defer os.RemoveAll(tmp)` cleanup is unchanged: the scratch tree
is still removed on every exit, it simply lives under `OPSCTL_ROOT` for its
lifetime instead of under `/tmp`.

Nothing else about `stage` changes: the unpack, preflight, SHA collision guard,
and tier placement are all as they are; only the scratch directory's filesystem
moves.

## Rejected

- **Copy the tiers instead of renaming (`io.Copy` + remove).** A copy works
  across devices but is slower, is not atomic, and needs its own partial-write
  cleanup. The rename is already correct once both ends share a filesystem;
  keeping it and moving the scratch dir is the smaller, safer change.
- **Fall back to copy only when `os.Rename` returns `EXDEV`.** Adds a second code
  path that almost never runs on the box (both ends will now always share a
  filesystem), so it would be effectively untested in practice. Rejecting it
  keeps one path that is exercised every time.
- **Point `$TMPDIR` at a dir under `/opt` in the systemd unit / env.** Fixes it
  for the unit's environment but not for an operator's interactive `sudo opsctl
  stage`, and couples correctness to an out-of-band env var instead of to
  opsctl's own layout. The scratch location is opsctl's decision to make, not the
  environment's.

## Verification

- R-65MT-7QEK — The stage scratch directory is created under `OPSCTL_ROOT`: the
  staging parent equals `Layout.Root` and is **never** the empty string / OS temp
  dir. Falsifiable against a fake layout: `stagingParent()` for a layout rooted at
  a test dir returns that dir (not `""`, not `os.TempDir()`); the buggy
  `os.MkdirTemp("", …)` form fails this because its parent is the OS temp dir, not
  the layout root. Pins the discriminating property — "on the install filesystem",
  not merely "some temp dir exists".
- R-66UP-LI59 — **Real-substrate (live box `int.ikigenba.com`).** On the box,
  where `/tmp` and `OPSCTL_ROOT` (`/opt`) are separate filesystems, `sudo opsctl
  stage <app> <version> --artifact <bundle>` completes and the tiers are placed,
  with **no** `EXDEV` / cross-device error. This claim — that a real cross-device
  topology no longer breaks the rename — cannot be falsified by a unit test on one
  temp filesystem (where source and destination trivially share a device); it
  requires a real box whose temp and install roots are genuinely distinct mounts.
  The observable proof is a completed `stage` (the version appears staged), not
  merely that the scratch path was computed.
