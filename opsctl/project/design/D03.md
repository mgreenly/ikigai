# Decision 3 — opsctl loads the box env file at startup

## Decision

Several verbs read box-operational configuration from the process environment:
`objectStore()` (`internal/opsctl/backup.go`) reads `IKIGENBA_BACKUP_BUCKET` and
`IKIGENBA_AWS_REGION` via `os.Getenv`, and `deploy` runs a mandatory pre-deploy
`backup` that depends on them. On the box those values live in
`/etc/ikigenba/env` (a systemd-style env file, managed out-of-repo). The systemd
**unit** for each app gets them via `EnvironmentFile=/etc/ikigenba/env`
(`internal/opsctl/templates.go`), but **opsctl itself is not run by systemd** —
the operator runs `sudo opsctl <verb>` interactively, so nothing loads that file
into opsctl's process. Interactive `deploy` / `backup` therefore find
`IKIGENBA_BACKUP_BUCKET` empty and fail their S3 step.

This Decision adds a startup step: **`main()` loads `/etc/ikigenba/env` into the
process environment before dispatching a verb**, so every verb sees the box env
whether opsctl was launched by systemd or by hand. A standalone loader parses the
file; `main` calls it with the fixed box path.

```go
// LoadEnvFile reads a systemd-style EnvironmentFile (KEY=VALUE per line, blank
// lines and lines beginning with '#' ignored) and sets each KEY into the
// process environment via os.Setenv. A variable ALREADY present in the
// environment is left untouched (the file is a fallback, so an operator's
// explicit `FOO=bar sudo opsctl …` and the systemd unit's own EnvironmentFile
// both still win). A missing file is not an error — off-box/dev has no
// /etc/ikigenba/env — so the loader returns nil for os.IsNotExist. A malformed
// line (no '=') is a hard error, surfaced to the caller.
func LoadEnvFile(path string) error

const boxEnvFile = "/etc/ikigenba/env"

// early in main(), before reading OPSCTL_ROOT / dispatching:
if err := LoadEnvFile(boxEnvFile); err != nil {
    fmt.Fprintf(os.Stderr, "opsctl: load %s: %v\n", boxEnvFile, err)
    os.Exit(2)
}
```

**Non-override semantics** are the load-bearing choice: the loader sets a KEY
only if it is not already set. That makes it safe under systemd (the unit's
`EnvironmentFile` already populated the vars; the loader is then a no-op for
them) and preserves an operator's explicit inline override. It also means the
loader never has to reason about precedence — "already set wins" is the whole
rule.

`objectStore()` and the rest are unchanged: they keep reading `os.Getenv`. The
existing "IKIGENBA_BACKUP_BUCKET is required" errors (`backup.go`) remain the
loud failure when the value is genuinely absent (no env file, var not exported) —
this Decision does not weaken them, it just makes the box's real value reach the
process on the interactive path.

## Rejected

- **Require the operator to `set -a; . /etc/ikigenba/env` before opsctl.** The
  status quo. It is an unwritten precondition the operator forgets, and its only
  symptom is a mid-deploy S3 failure. Making opsctl self-sufficient removes the
  footgun.
- **Read `/etc/ikigenba/env` from inside `objectStore()` on demand.** Scatters
  file IO into a leaf that today only reads env vars, and each new env-consuming
  verb would need the same graft. Loading once at startup keeps a single, obvious
  seam and matches how the value reaches the systemd-run unit (whole file, once).
- **Let the file override already-set vars (file wins).** Would silently
  countermand an operator's explicit `FOO=bar sudo opsctl …` and duplicate the
  systemd unit's own load with different precedence. "Already set wins" is the
  least-surprise rule and needs no precedence bookkeeping.
- **Fail hard when `/etc/ikigenba/env` is missing.** Off-box and in dev the file
  does not exist; opsctl's non-box verbs and its whole test suite must still run.
  A missing file is a normal no-op; only a *malformed* file is an error.

## Verification

- R-6AIE-QTDC — `LoadEnvFile` on a systemd-style file (KEY=VALUE lines, plus
  blank lines and `#` comment lines) sets each defined KEY into the process
  environment: after the call, `os.Getenv(KEY)` returns the file's value, and a
  commented / blank line contributes no variable. Falsifiable: an unset var stays
  empty if the loader ignores the file.
- R-6BQB-4L41 — A variable **already present** in the environment is **not**
  overwritten by the file: given `KEY` pre-set to `A` and the file assigning
  `KEY=B`, after `LoadEnvFile` `os.Getenv("KEY") == "A"`. Pins the non-override
  rule — a file-wins implementation returns `B` and fails this.
- R-6CY7-ICUQ — `LoadEnvFile` on a **non-existent** path returns `nil` (no error)
  and sets nothing. Falsifiable: an implementation that errors on missing files
  fails this, and it is what lets opsctl run off-box where `/etc/ikigenba/env` is
  absent.
- R-6FE0-9WC4 — **Real-substrate (live box `int.ikigenba.com`).** On the box,
  running `sudo opsctl` interactively **without** the operator first sourcing
  `/etc/ikigenba/env`, a verb that needs the box env resolves it: e.g.
  `IKIGENBA_BACKUP_BUCKET` is available to `backup`/`deploy` so the S3 step
  proceeds rather than failing "IKIGENBA_BACKUP_BUCKET is required". This proves
  `main` actually wires `LoadEnvFile` to the real `/etc/ikigenba/env`; it cannot
  be falsified by a unit test that hands the loader a temp path, because that
  never exercises the fixed box path + the interactive (non-systemd) launch. The
  observable proof is a verb that reaches its S3 step on the box with no manual
  env sourcing.
