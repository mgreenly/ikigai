# Decision 8 — deploy's state-ownership chown already owns the served tree; no separate www step

## Decision

`deploy` runs as root, and the `migrate` verb it invokes just before the swap
also runs as root, so a freshly-created DB and its `-wal`/`-shm`/`.generation`
sidecars come back owned `root:root`. The unprivileged service user then cannot
take a write lock, so `deploy` hands the tree back with a **recursive**
`ChownTree(app, app, l.StateDir())` (`deploy.go`) before the restart.

`state/www` lives under `state/`, so that one recursive sweep **already** leaves
the served tree owned `<app>:<app>` — which, under the current serving model
(D7), is exactly the correct end state. The sites process reads and writes the
tree as `<app>`, and nginx proxies to the process rather than reading the tree,
so app-user ownership is all that is required. **`deploy` therefore performs no
served-tree-specific perm step at all** — no `ensureWWWPerms`, no re-chown to a
`web` group, no setgid. The step D8 previously added (a post-chown re-assert of a
`web`/setgid invariant) existed only to *undo* this sweep's effect on the served
tree back to `<app>:web`; with the `web` model retired, that re-assert is deleted
and the sweep's natural result is kept.

```go
// deploy.go — the state-ownership chown stays exactly as it was; the follow-on
// ensureWWWPerms call that used to sit here is removed.
o.logf("chown %s -> %s:%s", l.StateDir(), app, app)
if err := o.System.ChownTree(ctx, app, app, l.StateDir()); err != nil {
    return fmt.Errorf("deploy: chown state dir: %w", err)
}
// (no served-tree perm step: state/www is under state/ and is now owned <app>:<app>)
```

**Boundary.** D8 removes the served-tree re-assert from `deploy` and keeps the
existing state-ownership chown (whose DB-locking purpose is unrelated and
unchanged). It does not change the migrate/stage/swap/restart steps, the apex
block render (D4), or any non-deploy verb.

## Rejected

- **Keep re-asserting a `web`/setgid invariant after the state chown.** The whole
  reason that re-assert existed — the sweep clobbered `state/www` from `<app>:web`
  back to `<app>:<app>`, locking nginx out — is gone: `<app>:<app>` is now the
  wanted state and nginx does not read the tree. Re-asserting a retired invariant
  is dead work. Rejected.
- **Narrow the state chown to exclude `state/www`.** Pointless now — `<app>:<app>`
  over the whole `state/` tree, `state/www` included, is precisely correct.
  Excluding it would add complexity to preserve nothing. Rejected.
- **Add an explicit `ChownTree(app, app, WWWDir())` for the served tree after the
  sweep.** Redundant: the recursive `ChownTree(app, app, StateDir())` already
  covers `state/www`. A second chown of a subtree the sweep just covered is a
  no-op. Rejected.

## Verification

- R-3MPQ-A91D — **Deploy leaves the served tree owned by the service user via the
  state sweep, with no `web`/setgid step.** After `Deploy` of a served-tree app,
  the fake `System` records the recursive `ChownTree(app, app, StateDir())` (which
  covers `WWWDir()`), and records **no** `ChownTree(_, "web", _)` and **no** `Chmod`
  of any www path anywhere in the deploy. Falsifiable: the prior `deploy.go`
  recorded a `ChownTree(app, "web", WWWDir())` + tier-dir `Chmod(02750)` ordered
  after the state sweep. *(unit: temp `OPSCTL_ROOT` + fake `System`)*
- R-AXY7-K8GA — **Real-substrate (live box `int.ikigenba.com`).** After a real
  `opsctl deploy sites <version>` on the box, an anonymous
  `GET https://int.ikigenba.com/srv/sites/public/<published-site>/` returns **200**
  — the deployed sites process serves the public tier through the front door. This
  cannot be falsified by the fake `System`; it requires a real deploy + live HTTP
  fetch. Operator-verified out-of-loop.
