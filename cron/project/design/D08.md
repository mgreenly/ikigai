# Decision 8 ‚Äî Composition-root normalization: the `Spec` is declared inline in `cmd/cron/main.go`

**Problem.** cron is the only converted service whose `appkit.Spec` does not live
in its `main.go`. Today `cmd/cron/main.go` is a two-line shim
(`appkit.Main(cronapp.Spec())`) and the real declaration lives in a separate
package, `cron/internal/cronapp/spec.go`, behind a `Spec()` constructor. Every
other service on the chassis (the crm and notify references) declares its `Spec`
**inline, fully formed, in `cmd/<svc>/main.go`** ‚Äî `main()` calls
`appkit.Main(<svc>Spec())` where `<svc>Spec()` is a same-package function (notify)
or the literal is built inline in `main()` (crm). The extra `internal/cronapp`
package buys nothing: it holds one function whose only caller is `main()` and
whose only test caller is `main_test.go`. It is ceremony that makes cron
inconsistent with the suite for no behavioral reason, and it puts cron's
composition root a package away from where every reviewer expects it.

**Decision.** Dissolve `internal/cronapp`; declare cron's `Spec` in
`cmd/cron/main.go`.

1. **The `Spec` builder moves verbatim into `package main`.** A same-package
   function `cronSpec() appkit.Spec` carries the exact body of today's
   `cronapp.Spec()` ‚Äî the `App`/`Mount`/`Port`/`MCP`/`Feed`/`Migrations` identity,
   the dynamic `Publishes` provider (`event.Publishes(store)`), the
   `ManifestExtras`, the `Handlers` hook (store construction, the `GET /{$}`
   landing mount, the gated `POST /mcp` mount), the `Producer` hook (tick worker
   construction), and the `Workers` slice (the tick loop). `main()` becomes
   `appkit.Main(cronSpec())`. This mirrors notify's `notifySpec()` exactly.
2. **The closed-over wiring vars move with it.** `cronSpec()` declares
   `var store *crontab.Store` and `var worker *tick.Worker` at its top and the
   `Handlers`/`Producer`/`Workers`/`Publishes` closures capture them ‚Äî the same
   deferred-construction pattern the current `cronapp.Spec()` uses and that crm
   (`var svc *crm.Service`) and notify use at their composition roots. The `Spec`
   value is still **fully formed at the point it is returned**: no field is
   assigned after construction, no `.Handlers`/`.Workers` is mutated post-hoc ‚Äî
   the closures are captured at literal-build time exactly as today.
3. **`internal/cronapp/spec.go` (and the empty package) is deleted.** Nothing
   else imports it; `main.go` and `main_test.go` are its only consumers.
4. **The two test callers re-point to the same package.**
   `cmd/cron/main_test.go`'s manifest byte-equality test (`R-8IAN-FB87`) calls
   `cronSpec()` instead of `cronapp.Spec()` and drops the `cron/internal/cronapp`
   import ‚Äî no assertion changes, only the constructor name. The
   `TestCompositionRootMountsLandingWithoutIdentityWrapper` guard (currently in
   `internal/web/web_test.go`, reading `../cronapp/spec.go`) re-points its source
   path to `cmd/cron/main.go`; its assertions (the ungated `GET /{$}` landing
   mount string and the `rt.RequireIdentity`-gated `POST /mcp` mount string) are
   unchanged because those exact lines move verbatim into `main.go`.

**Behavior is unchanged.** This is a pure relocation: the emitted manifest, the
served routes, the tick worker, and the reflection surface are byte-identical
before and after. The move is proven by the **existing** ids (`R-8IAN-FB87`
manifest equality, `R-ROUT-*` landing/MCP mount partition, `R-4LKF-FB23` the
boot-from-opsctl-layout health smoke), which continue to pass against the
relocated composition root ‚Äî so this Decision mints **no new** `R-XXXX-XXXX` id.

**Rejected.**
- **Keeping `internal/cronapp`.** The suite-wide reference is an inline
  composition root; a one-function package that only `main()` calls is the exact
  inconsistency this round removes. crm and notify already prove the inline shape.
- **Inlining the literal directly in `main()` (crm's exact form) instead of a
  `cronSpec()` function.** cron's manifest byte-equality test needs to obtain the
  `Spec` value to feed `manifest.Emit`; a `cronSpec()` builder (notify's form)
  gives the test a seam without standing up the whole binary. Both forms satisfy
  "inline in `main.go`"; the builder form is chosen because a real test consumes
  the `Spec`.
- **Deferring the move until after the surface conversions (WWW/MCP/registry).**
  Those conversions all edit the composition root; doing them against
  `internal/cronapp` and then relocating would move already-transformed code and
  churn the relocated tests twice. Normalizing the root **first** means every
  later phase edits `main.go` ‚Äî the reference location ‚Äî and the WWW/MCP test
  relocations land in `cmd/cron` directly.

**Verification.** Structural ‚Äî no `R-XXXX-XXXX` ids (the relocation preserves the
behaviors already pinned by `R-8IAN-FB87`, `R-ROUT-2P8Q`/`R-ROUT-4R1S`/
`R-ROUT-6T3U`, and `R-4LKF-FB23`). Realized by a plan phase whose done bar is:

- `cron/internal/cronapp/` no longer exists
  (`ls cron/internal/cronapp 2>/dev/null` reports no such directory) and
  `grep -rn "internal/cronapp" cron --include=*.go` returns no matches.
- `cron/cmd/cron/main.go` declares `func cronSpec() appkit.Spec` and `main()`
  calls `appkit.Main(cronSpec())`
  (`grep -n "func cronSpec" cron/cmd/cron/main.go` matches).
- The suite stays green (design Conventions commands) with no assertion changes ‚Äî
  the manifest byte-equality and the landing/MCP mount-partition tests pass
  against the relocated root.
