# Decision 7 ŌĆö Self-serve the landing page's fonts and eliminate the FOUT

**Problem.** The landing page flashes: it paints with the fallback system font,
then visibly **swaps** to the web fonts once they load. Worse, the swap is driven
by the **dashboard's** assets, not cron's own ŌĆö which also violates the
suite-wide requirement that every service be 100% independent.

The cause is a chain of apex-absolute references:

1. `landing.html` links the stylesheet origin-absolute:
   `<link rel="stylesheet" href="/static/tokens.css">`. A leading-slash URL
   resolves against the site root, so a browser on `/srv/cron/` requests
   `https://<host>/static/tokens.css` ŌĆö the `/srv/cron/` mount is gone. nginx
   routes apex `/static/` to the **dashboard** (`MOUNT=/`), so the page renders
   with the **dashboard's** tokens.css. cron's own copy (served at
   `/srv/cron/static/tokens.css` by the chassis static mount over `share/www`,
   D9) is never requested.
2. The `@font-face` `src` in cron's `tokens.css` is likewise origin-absolute
   (`url('/static/fonts/X.woff2')`) and resolves to the apex.
3. Every `@font-face` declares `font-display: swap`, which paints fallback and
   swaps to the web font after load ŌĆö i.e. after first paint.
4. Nothing preloads the fonts.

Because the stylesheet itself loads from the apex, fixing only cron's own
`tokens.css` would be inert ŌĆö the browser never fetches it. The link must point
at cron's own copy first.

**Decision.** Make the landing page load cron's **own** service-local CSS and
fonts, swap-free. Four coordinated changes; no new dependency, no Go logic (the
chassis static mount serves `share/www/static`, D9). The change set spans
`share/www/landing.html`, `share/www/static/tokens.css`, and `etc/nginx.conf`.

1. **`landing.html`: relativize the stylesheet link.**
   `href="/static/tokens.css"` ŌåÆ `href="static/tokens.css"`. Document-relative,
   it resolves against the mount-root document URL (`/srv/cron/`, canonical
   trailing slash) to `/srv/cron/static/tokens.css` ŌĆö cron's own copy. (No
   `<base href>` is introduced; `landing.html` carries no `Mount`, and the
   document-relative form needs none.)

2. **`tokens.css`: self-serve the fonts and stop the swap.** On all four
   `@font-face` blocks (Space Grotesk, IBM Plex Sans, and the two IBM Plex Mono
   weights): `font-display: swap` ŌåÆ `font-display: optional` (`optional` forbids
   a post-first-paint swap ŌĆö this removes the jar), and
   `src: url('/static/fonts/X.woff2')` ŌåÆ `url('fonts/X.woff2')` (relative to the
   now-self-served stylesheet ŌåÆ `/srv/cron/static/fonts/X.woff2`).

3. **`landing.html`: preload the above-the-fold fonts** in `<head>`, beside the
   stylesheet link:

   ```html
   <link rel="preload" as="font" type="font/woff2" crossorigin
         href="static/fonts/space-grotesk.woff2">
   <link rel="preload" as="font" type="font/woff2" crossorigin
         href="static/fonts/ibm-plex-sans.woff2">
   ```

   The **display** (`Space Grotesk`) and **body** (`IBM Plex Sans`) families,
   painted immediately. `crossorigin` is **mandatory** (fonts fetch in CORS mode;
   without it the preload double-fetches and is wasted). The document-relative
   `href` resolves to the same `/srv/cron/static/fonts/ŌĆ”` URL as the new
   `@font-face` `src`, so the preloaded bytes satisfy the `@font-face` request and
   first paint uses the real font. The mono family (the two IBM Plex Mono weights)
   is **not** preloaded (minor / below the fold; `optional` already prevents its
   jar).

4. **`etc/nginx.conf`: session-gate cron's own static path.** Today only the exact
   landing root (`= /srv/cron/`) is session-gated; everything else under the mount
   ŌĆö including `/srv/cron/static/ŌĆ”` ŌĆö falls to the bearer-gated prefix
   (`location /srv/cron/` ŌåÆ `auth_request /_authn`). A browser carries the
   dashboard **session cookie**, not a bearer token, so it would get **401** on
   the self-served CSS/fonts and the page would be unstyled. Add a session-gated
   static location mirroring the dashboard's own and wiki's precedent:

   ```nginx
   location /srv/cron/static/ {
       auth_request /_session-authn;
       proxy_pass http://127.0.0.1:3005/static/;
       proxy_set_header Host $host;
       proxy_set_header X-Forwarded-Proto $scheme;
       proxy_http_version 1.1;
   }
   ```

   The prefix `/srv/cron/static/` is replaced by upstream `/static/`, which the
   chassis static mount (`Spec.WWW`, D9) serves. The exact `= /srv/cron/` landing
   location, the
   bearer-gated `/srv/cron/` prefix (MCP, the `= /srv/cron/feed` denial, the PRM
   bootstrap), and the `@cron_authn_500` rate-limit recovery are unchanged.

**Supersession.** D7 corrects the apex-borrow that two shipped D3 ids currently
pin in their buggy form:
- **D3's R-ASST-5X9Y** asserts the landing body contains
  `href="/static/tokens.css"`. Its intent ("the template references cron's *own*
  asset path, not a cross-service URL") is fulfilled ŌĆö only the path form
  changes to the service-relative truth. The build updates that assertion to
  `href="static/tokens.css"`; the id is retained (its no-external-reference checks
  are unaffected ŌĆö a relative path carries no `https://`/`dashboard`).
- **D3's R-ASST-3V7W** asserts cron's served `tokens.css` contains
  `url('/static/fonts/ŌĆ”')`. The build updates those font-`src` substrings to the
  `url('fonts/ŌĆ”')` form; the same id's proof that the chassis static mount serves
  `/static/tokens.css` with `text/css; charset=utf-8` is unchanged. The id is
  retained.
- **D3's R-ASST-7Z2A** (the chassis static mount serves the four
  `/static/fonts/*.woff2` with `font/woff2` and real `wOF2` payloads) is
  **unchanged** and remains the real-substrate proof that cron serves its own font
  assets.

**Boundary.** D7 owns the landing page's asset *loading* (the link, the
`@font-face` rules, the preload) and the nginx static gate. It does not change the
design-token values, the landing markup/copy (D6) or the Home link (D6), or the
landing handler/route (D1/D2). It corrects D3's reasoning, which assumed an
origin-absolute `/static/ŌĆ”` is rewritten by nginx back to `/srv/cron/static/ŌĆ”` ŌĆö
nginx strips inbound, it does not rewrite outbound CSS, so the apex-borrow was
real.

**Rejected.**
- **`font-display: block` / `fallback`.** `block` is FOIT (invisible text then
  pop-in); `fallback` still permits a late swap in its window. `optional` is the
  only value that forbids a post-paint swap. Rejected.
- **Keep the apex-absolute link, fix only the dashboard.** Would fix the flash but
  leaves cron dependent on the dashboard serving its CSS ŌĆö violating the
  independence requirement. Rejected.
- **Introduce `<base href="{{.Mount}}">`.** Would require threading `Mount` into
  the landing template's data; the document-relative link/preload need no base and
  resolve correctly at the canonical trailing-slash mount root. Rejected.
- **Origin-absolute preload `href` to match the old `src`.** Would re-pin the
  apex-borrow; the whole point is to self-serve. Rejected.

- R-2513-R056 ŌĆö **The rendered `<head>` preloads the display and body fonts with
