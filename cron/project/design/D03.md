# Decision 3 — cron's own Carbon design assets (shipped in `share/www/static`)

**Decision.** cron ships its **own** copy of the Carbon design assets on disk and
serves them through the chassis static mount; the landing page references those
service-local assets, never a cross-service URL. Concretely:

- A `cron/share/www/static/` directory holds a **vendored copy** of
  `design/tokens.css` (the Carbon CSS custom properties) and the woff2 fonts the
  Carbon type pairing needs: `space-grotesk.woff2` (display), `ibm-plex-sans.woff2`
  (body), `ibm-plex-mono-400.woff2` + `ibm-plex-mono-500.woff2` (mono labels) —
  the same set the dashboard already ships.
- The chassis auto-mounts the site's `static/` subtree at `GET /static/…` when
  `Spec.WWW` is true (D9), serving each asset with correct content types
  (`text/css; charset=utf-8` for `tokens.css`, `font/woff2` for the fonts). cron
  registers **no** static handler of its own.
- `tokens.css` declares its `@font-face` rules pointing at the fonts
  **document-relative** (`url('fonts/…woff2')`, resolving to the service's
  `/srv/cron/static/fonts/…` under the mount, D7), and the landing template links
  `tokens.css` document-relative (`href="static/tokens.css"`, D7) from the same
  service-local `static/` path. No asset is fetched from the dashboard or any
  other service at runtime.

**Why cron ships its own copy on disk.** The suite reverted binary embedding (D9):
web assets are release artifacts on disk, visible on the box and diffable in the
versioned release tree. `module cron` still cannot reach across the module
boundary into the sibling `design/` tree, so the tokens + fonts are vendored into
`cron/share/www/static/` — keeping cron self-contained with no runtime dependency
on another app's assets, now realized as shipped files rather than an embedded
`embed.FS`.

**Carbon fidelity.** The page consumes the **semantic** Carbon tokens
(`--color-*`, `--space-*`, type tokens), not raw ramps, per `design/carbon.md`:
monochrome neutrals; blue `#2563EB` reserved as the only signal color (used here
only for a focus ring / link if present, never decoratively); 4px spacing grid;
tight 4px radius; hairline borders over shadow. The landing card is minimal and
on-system: service name in **Display** (Space Grotesk, 56/1.04/700), version in
the **Mono · Label** token (IBM Plex Mono, 12/1.4/500), centered within the
1120px max content width.

**Staleness note (vendored copy).** Because `tokens.css` and the fonts are a
vendored copy, they can drift from `design/` over time. That is an accepted cost
of the module boundary (the dashboard lives with the same), and is out of scope to
automate here; if the design system changes, the copy is re-vendored as a new plan
phase. The shipped files copy the current `design/tokens.css` and the four woff2
fonts verbatim.

**Rejected.**
- **Embedding the assets in the binary (`//go:embed`)** — the suite's original
  approach, deliberately reverted (D9): embedded assets are invisible on the box
  and cost a rebuild to edit in dev.
- **Linking the dashboard's `/static/tokens.css` at runtime** — couples cron to
  the dashboard's asset URLs and breaks the self-contained model; a dashboard
  asset rename would silently unstyle cron.
- **A shared design module imported by every app** — the suite deliberately keeps
  each app a standalone module; vendoring the shipped copy is the established
  pattern (dashboard, crm, notify).
- **Serving the fonts from a CDN** — the box is single-tenant and may run with no
  external egress guarantees; assets must be local.

**Verification** (over the chassis static mount configured against the repo-real
`cron/share/www` tree — D9's `R-LQUM-TC2G` pins the shipped-tree serve).
- R-ASST-3V7W — a `GET /static/tokens.css` through the chassis static mount is
  served with status `200` and a CSS content type (`text/css; charset=utf-8`).
- R-ASST-5X9Y — the landing template's stylesheet link and the page's font
  references resolve to cron's **own** service-local `static/` path (the rendered
  HTML contains `href="static/tokens.css"` and contains **no**
  cross-service/dashboard asset URL — no `https://`, `http://`, or `dashboard`).
- R-ASST-7Z2A — the woff2 font assets ship on disk and are served: a `GET` for a
  font path (e.g. `/static/fonts/space-grotesk.woff2`) returns `200` with a
  `font/woff2` content type and a real `wOF2` payload.
