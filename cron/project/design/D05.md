# Decision 5 — Docs state current truth: state the landing-page truth in cron's doctrine

**Decision.** cron's service doctrine must state current truth: cron now serves a
human web landing page (D1–D4) alongside its machine surface. Per the suite
doctrine — *docs state current truth; purge what the change falsified, rewrite in
place, no archaeology, no footnotes* — this Decision **states the landing-page
truth in cron's doctrine prose**.

**Delta from the crm template (read this — cron differs).** The crm version of
this Decision *purges* a stale "**no UI**" line from `crm/AGENTS.md`/`CLAUDE.md`.
cron's situation is different and the phase must be accurate to it:

- cron has **no `AGENTS.md` and no `CLAUDE.md`** — there is no service-doctrine
  markdown file to edit, and (verified by `grep -rni "no UI" cron/`) **no stale
  "no UI" claim exists anywhere in `cron/`** to purge. cron was never in the
  brief's list of apps carrying that line (crm, ledger, notify, dropbox, wiki).
- cron's **doctrine prose lives in the `cmd/cron/main.go` package-doc header
  comment** — the block comment above `package main` that describes cron as "the
  loopback-only scheduled-event-emitter service behind nginx" that "trusts the
  X-Owner-Email / X-Client-Id headers … and performs no token logic of its own,"
  then enumerates the surface it wires (the crontab CRUD MCP tools, the tick
  worker, the LIVE Publishes provider). That header frames cron as **machine-only**
  and now omits a true fact: the human landing page.

So this Decision is **additive truth-stating, not a purge**: it extends that
package-doc header to record that cron also serves a human web landing page.

**What changes (a documentation comment only — no behavior, no Go logic):**

- In `cron/cmd/cron/main.go`, extend the `package main` doc-comment header to
  state, in cron's own voice, that cron serves an **MCP surface for agents**
  (bearer-gated) **and a human web landing page** (dashboard-session-cookie-gated)
  under `/srv/cron/`; it still runs **no token logic of its own** — nginx remains
  the sole trust boundary for both doors. Keep the existing "no token logic"
  statement (still true); add only the landing-page fact.
- Touch only the doc comment. The `Spec`, the hooks, the `Publishes` provider, and
  the tick `Producer`/`Workers` wiring are unchanged — this Decision adds **no**
  code, changes **no** behavior, and the `GET /{$}` route line itself is D2's
  (already landed by the time this phase runs).
- Do **not** add a changelog, a "previously…", or a footnote. State the present
  as if it were always so.

**Why this is its own phase, performed by the build loop — not by the doc
author.** The doc author (this design+plan workspace) deliberately does **not**
edit `cmd/cron/main.go` now: code and docs must evolve **together**, so stating
the landing-page truth is sequenced as a plan phase that the build loop performs
**after** the landing page exists (D1–D4 shipped). Doing it earlier would make the
doctrine claim a UI the code does not yet serve; doing it never would leave cron's
doctrine silent about a real surface.

**Scope guard.** Only the landing-page fact is added. cron's other doctrine — the
crontab CRUD MCP domain, the dynamic `cron.<name>` event types and the LIVE
Publishes provider, the header-trust model, "no token logic," the loopback bind,
the deploy/manifest sections — is **unchanged and still true**; this Decision does
not touch it.

**Verification.** This is a **documentation** Decision with no Go behavior change
(it edits only a doc comment), so it mints no `R-XXXX-XXXX` ids; it is realized by
a **structural** plan phase whose done bar is a content check rather than an
id-tagged test:

- `cron/cmd/cron/main.go`'s package-doc header states the landing-page truth: it
  mentions that cron serves a human web **landing page** under `/srv/cron/`
  alongside the MCP surface.
- cron's doctrine carries **no** false "no UI"-style machine-only-only claim:
  `grep -rni "no UI" cron/` continues to find nothing.
- The suite stays green (`bin/check-migrations cron` and `cd cron && go build
  ./...`/`go vet ./...`/`go test ./...` unaffected — this is a comment-only
  change), and `gofmt -l cron` prints nothing.
