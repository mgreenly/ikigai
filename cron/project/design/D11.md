# Decision 11 — Adopt `registry`: resolve cron's port by name and guard the deploy artifacts against drift

**Problem.** cron hardcodes its loopback port. `cronSpec()` sets `Port: 3005` as
an integer literal, and cron's Go source carries the bare loopback **address**
literal `127.0.0.1:3005` in its nginx-fragment tests' `proxy_pass` assertions.
"Which port a service answers on" is duplicated across the suite and drifts
silently — a renumber only surfaces at deploy, uncatchable by the compiler. The
shared `registry` library (`module registry`, the repo-root leaf module) is the
single source of truth for the table; cron references itself **by name** through
it, and its tests police the two static deploy artifacts that must carry the
literal port against that authority.

**Decision.** cron writes no loopback-port literals in its Go source and proves it
with tests. Three parts, all living entirely under `cron/` and running in the
ordinary `cd cron && go test ./...`.

1. **Module wiring (inside `cron/` only).** `cron/go.mod` gains
   `require registry v0.0.0` and the committed `replace registry => ../registry`,
   mirroring the existing `appkit` and `eventplane` in-repo replace-siblings.
2. **Resolve the own port by name, once, at the composition root.**
   `cronSpec()`'s `Port: 3005` becomes `Port: registry.MustPort("cron")` — the
   strict form that panics on an unknown own-name (a build-time programming error,
   not a recoverable runtime condition). Resolution happens **once** at boot, never
   per request; the table is compile-time data. This is behavior-preserving:
   `registry` carries `cron=3005`, so the resolved value is byte-identical to the
   literal it replaced.
3. **Two guards prove the adoption and catch drift.**
   - **Source-scan guard (`R-LUIB-YNAJ`).** A test in `cmd/cron` walks every
     `*.go` file under the `cron/` module and fails if any file's source contains
     a bare `127.0.0.1:30xx` loopback-address literal. It assembles the forbidden
     needle from fragments at runtime (`"127.0.0.1:" + "30"`) and skips its own
     filename, so the guard's own source cannot trip it. After part 4 there are
     zero such literals; reintroducing a hardcoded loopback URL turns it red.
   - **Deploy-artifact drift guard (`R-LVQ8-CF18`).** cron's two static artifacts
     that legitimately carry the literal port — `etc/manifest.env` (`PORT=3005`)
     and `etc/nginx.conf` (`proxy_pass http://127.0.0.1:3005/…`) — are re-pointed
     at `registry` in their tests so a future renumber fails a cron test instead
     of silently reaching deploy: the manifest byte-equality test already emits
     from `cronSpec().Port` (now `registry.MustPort("cron")`); the nginx fragment
     tests assert the `proxy_pass` origins are `registry.BaseURL("cron")` (==
     `"http://127.0.0.1:3005"`) rather than the bare literal string.
4. **De-literalize the nginx-fragment test assertions.** The nginx tests
   (relocated to `cmd/cron` by D9) currently assert the exact string
   `proxy_pass http://127.0.0.1:3005/;` (landing) and
   `proxy_pass http://127.0.0.1:3005/static/;` (static). Both are rebuilt from
   `registry.BaseURL("cron")` (e.g. `"proxy_pass " + registry.BaseURL("cron") +
   "/;"`). This removes the last `127.0.0.1:3005` string literals from cron's Go
   source (satisfying the source-scan guard) while turning the still-literal
   `etc/nginx.conf` into a drift-caught artifact. `etc/nginx.conf` and
   `etc/manifest.env` themselves are **not** edited — their literals stay; the
   tests now police them.

**Scope.** These changes add the `registry` dependency, one `Spec` field change,
and test-only guards; no schema change (**no migration**), no edit to
`etc/nginx.conf`/`etc/manifest.env`, nothing outside `cron/`. cron is a producer
only with **no peer-feed URLs to resolve** — unlike notify, cron reads no
`<APP>_<SRC>_FEED_URL`, so there are no peer-feed literals to remove and the
registry adoption is the own-port half alone.

**External precondition (OUT OF SCOPE — not built by any cron phase).** The
repo-root `go.work` must carry `use ./registry`, and the `registry` module must
exist and be green (it does: `registry.go` carries `{"cron", 3005, Core}`). No
cron Decision or phase edits `../go.work`, `../registry/`, `../bin/`, nginx, or
any sibling module.

**Rejected.**
- **Per-request resolution / a `registry` lookup in a handler.** The table is
  compile-time constant; resolve at the composition root, store the value, reuse
  it.
- **`registry.Port("cron")` (the checked form) at the composition root.** An
  unknown own-name is a build-time programming error; `MustPort`'s panic is the
  correct fail-loud form.
- **An AST/int-literal scan for the bare `3005`.** Too noisy to be falsifiable — a
  bare `3005` carries no context marking it a port. Deriving the manifest and
  nginx expectations from `registry` (the drift guard) proves the own-port literal
  is gone far more precisely.
- **Leaving the nginx tests as literal `127.0.0.1:3005` assertions.** They would
  keep bare loopback literals in cron's Go source (violating the source-scan
  guard) and keep *passing* after a `registry` renumber — the exact silent drift
  this adoption removes.

**Verification.**
- R-LTAF-KVJU — **cron's own listen port is `registry.MustPort("cron")`**, not a
  `3005` integer literal, at the composition root. Falsifiable: the emitted
  manifest `PORT` must equal `registry.MustPort("cron")`; a hardcoded `Port: 3005`
  in the Spec fails the intent (proven concretely by the drift guard, which emits
  from `cronSpec().Port` and byte-matches the committed `PORT=3005`).
- R-LUIB-YNAJ — **no bare `127.0.0.1:30xx` loopback-address string literal exists
  in cron's Go source.** A guard test in `cmd/cron` walks the module's `*.go`
  files (skipping itself, needle assembled at runtime) and fails on any such
  literal. Falsifiable: reintroducing a hardcoded loopback URL like
  `"http://127.0.0.1:3005/feed"` makes it red; with parts 2 and 4 applied it is
  green.
- R-LVQ8-CF18 — **the committed `etc/manifest.env` and `etc/nginx.conf` ports
  agree with `registry`.** The manifest test emits with `registry.MustPort("cron")`
  and byte-matches the committed `manifest.env`; the nginx tests assert the
  fragment's landing and static `proxy_pass` origins equal `registry.BaseURL("cron")`.
  Falsifiable: a `registry` value differing from the committed artifacts' `3005`
  fails these tests (the intended drift alarm); matching values pass.
