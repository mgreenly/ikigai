# Decision 12 — Delete the chassis shims (`internal/db` wrappers) and true up the doctrine header

**Decision.** Two cleanups that land the reference shape whole, mirroring crm's
and notify's shim deletion:

1. **`internal/db` shrinks to its legitimate content.** The `Open` and `Migrate`
   wrapper functions in `cron/internal/db/db.go` are deleted — appkit owns DB
   opening and the migration run (`Spec.Migrations: db.FS`, applied by appkit's
   forward-only runner + downgrade guard). What remains is what genuinely must
   live in a cron package: the embedded migration set (`FS`/`migrationsFS`;
   `go:embed` cannot cross modules) and the app-side guard tests —
   `migrations_load_test.go` (the real set loads through `appkit/db`) and
   `migrations_outbox_test.go` (cron's `003_outbox.sql` contains `outbox.SchemaSQL`
   verbatim). Both stay.
2. **The test harnesses call `appkit/db` directly.** Every test that stood a
   schema up through the deleted wrappers is re-pointed to `appkit/db`:
   `internal/db/db_test.go` (its `TestOpenAndMigrate`/`TestMigrate_IsIdempotent`/
   `TestCrontabNameCheck` gain a local `migrateCron` helper over
   `appkitdb.LoadMigrations(FS, "migrations")` + `appkitdb.Migrate`, opening via
   `appkitdb.Open`), and the cross-package harnesses in `internal/crontab`,
   `internal/event`, and `internal/tick` (each opens via `appkitdb.Open` and
   migrates via `appkitdb.LoadMigrations` + `appkitdb.Migrate`). **No test
   assertion changes** — only the open/migrate plumbing. (The `internal/mcp`
   harness was already moved off the wrappers when D10 rebuilt it over the
   assembled `appkit/mcp` handler.)
3. **The doctrine header states the converted truth.** cron has **no `AGENTS.md`
   and no `CLAUDE.md`** — unlike notify, there is no service-doctrine markdown to
   edit. cron's doctrine prose is the **`package main` doc-comment header in
   `cmd/cron/main.go`** (the same header D05 established the landing-page truth in).
   Per the suite doctrine (docs state current truth; purge what the change
   falsified; no archaeology), that header is trued up to describe the converted
   shape: the `appkit.Spec` is declared **inline in `cmd/cron/main.go`** (no
   `internal/cronapp`), the port resolves via `registry.MustPort("cron")`, the
   human landing page is served from the on-disk `share/www` tree through the
   chassis (`Spec.WWW`/`rt.WWW()`, not an embedded template), and the MCP surface
   is the `internal/mcp` **tool table** (`Instructions` + `Tools(store)` +
   `NewHandler`) over the shared `appkit/mcp` transport (not a hand-rolled
   JSON-RPC handler). The dynamic `Publishes` provider, the tick `Producer`/
   `Workers`, "no token logic," and the loopback/manifest facts stay as they are.

**Rejected.**
- **Keeping the wrappers for import brevity** — a per-service shim that can drift,
  for two saved lines per call site; crm's and notify's shim deletions already
  made this call.
- **Deleting `internal/db` entirely** — the migrations embed must live in a cron
  package (`go:embed` cannot cross modules) and the outbox-DDL byte-equality guard
  is genuinely app-side; the package shrinks to its legitimate content instead.
- **Editing a `cron/AGENTS.md`** — there is none to edit (D05 records that cron's
  doctrine lives in the `main.go` header); inventing one now would be scope the
  conversion does not need. The header is the truth surface and is where the
  update lands.
- **Leaving the header for a later pass** — it would assert (or omit) wiring that
  the same tree changed; docs move with the code.

**Verification.** Structural — no `R-XXXX-XXXX` ids (shim deletion + doc truth;
the behaviors are already pinned by the domain/MCP suites and the D9–D11 ids).
Realized by a plan phase whose done bar is:

- `grep -n "func Open\|func Migrate" cron/internal/db/db.go` returns no matches,
  while `grep -n "go:embed migrations" cron/internal/db/db.go` still matches and
  `cron/internal/db/migrations_outbox_test.go` and
  `cron/internal/db/migrations_load_test.go` still exist.
- `grep -rn "db\.Open\|db\.Migrate" cron --include=*.go` returns no matches
  outside the `internal/db` package's own files (the cross-package harnesses call
  `appkit/db` directly).
- `cron/cmd/cron/main.go`'s package-doc header mentions the `share/www` web
  surface and the `appkit/mcp` tool table, and `grep -rni "no UI" cron/` continues
  to find nothing.
- The suite stays green (design Conventions commands) with no assertion changes.
