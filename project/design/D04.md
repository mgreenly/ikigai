# Decision 4 — Version production: `bump` / `ship` emit `v`-prefixed SemVer

**Decision.** The version *producers* are brought in line with the `v`-everywhere,
full-SemVer-2.0 contract (D03). The committed `<svc>/VERSION` file becomes a
single `v`-prefixed SemVer line (`v0.7.1`), and the two tools change:

- **`bin/bump`** reads `<svc>/VERSION`, increments the requested field
  (`major|minor|patch`) **carrying the `v`**, and writes the `v`-prefixed result
  back (still a path-limited commit of only that file, still on `main`). It
  **rejects** a `VERSION` file that is not valid `v`-prefixed SemVer rather than
  silently coping (the old "bare X.Y.Z, no leading v" contract is gone). The
  pre-release identifier, when one is set, is preserved/advanced per SemVer.
- **`bin/ship`** builds `main` static for `linux/amd64` (`GOWORK=off`) and stamps
  the artifact with the **full** version: the `VERSION` core plus a `+<SHA>`
  build-metadata suffix derived from the built git commit. The produced
  `libexec/<svc>-v<full>` name (D02) and the binary's embedded `version` verb
  output both carry that full `v`-prefixed string.

The version embedded in the binary (via build `-ldflags`) is the same string
`ship` stamps, so `<svc> version`, the `libexec/` filename, and opsctl's deploy
preflight (D03 gate) all agree by construction.

**Rejected.**
- *Fix only the consumer side (opsctl) and leave `bump`/`ship` emitting bare
  numbers.* Then opsctl's `v`-everywhere gate would reject the suite's own
  artifacts. "Any legal SemVer" is a constraint on **both** ends — produce it and
  accept it. Rejected.
- *Carry `-rel`/`+SHA` as out-of-band metadata rather than in the version string.*
  Defeats the point of a single canonical identity the filesystem and the
  `version` verb share. Rejected.

**Verification.**
- R-44HU-2IOD — `bump patch` on a `VERSION` of `v0.7.1` writes `v0.7.2` (keeps the
  `v`); `bump` against a bare `0.7.1` file **fails** with a non-zero exit
  (rejects the old form). Substrate: `bin/bump.test.sh` (shell, real git repo
  fixture).
- R-45PQ-GAF2 — `ship` stamps the artifact with the full version: the built
  binary's `version` output and the staged `libexec/` filename both equal
  `v<core>+<sha>` for the built commit (the `+<sha>` matches the source commit,
  not a placeholder). Substrate: `bin/ship.test.sh` (shell) asserting the stamped
  string; the embedded-version round-trip is a build smoke against a real `go
  build`.
