# Decision 2 — Versioned binaries via `libexec/` + `bin/run` symlink swap

**Decision.** A release artifact is exactly one static binary, stored as
`/opt/<svc>/libexec/<svc>-v<semver>` (the filename carries the **full** v-prefixed
SemVer identity, including any `-rel`/`+SHA` — see D03). The live version is
selected by a single symlink, `/opt/<svc>/bin/run`, which **is** the atomic
cutover point:

```
bin/run → ../libexec/<svc>-v<semver>
```

Deploy: place the new `libexec/<svc>-v<new>` file, then repoint `bin/run` to it
with the existing `atomicSwap` (a `rename(2)` over the symlink — atomic, no
dangling window). Rollback: repoint `bin/run` back to the still-present
`libexec/<svc>-v<old>`. Prune: delete old `libexec/<svc>-v*` files, keeping the
newest N by D03 ordering (never deleting the live target or the rollback target).

This **replaces** the two-level `bin/run → current → releases/<v>/<svc>`
indirection: `releases/` (dir-per-version) and the intermediate `current` symlink
are removed. The external contract is unchanged — the systemd unit still execs
`/usr/local/bin/ikigenba-launch <svc>`, which execs the stable path
`/opt/<svc>/bin/run`; only what that path *points at* moves. `bin/run` stays a
**symlink** (not a wrapper script): launch-time logic, if any, belongs in the
existing `ikigenba-launch` layer, so a second wrapper would be redundant and would
force the version to be encoded in a script body.

opsctl deploy/rollback/prune retarget from `ReleaseBinary`/`CurrentLink` to
`LibexecBinary`/`RunLink` (D01). The collision guard (refuse re-deploying a
version whose artifact is already placed unless `--force`) keys on
`LibexecBinary(v)`.

**Rejected.**
- *Keep `releases/<v>/` dir + `current` flip.* The dir-per-release exists to swap
  a multi-file release atomically as a set; the suite's "one static binary" bet
  means a release is exactly one file, so the capability is one we've designed
  never to need. Two indirections for nothing. Rejected.
- *`bin/run` as a wrapper script.* Viable (atomicSwap replaces a regular file
  atomically too, and it has precedent), but it encodes the live version inside
  the script body (cutover = rewriting the script) or reads a pointer file
  (re-introducing the indirection we just deleted), and duplicates the
  `ikigenba-launch` wrapper role. Rejected in favor of a plain symlink.

**Verification.**
- R-3TIQ-ML04 — after deploy of `v<new>`, `bin/run` resolves to
  `libexec/<svc>-v<new>` and that file is the just-placed binary; `releases/` and
  `current` do not exist. Substrate: real filesystem (temp `OPSCTL_ROOT`).
- R-3UQN-0CQT — rollback from `v<new>` repoints `bin/run` to
  `libexec/<svc>-v<old>`, and `libexec/<svc>-v<old>` is still present (rollback
  never deleted it). Substrate: real filesystem.
- R-3VYJ-E4HI — `bin/run` resolves to a runnable file at **every** point of a
  deploy swap (assert before, and immediately after, the atomic repoint — there
  is no observable state where `bin/run` dangles). Substrate: real filesystem.
